**salesforce** 플러그인은 Salesforce CLI를 Claude Code와 통합하여
org 인증과 SOQL 쿼리를 포함한 모든 `sf` 명령을 실행할 수 있는
범용 CLI 에이전트를 제공합니다. JWT, 액세스 토큰, SFDX URL 인증 방식을 통해
헤드리스 컨테이너 환경을 지원합니다. 아래 사용 가이드는 CLI 에이전트를 통해
자연어로 계정, 영업 기회, 케이스, 연락처를 조회하는 방법을 안내합니다.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## 설치

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## 사전 요구 사항

- **Salesforce CLI** (`@salesforce/cli`)
- API 액세스가 가능한 **Salesforce org**
- 객체(케이스, 영업 기회, 계정, 연락처) 조회 권한이 있는 사용자 계정

워크스테이션에 Salesforce CLI를 설치합니다:

```bash
brew install sf
```

설치를 확인합니다:

```bash
sf --version
```

`@salesforce/cli/2.x.x`와 같은 출력이 표시되어야 합니다.

## 인증 설정

### 1단계: Salesforce 도메인 확인

브라우저에서 Salesforce URL을 확인합니다. `https://acme.lightning.force.com`으로
Salesforce에 접속하는 경우, 로그인 도메인은
`https://acme.my.salesforce.com`입니다.

<Aside type="tip">
  회사에서 SSO를 사용하는 경우, 로그인이 일반 Salesforce 로그인 페이지 대신
  SSO 공급자를 통해 리디렉션되도록 `--instance-url`에 사용자 지정 도메인을
  반드시 지정해야 합니다.
</Aside>

### 2단계: 브라우저를 통한 인증

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

`YOUR-DOMAIN`을 1단계에서 확인한 회사 도메인으로 교체합니다. 브라우저가
열리면 SSO 또는 로그인 절차를 완료하고 앱을 승인합니다. 다음과 같은 메시지가
표시되어야 합니다:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### 3단계: 연결 확인

```bash
sf org display --target-org my-org
```

출력에 `Connected Status: Connected`와 함께 사용자 이름 및 인스턴스 URL이
표시되는지 확인합니다.

### 4단계: 컨테이너용 이식 가능한 인증 (선택 사항)

헤드리스 환경(CI/CD, 컨테이너, 원격 세션)에서 인증이 필요한 경우,
워크스테이션에서 SFDX 인증 URL을 내보냅니다:

```bash
sf org display --verbose --target-org my-org
```

**Sfdx Auth Url** 값(`force://`로 시작)을 복사합니다. 컨테이너에서 다음을
실행합니다:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX 인증 URL은 자격 증명이므로 비밀번호처럼 취급해야 합니다. 파일에
  커밋하거나 채팅에 붙여넣지 마십시오. 환경 변수와 stdin 파이프만
  사용하십시오.
</Aside>

### 인증 방법 참조

| 방법 | 적합한 환경 | 필요 사항 |
|--------|----------|----------|
| 웹 로그인 | 브라우저가 있는 워크스테이션 | 브라우저 + SSO |
| SFDX URL | 컨테이너, CI/CD, 이식 가능한 인증 | 인증된 세션의 인증 URL |
| JWT Bearer | 자동화된 파이프라인 | Connected App + 개인 키 + 소비자 키 |
| 액세스 토큰 | 환경 변수 인증 | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**참고:** 디바이스 플로우(`sf org login device`)는 2025년 8월부터 차단되어
지원되지 않습니다.

### 환경 변수

| 변수 | 목적 |
|----------|---------|
| `SF_ACCESS_TOKEN` | 액세스 토큰 인증용 Bearer 토큰 |
| `SFDX_AUTH_URL` | SFDX URL 인증용 Force 인증 URL |
| `SF_ORG_INSTANCE_URL` | Org 인스턴스 URL |
| `SF_JWT_KEY_FILE` | JWT 개인 키 경로 |
| `SF_CLIENT_ID` | Connected App 소비자 키 |
| `SF_USERNAME` | JWT용 Salesforce 사용자 이름 |

## 명령어

### /salesforce:sf-status

Salesforce org 연결 상태, CLI 버전 및 인증된 org를 확인합니다.

```
/salesforce:sf-status
```

**예상 결과:** sf CLI 버전, 인증된 org 별칭, 사용자 이름, 인스턴스 URL,
연결 상태 및 API 버전을 보여주는 표가 표시됩니다.

### /salesforce:sf-login

Salesforce org에 인증합니다. 선택적으로 별칭을 제공할 수 있습니다.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**예상 결과:** 해당 별칭의 org가 이미 인증되어 있으면 연결 세부 정보를
보고합니다. 그렇지 않으면 사용 가능한 자격 증명(환경 변수)을 확인하고
조건을 모두 충족하는 첫 번째 방법으로 인증합니다.

## 스킬

### salesforce-index

최상위 인텐트 라우터입니다. 자연어로 Salesforce, org 관리 또는 데이터 쿼리를
언급할 때 자동으로 활성화됩니다.

| 발화 내용 | 라우팅 대상 |
|---------|-----------|
| "login to salesforce", "authenticate org" | `salesforce-auth` 스킬 |
| "check org status", "list orgs" | `cli-operator` 에이전트 |
| "show me opportunities", "look up case" | `cli-operator` 에이전트 (SOQL 쿼리) |
| "generate Apex class", "create Flow" | afv-library 개발 스킬 |

### salesforce-auth

컨테이너에 최적화된 인증 스킬입니다. 네 가지 인증 방법을 지원하며
사용 가능한 자격 증명을 자동으로 감지합니다. 우선순위: 액세스 토큰,
JWT, SFDX URL, 웹 로그인 순으로 처리합니다.

## 에이전트

### cli-operator

안전 가드레일이 적용된 `sf` 명령을 실행하는 자율 Salesforce CLI 에이전트입니다.
모든 Salesforce 작업은 메인 세션 컨텍스트를 간결하게 유지하기 위해
이 에이전트에 위임됩니다.

**안전 규칙:**

- 기본적으로 읽기 전용 — 쓰기 작업은 명시적 승인이 필요합니다
- 미리보기 및 확인 단계 없이 배포하지 않습니다
- 출력에 액세스 토큰, 인증 URL 또는 자격 증명을 표시하지 않습니다
- 모든 사용자 제공 값을 검사합니다 (셸 메타문자 거부)
- 구조화된 파싱 가능한 결과를 위해 `--json` 출력을 사용합니다

**응답 형식:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## 사용 가이드: 계정 관리

<Aside type="note">
  아래 프롬프트는 플러그인에 하드코딩된 기능이 아닙니다. 이 프롬프트들은
  cli-operator 에이전트가 임의의 SOQL 쿼리를 포함한 모든 `sf` CLI 명령을
  실행할 수 있기 때문에 작동합니다. 결과는 org의 데이터 모델과 사용자
  권한에 따라 다릅니다. 여기의 프롬프트는 표준 Salesforce 객체(Account,
  Contact, Case, Opportunity, AccountTeamMember, OpportunityTeamMember)에서
  테스트하여 검증되었습니다.
</Aside>

이 프롬프트는 모든 Salesforce org에서 작동합니다. 자리 표시자 값을
본인의 정보로 교체하십시오. 각 프롬프트는 역할, 계정, 담당 지역에
따라 다른 결과를 생성합니다.

<Aside type="tip">
  최상의 결과를 얻으려면 사용자 레코드를 식별해야 하는 프롬프트에
  이메일 주소를 포함하십시오. Salesforce org에는 비슷한 이름의 사용자가
  여러 명 있을 수 있습니다.
</Aside>

### 시작하기

인증 후 모든 것이 정상적으로 작동하는지 확인합니다:

```
/salesforce:sf-status
```

그런 다음 간단한 자연어 쿼리를 시도합니다:

```
list my authenticated salesforce orgs
```

**예상 결과:** 별칭, 사용자 이름, 인스턴스 URL 및 연결 상태가 포함된
모든 연결된 org를 보여주는 표가 표시됩니다.

### 계정 찾기

담당하고 있는 계정을 확인합니다:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**예상 결과:** 계정 소유자별로 그룹화된 계정 목록과 각 계정에서의
팀 멤버 역할이 표시됩니다. 대부분의 역할이 비어 있는 경우, 이는 일반적인
데이터 품질 문제입니다 — 계정 팀 멤버십은 존재하지만 역할이
채워지지 않은 것입니다.

### 동료와 커버리지 교차 확인

팀원과의 계정 커버리지를 비교하여 공백을 찾습니다:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**예상 결과:** 중복 수와 공백 수를 보여주는 요약 표와 두 개의 목록이
표시됩니다. 최근에 역할을 변경한 경우 중복이 0개일 수 있으며, 이는
추가되어야 할 계정을 나타냅니다.

<Aside type="note">
  "Also check if Previous Person Name appears on any of those accounts."를
  추가하여 전임자를 확인할 수도 있습니다. 이를 통해 담당 지역에서
  누군가를 교체할 때 인수인계 공백을 파악할 수 있습니다.
</Aside>

### 담당 지역 파이프라인 개요

계정 전체의 모든 오픈 영업 기회를 확인합니다:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**예상 결과:** 총 영업 기회 수, 원시 파이프라인 및 가중 파이프라인이 포함된
요약이 표시됩니다. 그런 다음 스테이지 분포 표와 금액별 상위 영업 기회
목록이 표시됩니다. 기한이 지난 마감일이나 금액 미입력과 같은 데이터 품질
문제가 표시됩니다.

### 영업 기회 심층 분석

특정 거래에 대한 전체 정보를 가져옵니다:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**예상 결과:** 영업 기회 개요(스테이지, 금액, 마감일, 확률, 예측 카테고리,
소유자), 역할이 포함된 영업 기회 팀 멤버, 태그된 연락처, 최근 작업 및
이벤트, 그리고 시간에 따른 거래 금액과 마감일 변경 이력이 표시됩니다.

### 담당 지역 지원 케이스

계정의 오픈 지원 케이스를 확인합니다:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**예상 결과:** 우선순위 및 상태별 분류와 함께 오픈 케이스 수가 표시되고,
계정별로 그룹화된 케이스 세부 정보가 표시됩니다. 활동 없이 수개월 또는
수년이 지난 케이스는 데이터 정리 대상으로 표시됩니다.

### 분기별 파이프라인 예측

파이프라인의 예측 준비 뷰를 가져옵니다:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**예상 결과:** 분기별 영업 기회 수, 총 파이프라인, 가중 파이프라인을
보여주는 행이 포함된 표가 표시됩니다. 분기별 스테이지 믹스 분석과
상위 가중 거래가 강조 표시됩니다. 기한이 지난 영업 기회는 별도로
그룹화됩니다.

### 전체 계정 개요

특정 고객에 대해 심층적으로 분석합니다:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**예상 결과:** 회사 프로필(업종, 매출, 직원 수, 위치), 직함 및 이메일이
포함된 주요 연락처, 스테이지 및 금액이 포함된 오픈 영업 기회, 그리고
상태가 포함된 최근 지원 케이스가 표시됩니다.

### 케이스 조회

케이스 번호로 특정 지원 케이스를 조회합니다:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**예상 결과:** 케이스 주제, 상태, 우선순위, 설명, 고객 계정 프로필,
연락처 정보가 포함된 케이스 소유자가 표시됩니다.

## 사용 가이드: 데이터 쿼리

고급 사용자는 자연어로 특정 SOQL 스타일의 쿼리를 요청할 수 있습니다.
플러그인이 요청을 적절한 SOQL로 변환하여 실행합니다.

### 특정 필드

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### 날짜 필터링

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### 집계

```
count all open salesforce cases grouped by priority and status across my accounts
```

### 사용자 지정 객체

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  필드가 존재하지 않아 쿼리가 실패하면 플러그인이 해당 필드 없이
  자동으로 재시도합니다. org의 사용자 지정 필드는 다른 이름을 사용할 수
  있으므로 필드를 찾을 수 없는 경우 Salesforce 설정을 확인하십시오.
</Aside>

```bash
npx skills add forcedotcom/afv-library
```
