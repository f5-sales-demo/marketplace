**salesforce** प्लगइन Salesforce CLI को Claude Code के साथ एकीकृत करता है,
जो org प्रमाणीकरण और एक सामान्य-उद्देश्य CLI एजेंट प्रदान करता है जो SOQL क्वेरी सहित कोई भी `sf` कमांड चला सकता है। यह JWT, access-token और SFDX URL
प्रमाणीकरण फ्लो के साथ हेडलेस कंटेनर वातावरण का समर्थन करता है। नीचे दी गई उपयोग मार्गदर्शिका दर्शाती है कि CLI एजेंट के माध्यम से accounts, opportunities, cases और contacts को प्राकृतिक भाषा में क्वेरी करने के लिए इसका उपयोग कैसे करें।

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## इंस्टॉलेशन

```
/plugin install salesforce@f5-sales-demo-marketplace
```

## पूर्वापेक्षाएँ

- **Salesforce CLI** (`@salesforce/cli`)
- **Salesforce org** API एक्सेस के साथ
- ऑब्जेक्ट (Cases, Opportunities, Accounts, Contacts) को क्वेरी करने की अनुमतियों वाला एक उपयोगकर्ता खाता

अपने वर्कस्टेशन पर Salesforce CLI इंस्टॉल करें:

```bash
brew install sf
```

इंस्टॉलेशन सत्यापित करें:

```bash
sf --version
```

आपको `@salesforce/cli/2.x.x` जैसा आउटपुट दिखना चाहिए।

## प्रमाणीकरण सेटअप

### चरण 1: अपना Salesforce डोमेन खोजें

ब्राउज़र में अपना Salesforce URL देखें। यदि आप Salesforce को
`https://acme.lightning.force.com` पर एक्सेस करते हैं, तो आपका लॉगिन डोमेन
`https://acme.my.salesforce.com` है।

<Aside type="tip">
  यदि आपकी कंपनी SSO का उपयोग करती है, तो आपको
  `--instance-url` के साथ अपना कस्टम डोमेन निर्दिष्ट करना होगा ताकि लॉगिन
  सामान्य Salesforce लॉगिन पेज के बजाय आपके SSO प्रोवाइडर के माध्यम से
  रीडायरेक्ट हो।
</Aside>

### चरण 2: ब्राउज़र के माध्यम से प्रमाणीकरण करें

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

`YOUR-DOMAIN` को चरण 1 से अपने कंपनी डोमेन से बदलें। आपका ब्राउज़र
खुलता है — SSO या लॉगिन फ्लो पूरा करें और ऐप को अधिकृत करें। आपको दिखना चाहिए:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### चरण 3: कनेक्शन सत्यापित करें

```bash
sf org display --target-org my-org
```

पुष्टि करें कि आउटपुट आपके उपयोगकर्ता नाम और इंस्टेंस URL के साथ `Connected Status: Connected` दिखाता है।

### चरण 4: कंटेनर के लिए पोर्टेबल auth (वैकल्पिक)

यदि आपको हेडलेस वातावरण (CI/CD, कंटेनर, रिमोट सेशन) में प्रमाणीकरण करने की
आवश्यकता है, तो अपने वर्कस्टेशन से SFDX auth URL एक्सपोर्ट करें:

```bash
sf org display --verbose --target-org my-org
```

**Sfdx Auth Url** मान कॉपी करें (जो `force://` से शुरू होता है)। कंटेनर में, चलाएं:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX auth URL एक क्रेडेंशियल है — इसे पासवर्ड की तरह मानें। इसे
  फ़ाइलों में कमिट न करें या चैट में पेस्ट न करें। केवल
  एनवायरनमेंट वेरिएबल और stdin पाइप का उपयोग करें।
</Aside>

### प्रमाणीकरण विधियों का संदर्भ

| विधि | सबसे उपयुक्त | आवश्यकता |
|--------|----------|----------|
| Web Login | ब्राउज़र वाले वर्कस्टेशन | Browser + SSO |
| SFDX URL | कंटेनर, CI/CD, पोर्टेबल auth | प्रमाणीकृत सेशन से Auth URL |
| JWT Bearer | स्वचालित पाइपलाइन | Connected App + प्राइवेट की + consumer key |
| Access Token | एनवायरनमेंट वेरिएबल auth | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**नोट:** डिवाइस फ्लो (`sf org login device`) अगस्त 2025 से ब्लॉक है और
समर्थित नहीं है।

### एनवायरनमेंट वेरिएबल

| वेरिएबल | उद्देश्य |
|----------|---------|
| `SF_ACCESS_TOKEN` | access-token auth के लिए Bearer token |
| `SFDX_AUTH_URL` | SFDX URL auth के लिए Force auth URL |
| `SF_ORG_INSTANCE_URL` | Org इंस्टेंस URL |
| `SF_JWT_KEY_FILE` | JWT प्राइवेट की का पथ |
| `SF_CLIENT_ID` | Connected App consumer key |
| `SF_USERNAME` | JWT के लिए Salesforce उपयोगकर्ता नाम |

## कमांड

### /salesforce:sf-status

अपनी Salesforce org कनेक्शन स्थिति, CLI संस्करण और
प्रमाणीकृत orgs की जाँच करें।

```
/salesforce:sf-status
```

**क्या उम्मीद करें:** एक तालिका जो आपके sf CLI संस्करण, प्रमाणीकृत
org alias, उपयोगकर्ता नाम, इंस्टेंस URL, कनेक्टेड स्थिति और API संस्करण दिखाती है।

### /salesforce:sf-login

Salesforce org में प्रमाणीकरण करें। वैकल्पिक रूप से एक alias प्रदान करें।

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**क्या उम्मीद करें:** यदि उस alias वाला org पहले से प्रमाणीकृत है,
तो यह कनेक्शन विवरण रिपोर्ट करता है। यदि नहीं, तो यह उपलब्ध
क्रेडेंशियल (एनवायरनमेंट वेरिएबल) की जाँच करता है और पहली
पूरी तरह से संतुष्ट विधि का उपयोग करके प्रमाणीकरण करता है।

## स्किल

### salesforce-index

शीर्ष-स्तरीय intent राउटर। जब आप प्राकृतिक भाषा में Salesforce, org प्रबंधन,
या डेटा क्वेरी का उल्लेख करते हैं तो स्वचालित रूप से सक्रिय होता है।

| आप कहते हैं | रूट होता है |
|---------|-----------|
| "login to salesforce", "authenticate org" | `salesforce-auth` स्किल |
| "check org status", "list orgs" | `cli-operator` एजेंट |
| "show me opportunities", "look up case" | `cli-operator` एजेंट (SOQL क्वेरी) |
| "generate Apex class", "create Flow" | afv-library डेवलपमेंट स्किल |

### salesforce-auth

कंटेनर-अनुकूलित प्रमाणीकरण स्किल। चार auth विधियों का समर्थन करती है और
स्वचालित रूप से पता लगाती है कि कौन से क्रेडेंशियल उपलब्ध हैं। प्राथमिकता:
access-token, फिर JWT, फिर SFDX URL, फिर web login।

## एजेंट

### cli-operator

स्वायत्त Salesforce CLI एजेंट जो सुरक्षा गार्डरेल के साथ `sf` कमांड निष्पादित करता है। मुख्य सेशन संदर्भ को सुव्यवस्थित रखने के लिए सभी Salesforce ऑपरेशन इस एजेंट को सौंपे जाते हैं।

**सुरक्षा नियम:**

- डिफ़ॉल्ट रूप से केवल पढ़ने योग्य — राइट ऑपरेशन के लिए स्पष्ट अनुमोदन आवश्यक है
- बिना प्रिव्यू और पुष्टि चरण के कभी डिप्लॉय नहीं करता
- आउटपुट में access tokens, auth URLs या क्रेडेंशियल कभी नहीं दिखाता
- सभी उपयोगकर्ता-प्रदत्त मानों को सैनिटाइज़ करता है (shell मेटाकैरेक्टर अस्वीकार करता है)
- संरचित, पार्स करने योग्य परिणामों के लिए `--json` आउटपुट का उपयोग करता है

**प्रतिक्रिया प्रारूप:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## उपयोग मार्गदर्शिका: खाता प्रबंधन

<Aside type="note">
  नीचे दिए गए प्रॉम्प्ट प्लगइन की हार्ड-कोडेड सुविधाएँ नहीं हैं। वे काम करते हैं
  क्योंकि cli-operator एजेंट कोई भी `sf` CLI कमांड चला सकता है, जिसमें
  मनमानी SOQL क्वेरी शामिल हैं। परिणाम आपके org के डेटा मॉडल और
  आपकी उपयोगकर्ता अनुमतियों पर निर्भर करते हैं। यहाँ के प्रॉम्प्ट मानक Salesforce
  ऑब्जेक्ट (Account, Contact, Case, Opportunity, AccountTeamMember,
  OpportunityTeamMember) के साथ परीक्षित और सत्यापित किए गए हैं।
</Aside>

ये प्रॉम्प्ट किसी भी Salesforce org के साथ काम करते हैं। प्लेसहोल्डर
मानों को अपनी जानकारी से बदलें। प्रत्येक प्रॉम्प्ट आपकी भूमिका, accounts और
टेरिटरी के लिए विशिष्ट परिणाम देता है।

<Aside type="tip">
  सर्वोत्तम परिणामों के लिए, उन प्रॉम्प्ट में अपना ईमेल पता शामिल करें जिन्हें
  आपके उपयोगकर्ता रिकॉर्ड की पहचान करनी हो। Salesforce orgs में समान नामों वाले
  कई उपयोगकर्ता हो सकते हैं।
</Aside>

### शुरुआत करना

प्रमाणीकरण के बाद, सब कुछ काम करता है यह सत्यापित करें:

```
/salesforce:sf-status
```

फिर एक सरल प्राकृतिक भाषा क्वेरी आज़माएं:

```
list my authenticated salesforce orgs
```

**क्या उम्मीद करें:** एक तालिका जो सभी कनेक्टेड orgs को alias,
उपयोगकर्ता नाम, इंस्टेंस URL और कनेक्शन स्थिति के साथ दिखाती है।

### अपने accounts खोजें

पता करें कि आप किन accounts पर असाइन हैं:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**क्या उम्मीद करें:** account owner के अनुसार समूहीकृत accounts की एक सूची,
जो प्रत्येक पर आपकी टीम सदस्य भूमिका दिखाती है। यदि अधिकांश भूमिकाएं खाली दिखती हैं,
तो यह एक सामान्य डेटा हाइजीन अंतर है — account team सदस्यताएं मौजूद हैं
लेकिन भूमिकाएं भरी नहीं गई थीं।

### किसी सहयोगी के साथ कवरेज क्रॉस-रेफरेंस करें

गैप खोजने के लिए किसी टीममेट के साथ अपनी account कवरेज की तुलना करें:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**क्या उम्मीद करें:** ओवरलैप काउंट और गैप काउंट दिखाने वाली एक सारांश तालिका,
उसके बाद दो सूचियाँ। यदि आपने हाल ही में भूमिकाएं बदली हैं, तो आपको शून्य
ओवरलैप मिल सकता है — यह बताता है कि आपको किन accounts में जोड़ा जाना चाहिए।

<Aside type="note">
  आप यह जोड़कर पूर्ववर्ती व्यक्ति की भी जाँच कर सकते हैं: "Also check if
  Previous Person Name appears on any of those accounts." यह
  टेरिटरी पर किसी को रिप्लेस करते समय हैंडऑफ गैप की पहचान करने में मदद करता है।
</Aside>

### टेरिटरी पाइपलाइन अवलोकन

अपने accounts पर सभी ओपन opportunities देखें:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**क्या उम्मीद करें:** कुल opportunity काउंट, raw पाइपलाइन और
वेटेड पाइपलाइन के साथ एक सारांश। फिर एक stage वितरण तालिका और
amount के अनुसार शीर्ष opportunities की सूची। आउटपुट डेटा हाइजीन समस्याओं
जैसे past-due close dates या null amounts को फ्लैग करता है।

### Opportunity की गहन जाँच

किसी विशिष्ट डील की पूरी तस्वीर प्राप्त करें:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**क्या उम्मीद करें:** opportunity अवलोकन (stage, amount, close date,
probability, forecast category, owner), भूमिकाओं के साथ opportunity टीम सदस्य,
टैग किए गए contacts, हालिया tasks और events, और field बदलाव
इतिहास जो दिखाता है कि समय के साथ डील amount और close date कैसे बदले हैं।

### टेरिटरी में सपोर्ट cases

अपने accounts पर ओपन सपोर्ट cases की जाँच करें:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**क्या उम्मीद करें:** priority और status breakdown के साथ ओपन cases की गिनती,
फिर account के अनुसार समूहीकृत case विवरण। Stale cases (बिना किसी गतिविधि के
महीनों या वर्षों पुराने) को हाइजीन उम्मीदवारों के रूप में फ्लैग किया जाता है।

### तिमाही पाइपलाइन पूर्वानुमान

अपनी पाइपलाइन का forecast-ready व्यू प्राप्त करें:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**क्या उम्मीद करें:** प्रति तिमाही एक पंक्ति वाली तालिका जो opportunity
काउंट, कुल पाइपलाइन और वेटेड पाइपलाइन दिखाती है। प्रति तिमाही stage mix
breakdown शामिल है और शीर्ष वेटेड deals को हाइलाइट करती है। Past-due
opportunities को अलग से समूहीकृत किया जाता है।

### पूर्ण account अवलोकन

किसी विशिष्ट ग्राहक की गहन जाँच करें:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**क्या उम्मीद करें:** कंपनी प्रोफ़ाइल (industry, revenue, employees,
location), titles और email के साथ प्रमुख contacts, stages और amounts के साथ
ओपन opportunities, और status के साथ हालिया सपोर्ट cases।

### Case लुकअप

नंबर द्वारा एक विशिष्ट सपोर्ट case खोजें:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**क्या उम्मीद करें:** Case subject, status, priority, description,
ग्राहक account प्रोफ़ाइल, और संपर्क जानकारी के साथ case owner।

## उपयोग मार्गदर्शिका: डेटा क्वेरी

पावर उपयोगकर्ताओं के लिए, आप प्राकृतिक भाषा में विशिष्ट SOQL-शैली की
क्वेरी माँग सकते हैं। प्लगइन आपके अनुरोध को उचित SOQL में अनुवादित करता है
और चलाता है।

### विशिष्ट फ़ील्ड

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### तारीख फ़िल्टरिंग

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

  उस फ़ील्ड के बिना स्वचालित रूप से पुनः प्रयास करता है। आपके org में कस्टम फ़ील्ड
  के अलग-अलग नाम हो सकते हैं — यदि कोई फ़ील्ड नहीं मिलती तो अपनी
  Salesforce Setup जाँचें।
</Aside>
