**f5xc-devcontainer** प्लगइन f5xc-salesdemos डेवलपमेंट एनवायरनमेंट के लिए
कंटेनर अवेयरनेस प्रदान करता है। यह एक व्यापक टूल कैटलॉग बनाए रखता है,
लाइव कंटेनर इंट्रोस्पेक्शन सक्षम करता है, और GitHub issues के माध्यम से
पर्सिस्टेंस ट्रैकिंग के साथ टूल इंस्टॉलेशन को संभालता है।

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## इंस्टॉलेशन

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## स्किल्स

### tool-catalog

डेवकंटेनर में इंस्टॉल किए गए प्रत्येक CLI टूल का नॉलेज बेस,
श्रेणी (क्लाउड, नेटवर्किंग, सिक्योरिटी, डेवलपमेंट,
आदि) के अनुसार व्यवस्थित। जब आप पूछते हैं कि कौन सा टूल उपयोग करना है,
किसी टूल का उपयोग कैसे करना है, कुछ इंस्टॉल है या नहीं, या किसी
विशिष्ट ऑपरेशन में सहायता चाहिए, तब यह सक्रिय होता है।

### self-awareness

कंटेनर पहचान और स्व-निदान स्किल। GitHub API और लोकल
बिल्ड मेटाडेटा के माध्यम से चल रहे कंटेनर का लाइव
इंट्रोस्पेक्शन प्रदान करता है। पहचान, संस्करण, बिल्ड जानकारी,
हेल्थ चेक, और कंट्रीब्यूटर इतिहास संबंधी प्रश्नों के लिए सक्रिय होता है।

<Aside type="tip">
  Self-awareness को ट्रिगर करने के लिए "who are you?" या "what version am I running?" पूछें।
</Aside>

## एजेंट्स

### tool-advisor

टूल कैटलॉग पढ़ता है और किसी भी CLI टूल प्रश्न के लिए उद्देश्य,
क्विक-स्टार्ट कमांड, और प्रमाणीकरण आवश्यकताओं के साथ
अनुशंसाएँ लौटाता है।

### tool-auditor

ड्रिफ्ट का पता लगाने के लिए टूल कैटलॉग रेफरेंस फाइलों के विरुद्ध
डेवकंटेनर Dockerfile का ऑडिट करता है &mdash; ऐसे टूल जो इमेज में
जोड़े गए लेकिन कैटलॉग में नहीं हैं, या कैटलॉग प्रविष्टियाँ उन टूल्स
के लिए जो हटा दिए गए हैं।

### container-introspector

पहचान, वंशावली, स्व-निदान, कंट्रीब्यूटर इतिहास, और बिल्ड
वंशक्रम के लिए GitHub API और लोकल मेटाडेटा के माध्यम से लाइव
कंटेनर इंट्रोस्पेक्शन करता है। केवल रीड-ओनली ऑपरेशन।

### container-maintainer

सही पैकेज मैनेजर पैटर्न का उपयोग करके चल रहे कंटेनर में CLI टूल्स
को इंस्टॉल, हटाता, खोजता और अपडेट करता है। पर्सिस्टेंस ट्रैकिंग के लिए
डेवकंटेनर रिपॉजिटरी के विरुद्ध GitHub issues फाइल करता है।
