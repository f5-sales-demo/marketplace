**f5xc-github-ops** प्लगइन f5-sales-demo रिपॉजिटरी के लिए विशेष GitHub ऑपरेशन
स्वचालन प्रदान करता है। यह इश्यू निर्माण से लेकर पोस्ट-मर्ज
निगरानी तक संपूर्ण Git और GitHub लाइफसाइकिल को संभालता है,
इश्यू-संचालित विकास और प्री-कमिट लिंट गेट्स को लागू करता है।

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## इंस्टॉलेशन

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## स्किल्स

### workflow-lifecycle

रिपॉजिटरी गवर्नेंस वर्कफ़्लो जो इश्यू निर्माण, ब्रांच
नामकरण परंपराएं, PR निर्माण, त्रुटि फ़ीडबैक के साथ इश्यू में CI पोलिंग,
पोस्ट-मर्ज निगरानी, सत्यापन और कार्य
पूर्णता मानदंड को संभालता है। यह तब सक्रिय होता है जब आप "commit", "push",
"create a PR", "merge" कहते हैं, या कोई भी Git ऑपरेशन करते हैं।

### upstream-contribution

फोर्क-आधारित विकास के लिए अपस्ट्रीम योगदान वर्कफ़्लो। फोर्क
डिटेक्शन, अपस्ट्रीम रिसर्च (इश्यू, PR, CONTRIBUTING.md,
CI पैटर्न), क्रॉस-रेपो इश्यू और PR निर्माण, लोकल ट्रैकिंग
इश्यू प्रबंधन, अपस्ट्रीम CI निगरानी और मार्गदर्शित समाधान को संभालता है।

## एजेंट्स

### github-ops

सभी मैकेनिकल Git और GitHub ऑपरेशन के लिए विशेष एजेंट।
प्री-कमिट लिंट गेट्स, इश्यू निर्माण, ब्रांच निर्माण,
स्टेजिंग, कमिटिंग, पुशिंग, PR निर्माण, त्रुटि
फ़ीडबैक के साथ CI पोलिंग, इन्फ्रास्ट्रक्चर विफलता पुनः-प्रयास, पोस्ट-मर्ज निगरानी,
ब्रांच क्लीनअप और GitHub API के माध्यम से
रिपॉजिटरी सेटिंग्स प्रबंधन को संभालता है।

संरचित स्थिति रिपोर्ट लौटाता है: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED`, या `FAILED`।

<Aside type="caution">
  यह एजेंट Git ऑपरेशन के लिए **विशेष** हैंडलर है।
  उन रिपॉजिटरी में कभी भी सीधे `git commit`, `git push`, या `gh pr create` न चलाएं
  जो इस प्लगइन का उपयोग करती हैं।
</Aside>
