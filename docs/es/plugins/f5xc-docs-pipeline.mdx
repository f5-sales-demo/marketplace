El plugin **f5xc-docs-pipeline** proporciona conocimiento de arquitectura
del pipeline de documentación para repositorios de f5-sales-demo. Orienta
la autoría de contenido, clarifica la propiedad de la configuración en
compilaciones multi-repositorio y ofrece vista previa local basada en Docker.

<Badge text="v1.0.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-docs-pipeline@f5-sales-demo-marketplace
```

## Habilidades

### content-author

Guía de autoría de contenido que cubre la estructura de archivos, reglas de
sintaxis MDX, requisitos de frontmatter, referencias de imágenes y vista
previa local con Docker. Se activa al crear o editar contenido en `docs/`,
al trabajar con archivos MDX o al configurar la vista previa local.

### pipeline-navigator

Arquitectura del pipeline de documentación que explica dónde realizar
cambios en los cuatro repositorios fuente: docs-theme, docs-builder,
docs-control y docs-icons. Clarifica la propiedad de la configuración
(`astro.config.mjs`, `content.config.ts`), la cadena de despacho de
versiones y el seguimiento de archivos gestionados.

<Aside type="tip">
  Utilice pipeline-navigator cuando necesite entender qué repositorio es
  propietario de un archivo de configuración o componente específico.
</Aside>

## Comandos

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

Inicia un servidor de desarrollo Docker local para previsualizar la
documentación. Ejecuta la imagen de compilación Docker compartida con
su directorio `docs/` local montado para vista previa en tiempo real.
