El plugin **f5xc-sales-engineer** proporciona un marco de persona de Ingeniero de Ventas reutilizable para repositorios de demos de F5 XC. Gestiona el ciclo de vida completo de la reunión de demo — desde la verificación previa a la reunión hasta la ejecución en vivo, preguntas y respuestas, y desmontaje.

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## Habilidades

### demo-executor

Ejecución de demos basada en API con un ciclo de vida de reunión de cuatro etapas:

1. **Preparar** — verificación del entorno previa a la reunión mediante el agente `demo-housekeeping`
2. **Ejecutar** — ejecución de fases determinista con narración orientada al cliente
3. **Preguntas y respuestas** — modo improvisado con entorno en vivo
4. **Desmontaje** — limpieza posterior a la reunión mediante el agente `demo-housekeeping`

Lee contenido específico del producto desde `DEMO_PRODUCT_EXPERTISE.md` y los comandos de demo desde `docs/demo/`.

### presenter

Presentación de recorrido tal como fue construido utilizando un entorno de demo preconfigurado. Sigue un patrón mostrar → narrar → conectar → pausar en cada paso. Lee el orden del recorrido desde `DEMO_WALKTHROUGH_CONFIG.md`.

### subject-matter-expert

Preguntas y respuestas respaldadas por referencias con requisitos estrictos de citas. Nunca adivina — cada respuesta incluye una referencia o prueba. Delega la investigación al agente `demo-researcher` cuando es necesario.

### sales-engineer

Índice de roles que dirige la intención del usuario a la habilidad de persona correcta. Se activa cuando la solicitud está relacionada con demos pero no coincide con una frase de activación específica.

## Agentes

### demo-housekeeping

Agente autónomo para la verificación previa a la reunión y el desmontaje posterior. Ejecuta la Matriz de Verificación de Disponibilidad (T0–T5) definida en `DEMO_READINESS_MATRIX.md` y los comandos de desmontaje desde `docs/demo/phase-4-teardown.mdx`.

### demo-researcher

Bibliotecario de investigación de solo lectura que encuentra respuestas verificadas con citas. Busca primero en `docs/` local, luego en fuentes externas indexadas desde `DEMO_SOURCE_INDEX.md`, con WebSearch como alternativa.

## Archivos de Convención

<Aside type="tip" title="Contenido específico del producto">
  Este plugin es independiente del producto. Cada repositorio proporciona
  contenido específico del producto a través de archivos de convención estandarizados.
</Aside>

Cree estos archivos en la raíz de su repositorio para configurar el plugin:

| Archivo | Propósito | Requerido para |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | Capacidades del producto, señales de detección, cobertura de amenazas, alineación de cumplimiento, referencia de API | Todas las habilidades |
| `DEMO_WALKTHROUGH_CONFIG.md` | URL de la aplicación de demo, orden del recorrido, tiempos de detección | presenter |
| `DEMO_SOURCE_INDEX.md` | Catálogo de fuentes de investigación con documentación local y URLs externas | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | Variables requeridas/opcionales, verificaciones de disponibilidad (T0–T5), endpoints de API | demo-housekeeping |
| `docs/demo/` | Archivos de fase con comandos cURL y puertas de evidencia | demo-executor |

## Frases de Activación

El mapa de activación `.claude/CLAUDE.md` (gestionado por `docs-control`) activa las habilidades según la intención del usuario:

| Frase | Habilidad | Etapa |
| ------ | ----- | ----- |
| "run the demo", "execute the demo" | demo-executor | Ejecutar |
| "prepare the demo", "pre-flight" | demo-executor | Preparar |
| "Q&A", "question and answer" | demo-executor | Preguntas y respuestas |
| "tear down", "clean up" | demo-executor | Desmontaje |
| "walk through the demo", "present" | presenter | — |
| "answer questions", "explain" | subject-matter-expert | — |

## Estilo de Narración

Todas las personas comparten un estilo de narración consistente:

- **Presente, primera persona del plural** — "Lo que estamos viendo aquí…"
- **Una preocupación por párrafo** — cada narración responde: ¿Qué es esto? ¿Por qué importa? ¿Qué debo hacer al respecto?
- **Nombrar la señal** — indicar qué capacidad está en funcionamiento
- **Referencia de cumplimiento cuando sea relevante** — mencionar la alineación regulatoria de forma natural
- **Marcador de pausa de ritmo** — `> ⏸ *[Pause for audience — ready to continue?]*`
