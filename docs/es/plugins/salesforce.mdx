El plugin **salesforce** integra Salesforce CLI con Claude Code,
proporcionando autenticación de organización y un agente CLI de propósito general que puede
ejecutar cualquier comando `sf`, incluidas consultas SOQL. Admite entornos de
contenedores sin interfaz gráfica con flujos de autenticación JWT, token de acceso y
URL de SFDX. La guía de uso a continuación demuestra cómo utilizar el lenguaje natural
para consultar cuentas, oportunidades, casos y contactos
a través del agente CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## Instalación

```
/plugin install salesforce@f5-sales-demo-marketplace
```

## Requisitos previos

- **Salesforce CLI** (`@salesforce/cli`)
- **Organización de Salesforce** con acceso a la API
- Una cuenta de usuario con permisos para consultar objetos (Cases, Opportunities, Accounts, Contacts)

Instale Salesforce CLI en su estación de trabajo:

```bash
brew install sf
```

Verifique la instalación:

```bash
sf --version
```

Debería ver una salida como `@salesforce/cli/2.x.x`.

## Configuración de autenticación

### Paso 1: Encuentre su dominio de Salesforce

Consulte la URL de Salesforce en el navegador. Si accede a Salesforce en
`https://acme.lightning.force.com`, su dominio de inicio de sesión es
`https://acme.my.salesforce.com`.

<Aside type="tip">
  Si su empresa utiliza SSO, debe especificar su dominio personalizado con
  `--instance-url` para que el inicio de sesión se redirija a través de su proveedor de SSO
  en lugar de la página de inicio de sesión genérica de Salesforce.
</Aside>

### Paso 2: Autentíquese mediante el navegador

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

Reemplace `YOUR-DOMAIN` con el dominio de su empresa del Paso 1. Su navegador
se abrirá — complete el flujo de SSO o inicio de sesión y autorice la aplicación. Debería
ver:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### Paso 3: Verifique la conexión

```bash
sf org display --target-org my-org
```

Confirme que la salida muestre `Connected Status: Connected` con su nombre de usuario
y URL de instancia.

### Paso 4: Autenticación portátil para contenedores (opcional)

Si necesita autenticarse en un entorno sin interfaz gráfica (CI/CD, contenedor,
sesión remota), exporte la URL de autenticación SFDX desde su estación de trabajo:

```bash
sf org display --verbose --target-org my-org
```

Copie el valor de **Sfdx Auth Url** (comienza con `force://`). En el
contenedor, ejecute:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  La URL de autenticación SFDX es una credencial — trátela como una contraseña. No la
  incluya en archivos ni la pegue en el chat. Utilice únicamente variables de entorno
  y tuberías de stdin.
</Aside>

### Referencia de métodos de autenticación

| Método | Ideal para | Requiere |
|--------|----------|----------|
| Web Login | Estaciones de trabajo con navegador | Navegador + SSO |
| SFDX URL | Contenedores, CI/CD, autenticación portátil | URL de autenticación de una sesión autenticada |
| JWT Bearer | Pipelines automatizados | Aplicación conectada + clave privada + clave de consumidor |
| Access Token | Autenticación mediante variable de entorno | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**Nota:** El flujo de dispositivo (`sf org login device`) está bloqueado desde agosto
de 2025 y no es compatible.

### Variables de entorno

| Variable | Propósito |
|----------|---------|
| `SF_ACCESS_TOKEN` | Token Bearer para autenticación por token de acceso |
| `SFDX_AUTH_URL` | URL de autenticación Force para autenticación por URL de SFDX |
| `SF_ORG_INSTANCE_URL` | URL de instancia de la organización |
| `SF_JWT_KEY_FILE` | Ruta a la clave privada JWT |
| `SF_CLIENT_ID` | Clave de consumidor de la aplicación conectada |
| `SF_USERNAME` | Nombre de usuario de Salesforce para JWT |

## Comandos

### /salesforce:sf-status

Compruebe el estado de conexión de su organización de Salesforce, la versión de CLI y las
organizaciones autenticadas.

```
/salesforce:sf-status
```

**Qué esperar:** Una tabla que muestra la versión de su CLI sf, el alias de la organización autenticada,
el nombre de usuario, la URL de instancia, el estado de conexión y la versión de la API.

### /salesforce:sf-login

Autentíquese en una organización de Salesforce. Opcionalmente, proporcione un alias.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**Qué esperar:** Si una organización con ese alias ya está autenticada,
se informan los detalles de la conexión. Si no, verifica las credenciales disponibles
(variables de entorno) y se autentica utilizando el primer
método completamente satisfecho.

## Habilidades

### salesforce-index

Enrutador de intención de nivel superior. Se activa automáticamente cuando usted menciona
Salesforce, gestión de organizaciones o consultas de datos en lenguaje natural.

| Usted dice | Se enruta a |
|---------|-----------|
| "login to salesforce", "authenticate org" | Habilidad `salesforce-auth` |
| "check org status", "list orgs" | Agente `cli-operator` |
| "show me opportunities", "look up case" | Agente `cli-operator` (consulta SOQL) |
| "generate Apex class", "create Flow" | Habilidades de desarrollo de la biblioteca afv |

### salesforce-auth

Habilidad de autenticación adaptada para contenedores. Admite cuatro métodos de autenticación y
detecta automáticamente qué credenciales están disponibles. Prioriza:
token de acceso, luego JWT, luego URL de SFDX y finalmente inicio de sesión web.

## Agentes

### cli-operator

Agente autónomo de Salesforce CLI que ejecuta comandos `sf` con
salvaguardas de seguridad. Todas las operaciones de Salesforce se delegan a este agente para
mantener el contexto de la sesión principal liviano.

**Reglas de seguridad:**

- Solo lectura por defecto — las operaciones de escritura requieren aprobación explícita
- Nunca realiza implementaciones sin una vista previa y un paso de confirmación
- Nunca muestra tokens de acceso, URLs de autenticación ni credenciales en la salida
- Sanea todos los valores proporcionados por el usuario (rechaza metacaracteres de shell)
- Utiliza la salida `--json` para resultados estructurados y analizables

**Formato de respuesta:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## Guía de uso: Gestión de cuentas

<Aside type="note">
  Los prompts a continuación no son funcionalidades codificadas del plugin. Funcionan
  porque el agente cli-operator puede ejecutar cualquier comando `sf` de CLI, incluidas
  consultas SOQL arbitrarias. Los resultados dependen del modelo de datos de su organización y
  de los permisos de su usuario. Los prompts aquí han sido probados y verificados
  para funcionar con objetos estándar de Salesforce (Account, Contact, Case,
  Opportunity, AccountTeamMember, OpportunityTeamMember).
</Aside>

Estos prompts funcionan con cualquier organización de Salesforce. Reemplace los valores de marcador de posición
con su propia información. Cada prompt produce resultados específicos
a su rol, cuentas y territorio.

<Aside type="tip">
  Para obtener los mejores resultados, incluya su dirección de correo electrónico en los prompts que necesitan
  identificar su registro de usuario. Las organizaciones de Salesforce pueden tener múltiples usuarios
  con nombres similares.
</Aside>

### Primeros pasos

Después de autenticarse, verifique que todo funcione:

```
/salesforce:sf-status
```

Luego intente una consulta simple en lenguaje natural:

```
list my authenticated salesforce orgs
```

**Qué esperar:** Una tabla que muestra todas las organizaciones conectadas con alias,
nombre de usuario, URL de instancia y estado de conexión.

### Encuentre sus cuentas

Descubra a qué cuentas está asignado:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**Qué esperar:** Una lista de cuentas agrupadas por propietario de cuenta, que muestra
su rol de miembro del equipo en cada una. Si la mayoría de los roles aparecen vacíos, eso es
una brecha de higiene de datos común — las membresías del equipo de cuenta existen pero los roles
no fueron completados.

### Verificación cruzada de cobertura con un colega

Compare su cobertura de cuentas con la de un compañero para encontrar brechas:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**Qué esperar:** Una tabla resumen que muestra el recuento de superposición y el recuento de brechas,
seguida de dos listas. Si cambió de rol recientemente, puede encontrar
superposición cero — esto revela qué cuentas necesita que le sean añadidas.

<Aside type="note">
  También puede verificar a un predecesor añadiendo: "Also check if
  Previous Person Name appears on any of those accounts." Esto ayuda a
  identificar brechas de traspaso al reemplazar a alguien en un territorio.
</Aside>

### Resumen del pipeline del territorio

Vea todas las oportunidades abiertas en sus cuentas:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**Qué esperar:** Un resumen con el recuento total de oportunidades, el pipeline bruto
y el pipeline ponderado. Luego una tabla de distribución por etapa y una lista de las mejores
oportunidades por monto. La salida señala problemas de higiene de datos como
fechas de cierre vencidas o montos nulos.

### Análisis detallado de oportunidades

Obtenga el panorama completo de un negocio específico:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**Qué esperar:** La descripción general de la oportunidad (etapa, monto, fecha de cierre,
probabilidad, categoría de pronóstico, propietario), los miembros del equipo de oportunidades con
roles, los contactos etiquetados, las tareas y eventos recientes, y el historial de cambios de
campo que muestra cómo el monto del negocio y la fecha de cierre han evolucionado con el tiempo.

### Casos de soporte en todo el territorio

Verifique los casos de soporte abiertos en sus cuentas:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**Qué esperar:** Un recuento de casos abiertos con desglose por prioridad y estado,
luego los detalles de casos agrupados por cuenta. Los casos obsoletos (con meses o
años de antigüedad sin actividad) se señalan como candidatos de higiene.

### Pronóstico del pipeline trimestral

Obtenga una vista lista para el pronóstico de su pipeline:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**Qué esperar:** Una tabla con una fila por trimestre que muestra el recuento de oportunidades,
el pipeline total y el pipeline ponderado. Incluye un desglose de la combinación de etapas por trimestre
y resalta los negocios de mayor peso. Las oportunidades vencidas se agrupan por separado.

### Descripción general completa de la cuenta

Análisis detallado de un cliente específico:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**Qué esperar:** Perfil de la empresa (industria, ingresos, empleados,
ubicación), contactos clave con títulos y correo electrónico, oportunidades abiertas con
etapas y montos, y casos de soporte recientes con estado.

### Búsqueda de casos

Busque un caso de soporte específico por número:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**Qué esperar:** El asunto del caso, estado, prioridad, descripción, el
perfil de la cuenta del cliente y el propietario del caso con información de contacto.

## Guía de uso: Consultas de datos

Para usuarios avanzados, puede solicitar consultas específicas al estilo SOQL en lenguaje
natural. El plugin traduce su solicitud al SOQL apropiado y lo ejecuta.

### Campos específicos

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### Filtrado por fecha

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### Agregación

```
count all open salesforce cases grouped by priority and status across my accounts
```

### Objetos personalizados

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  Si una consulta falla porque un campo no existe, el plugin
  reintenta automáticamente sin ese campo. Los campos personalizados en su organización
  pueden usar nombres diferentes — consulte la Configuración de Salesforce si un campo
  no se encuentra.
</Aside>

npx skills add forcedotcom/afv-library
