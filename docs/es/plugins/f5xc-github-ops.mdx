El plugin **f5xc-github-ops** proporciona automatización exclusiva de
operaciones GitHub para los repositorios de f5xc-salesdemos. Gestiona
el ciclo de vida completo de Git y GitHub desde la creación de issues
hasta el monitoreo post-merge, aplicando desarrollo dirigido por issues
y validaciones de lint previas al commit.

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## Habilidades

### workflow-lifecycle

Flujo de trabajo de gobernanza del repositorio que gestiona la creación
de issues, convenciones de nomenclatura de ramas, creación de PRs,
consulta de CI con retroalimentación de errores a issues, monitoreo
post-merge, verificación y criterios de finalización de tareas. Se
activa cuando se dice "commit", "push", "create a PR", "merge" o se
realiza cualquier operación Git.

### upstream-contribution

Flujo de trabajo de contribución upstream para desarrollo basado en
forks. Gestiona la detección de forks, investigación upstream (issues,
PRs, CONTRIBUTING.md, patrones de CI), creación de issues y PRs entre
repositorios, gestión de issues de seguimiento local, monitoreo de CI
upstream y resolución guiada.

## Agentes

### github-ops

Agente exclusivo para todas las operaciones mecánicas de Git y GitHub.
Gestiona validaciones de lint previas al commit, creación de issues,
creación de ramas, preparación (staging), commits, push, creación de
PRs, consulta de CI con retroalimentación de errores, reintento ante
fallos de infraestructura, monitoreo post-merge, limpieza de ramas y
gestión de configuración del repositorio a través de la API de GitHub.

Devuelve informes de estado estructurados: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` o `FAILED`.

<Aside type="caution">
  Este agente es el gestor **exclusivo** de las operaciones Git.
  Nunca ejecute `git commit`, `git push` o `gh pr create` directamente
  en repositorios que utilicen este plugin.
</Aside>
