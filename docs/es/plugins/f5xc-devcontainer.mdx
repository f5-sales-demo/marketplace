El plugin **f5xc-devcontainer** proporciona conciencia de contenedor para
el entorno de desarrollo f5xc-salesdemos. Mantiene un
catálogo completo de herramientas, habilita la introspección de contenedor en vivo
y gestiona la instalación de herramientas con seguimiento de persistencia mediante
issues de GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Instalación

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Habilidades

### tool-catalog

Base de conocimiento de cada herramienta de CLI instalada en el contenedor de desarrollo,
organizada por categoría (nube, redes, seguridad, desarrollo,
etc.). Se activa cuando se pregunta qué herramienta utilizar, cómo usar
una herramienta, si algo está instalado, o cuando se necesita ayuda con una
operación específica.

### self-awareness

Habilidad de identidad y autodiagnóstico del contenedor. Proporciona
introspección en vivo del contenedor en ejecución mediante la API de GitHub y metadatos
locales de compilación. Se activa ante preguntas sobre identidad, versión, información de compilación,
verificaciones de estado e historial de colaboradores.

<Aside type="tip">
  Pregunta "¿quién eres?" o "¿qué versión estoy ejecutando?" para activar
  la habilidad de autoconciencia.
</Aside>

## Agentes

### tool-advisor

Lee el catálogo de herramientas y devuelve recomendaciones con el propósito,
comandos de inicio rápido y requisitos de autenticación para cualquier pregunta
sobre herramientas de CLI.

### tool-auditor

Audita el Dockerfile del contenedor de desarrollo contra los archivos de referencia del catálogo
de herramientas para detectar desviaciones &mdash; herramientas añadidas a la imagen
pero no catalogadas, o entradas del catálogo para herramientas que han sido
eliminadas.

### container-introspector

Realiza introspección en vivo del contenedor mediante la API de GitHub y metadatos
locales para identidad, genealogía, autodiagnóstico, historial de colaboradores
y linaje de compilación. Solo operaciones de lectura.

### container-maintainer

Instala, elimina, busca y actualiza herramientas de CLI en el contenedor
en ejecución utilizando los patrones correctos del gestor de paquetes. Registra issues de GitHub
contra el repositorio del contenedor de desarrollo para el seguimiento de persistencia.
