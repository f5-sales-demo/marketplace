## marketplace.json 結構描述

Marketplace 清單檔位於儲存庫根目錄的
`.claude-plugin/marketplace.json`。

### 頂層欄位

| 欄位 | 必填 | 說明 |
|-------|----------|-------------|
| `$schema` | 否 | 用於驗證的 JSON Schema URL |
| `name` | 是 | Marketplace 唯一識別碼 |
| `version` | 否 | Marketplace 結構描述版本（語意化版本） |
| `metadata.description` | 是 | 人類可讀的 Marketplace 描述 |
| `owner.name` | 是 | 擁有此 Marketplace 的組織或使用者 |
| `owner.url` | 否 | 擁有者個人檔案或網站的 URL |
| `owner.email` | 否 | 聯絡電子郵件 |
| `plugins` | 是 | 外掛程式項目陣列 |

### 外掛程式項目欄位

| 欄位 | 必填 | 說明 |
|-------|----------|-------------|
| `name` | 是 | 外掛程式識別碼（必須與 `plugin.json` 相符） |
| `description` | 是 | 外掛程式的簡短描述 |
| `version` | 是 | 語意化版本字串 |
| `author.name` | 是 | 外掛程式作者 |
| `source` | 是 | 外掛程式的路徑或 URL（請參閱來源類型） |
| `category` | 是 | 外掛程式分類（例如 `productivity`） |
| `homepage` | 否 | 外掛程式文件或首頁的 URL |
| `license` | 否 | SPDX 授權識別碼 |
| `keywords` | 否 | 搜尋關鍵字陣列 |
| `tags` | 否 | 用於篩選的標籤陣列 |
| `repository` | 否 | 儲存庫 URL |

## 外掛程式來源類型

marketplace.json 中的 `source` 欄位支援多種
格式：

| 類型 | 範例 | 說明 |
|------|---------|-------------|
| 相對路徑 | `./plugins/my-plugin` | 位於同一儲存庫中的外掛程式 |
| GitHub 簡寫 | `owner/repo` | 位於儲存庫根目錄的外掛程式 |
| GitHub 含路徑 | `owner/repo/path/to/plugin` | 位於子目錄中的外掛程式 |
| Git URL | `https://github.com/owner/repo.git` | Git 儲存庫 |
| npm 套件 | `npm:@scope/package` | 已發佈的 npm 套件 |

## plugin.json 結構描述

每個外掛程式在其目錄內的
`.claude-plugin/plugin.json` 有一個清單檔。

| 欄位 | 必填 | 說明 |
|-------|----------|-------------|
| `name` | 是 | 外掛程式識別碼 |
| `description` | 是 | 外掛程式的功能說明 |
| `version` | 是 | 語意化版本 |
| `author.name` | 是 | 外掛程式作者 |
| `author.url` | 否 | 作者個人檔案或網站的 URL |
| `homepage` | 否 | 文件 URL |
| `keywords` | 否 | 搜尋關鍵字 |
| `license` | 否 | SPDX 授權識別碼 |
| `repository` | 否 | 儲存庫 URL |

## 外掛程式目錄配置

```
plugins/my-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## 環境變數

外掛程式可以在其技能和命令
檔案中使用這些變數：

| 變數 | 說明 |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | 外掛程式根目錄的絕對路徑 |

## 驗證

在本機驗證您的 Marketplace 和外掛程式清單檔：

```bash
claude plugin validate .
```

此命令會檢查：

- `marketplace.json` 結構描述合規性
- 所有參照的外掛程式目錄是否存在
- 每個外掛程式是否有有效的 `plugin.json`
- 技能是否有格式正確的 `SKILL.md` 前置資料
- 命令是否有必填的前置資料欄位

## 團隊配置

### 預先配置 Marketplace

在任何儲存庫的 `.claude/settings.json` 中新增：

```json
{
  "extraKnownMarketplaces": [
    "f5xc-salesdemos/marketplace"
  ]
}
```

### 預先啟用外掛程式

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5xc-salesdemos-marketplace"
  ]
}
```

<Aside type="tip">
  將這些設定提交到您的儲存庫，可確保每位
  團隊成員自動獲得相同的 Marketplace 和外掛程式
  配置。
</Aside>

## 疑難排解

### 找不到 Marketplace

確認 Marketplace 是否已新增：

```
/plugin marketplace list
```

如果缺少，請重新新增：

```
/plugin marketplace add f5xc-salesdemos/marketplace
```

### 外掛程式未啟用

- 確認外掛程式已安裝：
  `/plugin list`
- 確認您的 Claude Code 版本支援外掛程式
  （v1.0.33+）
- 確認技能的觸發條件與您目前的
  情境相符

### 技能未自動觸發

技能根據 SKILL.md 前置資料中的 `description` 欄位
進行啟用。如果技能未如預期啟用：

- 確認描述是否準確說明了
  觸發情境
- 使用命令直接執行（例如 `/f5xc-docs-tools:review-mdx`）
  作為替代方案

### 審查後出現建置錯誤

外掛程式會回報問題但不會自動修正。請手動
套用建議的修正，然後重新執行審查以
確認問題已解決。
