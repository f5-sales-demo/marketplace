**f5xc-devcontainer** 插件為 f5-sales-demo 開發環境提供容器感知功能。它維護一份完整的工具目錄、支援即時容器自省，並透過 GitHub Issues 處理具備持久性追蹤的工具安裝作業。

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## 安裝

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## 技能

### tool-catalog

devcontainer 中所有已安裝 CLI 工具的知識庫，依類別（雲端、網路、安全性、開發等）進行組織。當您詢問應使用哪種工具、如何使用某工具、某工具是否已安裝，或需要特定操作協助時，此技能將自動啟動。

### self-awareness

容器識別與自我診斷技能。透過 GitHub API 及本地建置元數據提供執行中容器的即時自省功能。適用於識別身份、版本資訊、建置資訊、健康狀態檢查及貢獻者歷史等問題。

<Aside type="tip">
  輸入「who are you?」或「what version am I running?」即可觸發 self-awareness 技能。
</Aside>

## 代理程式

### tool-advisor

讀取工具目錄，並針對任何 CLI 工具問題回傳包含用途說明、快速入門指令及驗證需求的建議。

### tool-auditor

根據工具目錄參考文件稽核 devcontainer Dockerfile，以偵測差異——包括已加入映像但未列入目錄的工具，或目錄中已記載但工具已被移除的條目。

### container-introspector

透過 GitHub API 及本地元數據執行即時容器自省，涵蓋身份識別、溯源追蹤、自我診斷、貢獻者歷史及建置血緣分析。僅支援唯讀操作。

### container-maintainer

使用正確的套件管理器模式，在執行中的容器內安裝、移除、搜尋及更新 CLI 工具。並向 devcontainer 儲存庫提交 GitHub Issues 以進行持久性追蹤。
