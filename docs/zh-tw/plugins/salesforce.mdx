**salesforce** 外掛程式將 Salesforce CLI 與 Claude Code 整合，
提供組織驗證及通用 CLI 代理程式，可執行任何 `sf` 指令，包括 SOQL 查詢。
它支援採用 JWT、存取權杖及 SFDX URL 驗證流程的無頭容器環境。
以下使用指南示範如何透過 CLI 代理程式以自然語言查詢帳戶、商機、案例及聯絡人。

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## 安裝

```
/plugin install salesforce@f5-sales-demo-marketplace
```

## 先決條件

- **Salesforce CLI** (`@salesforce/cli`)
- 具備 API 存取權限的 **Salesforce 組織**
- 擁有查詢物件（案例、商機、帳戶、聯絡人）權限的使用者帳戶

在您的工作站上安裝 Salesforce CLI：

```bash
brew install sf
```

確認安裝結果：

```bash
sf --version
```

您應會看到類似 `@salesforce/cli/2.x.x` 的輸出內容。

## 驗證設定

### 步驟一：找到您的 Salesforce 網域

查看瀏覽器中的 Salesforce URL。若您透過
`https://acme.lightning.force.com` 存取 Salesforce，則您的登入網域為
`https://acme.my.salesforce.com`。

<Aside type="tip">
  若您的公司使用 SSO，您必須透過 `--instance-url` 指定自訂網域，
  使登入流程透過您的 SSO 提供者進行，而非使用一般的 Salesforce 登入頁面。
</Aside>

### 步驟二：透過瀏覽器進行驗證

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

將 `YOUR-DOMAIN` 替換為步驟一中您公司的網域。瀏覽器將開啟——
完成 SSO 或登入流程並授權應用程式。您應會看到：

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### 步驟三：確認連線

```bash
sf org display --target-org my-org
```

確認輸出內容顯示 `Connected Status: Connected`，並包含您的使用者名稱及執行個體 URL。

### 步驟四：容器的可攜式驗證（選用）

若您需要在無頭環境（CI/CD、容器、遠端工作階段）中進行驗證，
請從您的工作站匯出 SFDX 驗證 URL：

```bash
sf org display --verbose --target-org my-org
```

複製 **Sfdx Auth Url** 的值（以 `force://` 開頭）。在容器中執行：

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX 驗證 URL 屬於憑證——請視同密碼處理。請勿將其提交至檔案或貼入對話中。
  僅使用環境變數及標準輸入管道。
</Aside>

### 驗證方法參考

| 方法 | 最適合 | 需求 |
|--------|----------|----------|
| 網頁登入 | 具備瀏覽器的工作站 | 瀏覽器 + SSO |
| SFDX URL | 容器、CI/CD、可攜式驗證 | 來自已驗證工作階段的驗證 URL |
| JWT Bearer | 自動化管道 | 已連接應用程式 + 私鑰 + 消費者金鑰 |
| 存取權杖 | 環境變數驗證 | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**注意：** 裝置流程（`sf org login device`）自 2025 年 8 月起已封鎖，不再受支援。

### 環境變數

| 變數 | 用途 |
|----------|---------|
| `SF_ACCESS_TOKEN` | 存取權杖驗證的 Bearer 權杖 |
| `SFDX_AUTH_URL` | SFDX URL 驗證的強制驗證 URL |
| `SF_ORG_INSTANCE_URL` | 組織執行個體 URL |
| `SF_JWT_KEY_FILE` | JWT 私鑰的路徑 |
| `SF_CLIENT_ID` | 已連接應用程式的消費者金鑰 |
| `SF_USERNAME` | JWT 的 Salesforce 使用者名稱 |

## 指令

### /salesforce:sf-status

檢查您的 Salesforce 組織連線狀態、CLI 版本及已驗證的組織。

```
/salesforce:sf-status
```

**預期結果：** 一個表格，顯示您的 sf CLI 版本、已驗證的組織別名、使用者名稱、
執行個體 URL、連線狀態及 API 版本。

### /salesforce:sf-login

驗證至 Salesforce 組織。可選擇性地提供別名。

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**預期結果：** 若具有該別名的組織已完成驗證，則會回報連線詳細資訊。
若尚未驗證，則會檢查可用的憑證（環境變數），並使用第一個完全滿足條件的方法進行驗證。

## 技能

### salesforce-index

頂層意圖路由器。當您以自然語言提及 Salesforce、組織管理或資料查詢時，會自動啟動。

| 您的說法 | 路由至 |
|---------|-----------|
| 「登入 salesforce」、「驗證組織」 | `salesforce-auth` 技能 |
| 「檢查組織狀態」、「列出組織」 | `cli-operator` 代理程式 |
| 「顯示我的商機」、「查詢案例」 | `cli-operator` 代理程式（SOQL 查詢） |
| 「產生 Apex 類別」、「建立 Flow」 | afv-library 開發技能 |

### salesforce-auth

適用於容器的驗證技能。支援四種驗證方法，並自動偵測可用的憑證。
優先順序為：存取權杖、JWT、SFDX URL，最後是網頁登入。

## 代理程式

### cli-operator

具備安全防護機制的自主 Salesforce CLI 代理程式，可執行 `sf` 指令。
所有 Salesforce 操作均委派給此代理程式，以保持主要工作階段的上下文精簡。

**安全規則：**

- 預設為唯讀——寫入操作需要明確核准
- 部署前必須先預覽並確認，絕不直接部署
- 絕不在輸出中顯示存取權杖、驗證 URL 或憑證
- 清理所有使用者提供的值（拒絕 shell 元字元）
- 使用 `--json` 輸出以取得結構化、可解析的結果

**回應格式：**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## 使用指南：帳戶管理

<Aside type="note">
  以下提示並非外掛程式的固定功能。它們能夠運作，是因為 cli-operator 代理程式
  可以執行任何 `sf` CLI 指令，包括任意 SOQL 查詢。結果取決於您組織的資料模型
  及您的使用者權限。此處的提示已通過測試，確認可與標準 Salesforce 物件
  （Account、Contact、Case、Opportunity、AccountTeamMember、OpportunityTeamMember）搭配使用。
</Aside>

這些提示適用於任何 Salesforce 組織。請將預留位置值替換為您自己的資訊。
每個提示所產生的結果，皆針對您的角色、帳戶及負責區域而定。

<Aside type="tip">
  為獲得最佳結果，在需要識別您的使用者記錄的提示中，請加入您的電子郵件地址。
  Salesforce 組織中可能存在多個名稱相似的使用者。
</Aside>

### 開始使用

完成驗證後，確認一切運作正常：

```
/salesforce:sf-status
```

接著嘗試簡單的自然語言查詢：

```
list my authenticated salesforce orgs
```

**預期結果：** 一個表格，顯示所有已連線組織的別名、使用者名稱、執行個體 URL 及連線狀態。

### 尋找您的帳戶

探索您被指派至哪些帳戶：

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**預期結果：** 依帳戶擁有者分組的帳戶清單，顯示您在每個帳戶中的團隊成員角色。
若大多數角色顯示為空白，這是常見的資料衛生問題——帳戶團隊成員資格存在，但角色未填寫。

### 與同事交叉比對負責範圍

與隊友比較帳戶負責範圍，找出缺口：

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**預期結果：** 一個顯示重疊數量及缺口數量的摘要表格，後接兩份清單。
若您最近更換了職務，可能會發現重疊為零——這揭示了您需要被加入的帳戶。

<Aside type="note">
  您也可以查詢前任負責人，方法是加上：「Also check if
  Previous Person Name appears on any of those accounts.」
  這有助於在接手某人的負責區域時，識別交接缺口。
</Aside>

### 負責區域管道總覽

查看您帳戶中所有開放中的商機：

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**預期結果：** 含商機總數、原始管道及加權管道的摘要。接著是階段分佈表格，
以及依金額排列的主要商機清單。輸出內容會標記資料衛生問題，例如逾期的結案日期或金額為空值。

### 商機深入分析

取得特定交易的完整概況：

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**預期結果：** 商機概況（階段、金額、結案日期、可能性、預測類別、負責人）、
商機團隊成員及其角色、已標記的聯絡人、近期任務與活動，
以及顯示交易金額與結案日期隨時間變化的欄位變更歷程。

### 負責區域的支援案例

查看您帳戶中開放中的支援案例：

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**預期結果：** 開放案例數量及優先順序與狀態的分佈，
接著是依帳戶分組的案例詳細資訊。長期未有活動的陳舊案例（數月或數年）會被標記為衛生候選項目。

### 季度管道預測

取得適用於預測的管道視圖：

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**預期結果：** 每季一列的表格，顯示商機數量、總管道及加權管道。
包含每季的階段組合分佈，並突顯加權金額最高的交易。逾期商機會單獨分組列出。

### 完整帳戶概況

深入了解特定客戶：

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**預期結果：** 公司概況（產業、營收、員工人數、地點）、附有職稱及電子郵件的主要聯絡人、
含階段及金額的開放商機，以及附有狀態的近期支援案例。

### 案例查詢

依案例編號查詢特定支援案例：

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**預期結果：** 案例主旨、狀態、優先順序、說明、客戶帳戶概況，
以及附有聯絡資訊的案例負責人。

## 使用指南：資料查詢

進階使用者可以以自然語言提出特定的 SOQL 式查詢。
外掛程式會將您的請求轉換為適當的 SOQL 並執行。

### 指定欄位

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### 日期篩選

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### 彙總

```
count all open salesforce cases grouped by priority and status across my accounts
```

### 自訂物件

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  若查詢因欄位不存在而失敗，外掛程式會自動移除該欄位並重試。
  您組織中的自訂欄位可能使用不同的名稱——若找不到某個欄位，
  請在 Salesforce 設定中確認。
</Aside>

```bash
npx skills add forcedotcom/afv-library
```
