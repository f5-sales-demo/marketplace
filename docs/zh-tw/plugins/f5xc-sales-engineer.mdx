**f5xc-sales-engineer** 外掛程式為 F5 XC 示範儲存庫提供可重複使用的銷售工程師角色框架。它處理完整的示範會議生命週期——從會議前驗證，到現場執行、問答，以及拆除。

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## 安裝

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## 技能

### demo-executor

以 API 驅動的示範執行，包含四階段會議生命週期：

1. **準備（Prepare）** — 透過 `demo-housekeeping` 代理程式進行會議前環境驗證
2. **執行（Execute）** — 具有確定性的階段執行，並提供面向客戶的解說
3. **問答（Q&A）** — 搭配現場環境的即興模式
4. **拆除（Teardown）** — 透過 `demo-housekeeping` 代理程式進行會議後清理

從 `DEMO_PRODUCT_EXPERTISE.md` 讀取產品專屬內容，並從 `docs/demo/` 讀取示範指令。

### presenter

使用預先設定的示範環境進行實際建置狀態的逐步演示。在每個步驟遵循「展示 → 解說 → 連結 → 暫停」的模式。從 `DEMO_WALKTHROUGH_CONFIG.md` 讀取演示順序。

### subject-matter-expert

具有嚴格引用要求的參考文獻支援問答。絕不猜測——每個答案均包含參考資料或證明。在需要時將研究工作委派給 `demo-researcher` 代理程式。

### sales-engineer

將使用者意圖路由至正確角色技能的角色索引。當請求與示範相關但不符合特定觸發詞時啟動。

## 代理程式

### demo-housekeeping

用於會議前驗證及會議後拆除的自主代理程式。執行定義於 `DEMO_READINESS_MATRIX.md` 的就緒驗證矩陣（T0–T5），並執行來自 `docs/demo/phase-4-teardown.mdx` 的拆除指令。

### demo-researcher

僅限讀取的研究資料庫管理員，負責找出附有引用的已驗證答案。優先搜尋本機 `docs/`，其次搜尋 `DEMO_SOURCE_INDEX.md` 中已建立索引的外部來源，並以網路搜尋作為備援。

## 慣例檔案

<Aside type="tip" title="產品專屬內容">
  此外掛程式與產品無關。每個儲存庫透過標準化慣例檔案提供產品專屬內容。
</Aside>

在您的儲存庫根目錄建立以下檔案以設定外掛程式：

| 檔案 | 用途 | 必要技能 |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | 產品功能、偵測訊號、威脅涵蓋範圍、合規對應、API 參考 | 所有技能 |
| `DEMO_WALKTHROUGH_CONFIG.md` | 示範應用程式 URL、演示順序、偵測時機 | presenter |
| `DEMO_SOURCE_INDEX.md` | 包含本機文件及外部 URL 的研究來源目錄 | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | 必要／選用變數、就緒檢查（T0–T5）、API 端點 | demo-housekeeping |
| `docs/demo/` | 含有 cURL 指令及證據閘道的階段檔案 | demo-executor |

## 觸發詞

`.claude/CLAUDE.md` 觸發對應（由 `docs-control` 管理）會根據使用者意圖啟動對應技能：

| 詞語 | 技能 | 階段 |
| ------ | ----- | ----- |
| "run the demo"、"execute the demo" | demo-executor | 執行（Execute） |
| "prepare the demo"、"pre-flight" | demo-executor | 準備（Prepare） |
| "Q&A"、"question and answer" | demo-executor | 問答（Q&A） |
| "tear down"、"clean up" | demo-executor | 拆除（Teardown） |
| "walk through the demo"、"present" | presenter | — |
| "answer questions"、"explain" | subject-matter-expert | — |

## 解說風格

所有角色共用一致的解說風格：

- **現在式、第一人稱複數** — 「我們目前看到的是……」
- **每段落一個關注點** — 每段解說回答：這是什麼？為何重要？應該採取什麼行動？
- **點名訊號** — 指出目前發揮作用的是哪項功能
- **適時加入合規連結** — 自然地提及法規對應情況
- **節奏暫停標記** — `> ⏸ *[Pause for audience — ready to continue?]*`
