本指南說明如何建立新的外掛程式並將其新增至
f5xc-salesdemos 市集。

## 外掛程式結構

每個外掛程式都位於 `plugins/` 下的獨立目錄中，並遵循
以下結構：

<FileTree>
- plugins/
  - my-plugin/
    - .claude-plugin/
      - plugin.json
    - skills/
      - my-skill/
        - SKILL.md
        - references/
          - reference-data.md
    - commands/
      - my-command.md
    - agents/
      - my-agent.md
    - README.md
</FileTree>

## 建立外掛程式

<Steps>

1. **建立外掛程式目錄**

   ```bash
   mkdir -p plugins/my-plugin/.claude-plugin
   mkdir -p plugins/my-plugin/skills
   mkdir -p plugins/my-plugin/commands
   ```

2. **撰寫 `plugin.json`**

   建立 `plugins/my-plugin/.claude-plugin/plugin.json`：

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": {
       "name": "f5xc-salesdemos"
     },
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "keywords": ["relevant", "keywords"],
     "license": "Apache-2.0",
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

3. **新增技能**

   建立 `plugins/my-plugin/skills/my-skill/SKILL.md` 並包含
   frontmatter：

   ```markdown
   ---
   name: my-skill
   description: One-line description of what this skill does
   ---

   Detailed instructions for Claude when this skill activates.
   ```

   將任何參考資料放置在 `SKILL.md` 旁邊的 `references/`
   子目錄中。

4. **新增命令**

   建立 `plugins/my-plugin/commands/my-command.md`：

   ```markdown
   ---
   description: What this command does
   argument_hint: "[optional-args]"
   allowed_tools:
     - Read
     - Glob
     - Grep
   ---

   Instructions for Claude when the user invokes this command.
   ```

5. **將外掛程式新增至 marketplace.json**

   在 `.claude-plugin/marketplace.json` 的 `plugins` 陣列中
   新增一筆項目：

   ```json
   {
     "name": "my-plugin",
     "description": "What this plugin does",
     "version": "1.0.0",
     "author": { "name": "f5xc-salesdemos" },
     "source": "./plugins/my-plugin",
     "category": "productivity",
     "homepage": "https://github.com/f5xc-salesdemos/marketplace/tree/main/plugins/my-plugin",
     "license": "Apache-2.0",
     "keywords": ["relevant", "keywords"],
     "tags": ["searchable", "tags"],
     "repository": "https://github.com/f5xc-salesdemos/marketplace"
   }
   ```

6. **在本機測試**

   在不發佈的情況下測試您的外掛程式，將 Claude Code
   指向本機目錄：

   ```bash
   claude --plugin-dir ./plugins/my-plugin
   ```

   驗證技能是否在正確的情境中啟動，以及命令是否產生
   預期的輸出。

</Steps>

## 提交您的外掛程式

<Aside type="tip" title="儲存庫工作流程">
  本儲存庫強制執行嚴格的治理工作流程。
  請參閱 [CONTRIBUTING.md](https://github.com/f5xc-salesdemos/marketplace/blob/main/CONTRIBUTING.md)
  了解完整流程。
</Aside>

<Steps>

1. 建立一個 GitHub issue 來描述新的外掛程式
2. 從 `main` 建立一個功能分支
3. 新增您的外掛程式目錄並更新 `marketplace.json`
4. 在 `docs/plugins/my-plugin.mdx` 新增一個文件頁面
5. 開啟一個 PR 並連結至 issue，使用 `Closes #N`
6. 修復所有 CI 失敗，並在檢查通過後合併

</Steps>

## 最佳實踐

- **保持技能聚焦** — 一個技能應該只做好一件事
- **使用參考檔案** — 將大型資料集、結構描述和查詢表放在
  `references/` 中，而不是內嵌在 SKILL.md 中
- **智慧地限定範圍** — 盡可能檢查未提交的變更以縮小
  工作集
- **依嚴重程度分組輸出** — 使用 ERROR、WARNING 和 INFO
  層級，讓使用者能夠優先處理修復
- **撰寫 README** — 包含使用範例以及每個技能和命令的
  功能描述
