يوفر المكوّن الإضافي **f5xc-devcontainer** وعيًا بالحاوية لبيئة تطوير f5-sales-demo. يحتفظ بكتالوج شامل للأدوات، ويتيح الاستبطان المباشر للحاوية، ويتولى تثبيت الأدوات مع تتبع الاستمرارية عبر مشكلات GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## المهارات

### tool-catalog

قاعدة معرفية بكل أداة CLI مثبتة في حاوية التطوير، منظمة حسب الفئة (السحابة، الشبكات، الأمان، التطوير، إلخ). تُفعَّل عند سؤالك عن الأداة المناسبة للاستخدام، أو كيفية استخدام أداة معينة، أو ما إذا كان شيء ما مثبتًا، أو عند الحاجة إلى مساعدة في عملية محددة.

### self-awareness

مهارة هوية الحاوية والتشخيص الذاتي. توفر استبطانًا مباشرًا للحاوية قيد التشغيل عبر GitHub API وبيانات البناء المحلية. تُفعَّل لأسئلة الهوية، والإصدار، ومعلومات البناء، وفحوصات الصحة، وتاريخ المساهمين.

<Aside type="tip">
  اسأل "من أنت؟" أو "ما الإصدار الذي أشغّله؟" لتشغيل مهارة الوعي الذاتي.
</Aside>

## الوكلاء

### tool-advisor

يقرأ كتالوج الأدوات ويعيد توصيات تتضمن الغرض منها، وأوامر البدء السريع، ومتطلبات المصادقة لأي سؤال يتعلق بأداة CLI.

### tool-auditor

يراجع ملف Dockerfile الخاص بحاوية التطوير مقارنةً بملفات مرجع كتالوج الأدوات للكشف عن الانحراف — الأدوات المضافة إلى الصورة دون تسجيلها في الكتالوج، أو إدخالات الكتالوج للأدوات التي تمت إزالتها.

### container-introspector

يُجري استبطانًا مباشرًا للحاوية عبر GitHub API والبيانات الوصفية المحلية للهوية، والنسب، والتشخيص الذاتي، وتاريخ المساهمين، وسلسلة البناء. عمليات للقراءة فقط.

### container-maintainer

يثبّت ويزيل ويبحث عن أدوات CLI ويحدّثها في الحاوية قيد التشغيل باستخدام أنماط مدير الحزم الصحيحة. يرفع مشكلات GitHub ضد مستودع حاوية التطوير لتتبع الاستمرارية.
