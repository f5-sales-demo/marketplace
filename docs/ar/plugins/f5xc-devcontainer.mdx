توفر إضافة **f5xc-devcontainer** الوعي بالحاوية لبيئة تطوير
f5xc-salesdemos. فهي تحافظ على كتالوج شامل للأدوات، وتتيح
الاستبطان المباشر للحاوية، وتتولى تثبيت الأدوات مع تتبع
الاستمرارية عبر مشكلات GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="إنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## المهارات

### tool-catalog

قاعدة معرفية لكل أداة سطر أوامر مثبتة في حاوية التطوير،
منظمة حسب الفئة (السحابة، الشبكات، الأمان، التطوير،
إلخ). تنشط عندما تسأل عن الأداة التي يجب استخدامها، أو كيفية
استخدام أداة ما، أو ما إذا كان شيء ما مثبتاً، أو تحتاج
مساعدة في عملية محددة.

### self-awareness

مهارة هوية الحاوية والتشخيص الذاتي. توفر استبطاناً مباشراً
للحاوية قيد التشغيل عبر واجهة برمجة تطبيقات GitHub وبيانات
البناء المحلية التعريفية. تنشط لأسئلة الهوية والإصدار ومعلومات
البناء وفحوصات السلامة وسجل المساهمين.

<Aside type="tip">
  اسأل "من أنت؟" أو "ما الإصدار الذي أعمل عليه؟" لتفعيل
  الوعي الذاتي.
</Aside>

## الوكلاء

### tool-advisor

يقرأ كتالوج الأدوات ويعيد توصيات مع الغرض وأوامر البدء
السريع ومتطلبات المصادقة لأي سؤال يتعلق بأدوات سطر الأوامر.

### tool-auditor

يدقق ملف Dockerfile الخاص بحاوية التطوير مقابل ملفات مرجع
كتالوج الأدوات للكشف عن الانحراف &mdash; أدوات أُضيفت إلى
الصورة لكنها غير مفهرسة في الكتالوج، أو إدخالات كتالوج
لأدوات تمت إزالتها.

### container-introspector

ينفذ استبطاناً مباشراً للحاوية عبر واجهة برمجة تطبيقات GitHub
والبيانات التعريفية المحلية للهوية والنسب والتشخيص الذاتي
وسجل المساهمين وسلسلة البناء. عمليات للقراءة فقط.

### container-maintainer

يثبت ويزيل ويبحث ويحدّث أدوات سطر الأوامر في الحاوية قيد
التشغيل باستخدام أنماط مدير الحزم الصحيحة. يرفع مشكلات
على GitHub في مستودع حاوية التطوير لتتبع الاستمرارية.
