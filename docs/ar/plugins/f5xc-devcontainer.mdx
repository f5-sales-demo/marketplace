يوفر المكوّن الإضافي **f5xc-devcontainer** الوعيَ بالحاويات لبيئة تطوير f5xc-salesdemos. يحتفظ بكتالوج شامل للأدوات، ويتيح الفحص المباشر للحاوية، ويتولى تثبيت الأدوات مع تتبع الاستمرارية عبر GitHub issues.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## المهارات

### tool-catalog

قاعدة معرفية بكل أداة CLI مثبّتة في حاوية التطوير، منظّمة حسب الفئة (السحابة، الشبكات، الأمان، التطوير، إلخ). تُفعَّل عند سؤالك عن الأداة المناسبة للاستخدام، أو كيفية استخدام أداة معينة، أو ما إذا كان شيء ما مثبّتًا، أو عند الحاجة إلى مساعدة في عملية محددة.

### self-awareness

مهارة هوية الحاوية والتشخيص الذاتي. توفر فحصًا مباشرًا للحاوية قيد التشغيل عبر GitHub API وبيانات البناء المحلية. تُفعَّل للتساؤلات المتعلقة بالهوية، والإصدار، ومعلومات البناء، وفحوصات الصحة، وتاريخ المساهمين.

<Aside type="tip">
  اسأل "من أنت؟" أو "ما الإصدار الذي أستخدمه؟" لتفعيل مهارة الوعي الذاتي.
</Aside>

## العوامل

### tool-advisor

يقرأ كتالوج الأدوات ويعيد توصيات تتضمن الغرض من الأداة، وأوامر البدء السريع، ومتطلبات المصادقة لأي سؤال يتعلق بأداة CLI.

### tool-auditor

يراجع ملف Dockerfile الخاص بحاوية التطوير في مقابل الملفات المرجعية لكتالوج الأدوات للكشف عن الانحراف &mdash; أي الأدوات المضافة إلى الصورة دون إدراجها في الكتالوج، أو الإدخالات الكتالوجية للأدوات التي أُزيلت.

### container-introspector

يُجري فحصًا مباشرًا للحاوية عبر GitHub API والبيانات الوصفية المحلية للهوية، والنسب، والتشخيص الذاتي، وتاريخ المساهمين، وسلسلة البناء. عمليات القراءة فقط.

### container-maintainer

يُثبّت أدوات CLI ويُزيلها ويبحث عنها ويُحدّثها في الحاوية قيد التشغيل باستخدام أنماط مدير الحزم الصحيحة. يُنشئ GitHub issues في مستودع حاوية التطوير لتتبع الاستمرارية.
