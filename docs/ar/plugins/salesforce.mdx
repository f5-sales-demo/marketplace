يدمج ملحق **salesforce** واجهة Salesforce CLI مع Claude Code،
ويوفر مصادقة المؤسسة ووكيل CLI للأغراض العامة القادر على
تشغيل أي أمر `sf` بما في ذلك استعلامات SOQL. يدعم البيئات الحاوية
بلا رأس (headless) مع تدفقات مصادقة JWT وtoken-الوصول وURL الـ SFDX.
يوضح دليل الاستخدام أدناه كيفية استخدام اللغة الطبيعية للاستعلام عن الحسابات
والفرص والحالات وجهات الاتصال عبر وكيل CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## التثبيت

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## المتطلبات المسبقة

- **Salesforce CLI** (`@salesforce/cli`)
- **مؤسسة Salesforce** مع وصول API
- حساب مستخدم بصلاحيات الاستعلام عن الكائنات (Cases وOpportunities وAccounts وContacts)

قم بتثبيت Salesforce CLI على محطة عملك:

```bash
brew install sf
```

تحقق من التثبيت:

```bash
sf --version
```

يجب أن ترى مخرجات مثل `@salesforce/cli/2.x.x`.

## إعداد المصادقة

### الخطوة 1: ابحث عن نطاق Salesforce الخاص بك

انظر إلى عنوان URL الخاص بـ Salesforce في المتصفح. إذا كنت تصل إلى Salesforce على
`https://acme.lightning.force.com`، فإن نطاق تسجيل الدخول الخاص بك هو
`https://acme.my.salesforce.com`.

<Aside type="tip">
  إذا كانت شركتك تستخدم SSO، فيجب عليك تحديد نطاقك المخصص باستخدام
  `--instance-url` حتى يتم توجيه تسجيل الدخول عبر موفر SSO الخاص بك
  بدلاً من صفحة تسجيل دخول Salesforce العامة.
</Aside>

### الخطوة 2: المصادقة عبر المتصفح

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

استبدل `YOUR-DOMAIN` بنطاق شركتك من الخطوة 1. يفتح متصفحك —
أكمل تدفق SSO أو تسجيل الدخول وفوّض التطبيق. يجب أن ترى:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### الخطوة 3: تحقق من الاتصال

```bash
sf org display --target-org my-org
```

تأكد من أن المخرجات تُظهر `Connected Status: Connected` مع اسم المستخدم
وعنوان URL للمثيل.

### الخطوة 4: مصادقة محمولة للحاويات (اختياري)

إذا كنت بحاجة إلى المصادقة في بيئة بلا رأس (CI/CD، حاوية،
جلسة عن بُعد)، صدّر عنوان URL لمصادقة SFDX من محطة عملك:

```bash
sf org display --verbose --target-org my-org
```

انسخ قيمة **Sfdx Auth Url** (تبدأ بـ `force://`). في
الحاوية، شغّل:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  عنوان URL لمصادقة SFDX هو بيانات اعتماد — تعامل معه مثل كلمة المرور. لا
  تقم بإيداعه في الملفات أو لصقه في الدردشة. استخدم متغيرات البيئة
  وأنابيب stdin فقط.
</Aside>

### مرجع طرق المصادقة

| الطريقة | الأنسب لـ | يتطلب |
|--------|----------|----------|
| تسجيل الدخول عبر الويب | محطات العمل التي تمتلك متصفحاً | متصفح + SSO |
| SFDX URL | الحاويات، CI/CD، المصادقة المحمولة | عنوان URL للمصادقة من جلسة مصادق عليها |
| JWT Bearer | خطوط الأنابيب الآلية | Connected App + مفتاح خاص + مفتاح المستهلك |
| رمز الوصول | مصادقة متغير البيئة | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**ملاحظة:** تدفق الجهاز (`sf org login device`) محظور منذ أغسطس
2025 وغير مدعوم.

### متغيرات البيئة

| المتغير | الغرض |
|----------|---------|
| `SF_ACCESS_TOKEN` | رمز Bearer لمصادقة رمز الوصول |
| `SFDX_AUTH_URL` | عنوان URL لمصادقة Force لمصادقة SFDX URL |
| `SF_ORG_INSTANCE_URL` | عنوان URL لمثيل المؤسسة |
| `SF_JWT_KEY_FILE` | المسار إلى المفتاح الخاص لـ JWT |
| `SF_CLIENT_ID` | مفتاح مستهلك Connected App |
| `SF_USERNAME` | اسم مستخدم Salesforce لـ JWT |

## الأوامر

### /salesforce:sf-status

تحقق من حالة اتصال مؤسسة Salesforce الخاصة بك وإصدار CLI
والمؤسسات المصادق عليها.

```
/salesforce:sf-status
```

**ما يمكن توقعه:** جدول يُظهر إصدار sf CLI الخاص بك، واسم مستعار المؤسسة
المصادق عليها، واسم المستخدم، وعنوان URL للمثيل، وحالة الاتصال، وإصدار API.

### /salesforce:sf-login

المصادقة على مؤسسة Salesforce. يمكنك اختيارياً تقديم اسم مستعار.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**ما يمكن توقعه:** إذا كانت مؤسسة بهذا الاسم المستعار مصادقاً عليها بالفعل،
فإنها تُبلّغ عن تفاصيل الاتصال. إذا لم يكن كذلك، فإنها تتحقق من بيانات
الاعتماد المتاحة (متغيرات البيئة) وتصادق باستخدام الطريقة الأولى المستوفاة بالكامل.

## المهارات

### salesforce-index

موجّه النوايا على المستوى الأعلى. يُنشَّط تلقائياً عندما تذكر
Salesforce أو إدارة المؤسسة أو استعلامات البيانات باللغة الطبيعية.

| ما تقوله | يوجّه إلى |
|---------|-----------|
| "login to salesforce"، "authenticate org" | مهارة `salesforce-auth` |
| "check org status"، "list orgs" | وكيل `cli-operator` |
| "show me opportunities"، "look up case" | وكيل `cli-operator` (استعلام SOQL) |
| "generate Apex class"، "create Flow" | مهارات تطوير afv-library |

### salesforce-auth

مهارة مصادقة متكيفة مع الحاويات. تدعم أربع طرق مصادقة
وتكتشف تلقائياً بيانات الاعتماد المتاحة. تُعطي الأولوية لـ:
رمز الوصول، ثم JWT، ثم SFDX URL، ثم تسجيل الدخول عبر الويب.

## الوكلاء

### cli-operator

وكيل Salesforce CLI مستقل ينفذ أوامر `sf` مع ضمانات السلامة.
يتم تفويض جميع عمليات Salesforce إلى هذا الوكيل للحفاظ على سياق
الجلسة الرئيسية خفيفاً.

**قواعد السلامة:**

- للقراءة فقط بشكل افتراضي — عمليات الكتابة تتطلب موافقة صريحة
- لا ينشر أبداً بدون خطوة معاينة وتأكيد
- لا يُظهر أبداً رموز الوصول أو عناوين URL للمصادقة أو بيانات الاعتماد في المخرجات
- يُعقّم جميع القيم المقدمة من المستخدم (يرفض الأحرف الخاصة بالصدفة)
- يستخدم مخرجات `--json` للنتائج المنظمة القابلة للتحليل

**تنسيق الاستجابة:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## دليل الاستخدام: إدارة الحسابات

<Aside type="note">
  الأوامر أدناه ليست ميزات مشفرة في الملحق. إنها تعمل
  لأن وكيل cli-operator يمكنه تشغيل أي أمر `sf` CLI، بما في ذلك
  استعلامات SOQL التعسفية. تعتمد النتائج على نموذج بيانات مؤسستك
  وأذونات المستخدم لديك. تم اختبار الأوامر هنا والتحقق منها
  للعمل مع كائنات Salesforce القياسية (Account وContact وCase
  وOpportunity وAccountTeamMember وOpportunityTeamMember).
</Aside>

تعمل هذه الأوامر مع أي مؤسسة Salesforce. استبدل القيم
النائبة بمعلوماتك الخاصة. كل أمر يُنتج نتائج خاصة
بدورك وحساباتك ومنطقتك.

<Aside type="tip">
  للحصول على أفضل النتائج، أدرج عنوان بريدك الإلكتروني في الأوامر التي تحتاج
  إلى تحديد سجل المستخدم الخاص بك. يمكن أن تحتوي مؤسسات Salesforce على مستخدمين
  متعددين بأسماء متشابهة.
</Aside>

### البدء

بعد المصادقة، تحقق من أن كل شيء يعمل:

```
/salesforce:sf-status
```

ثم جرب استعلاماً بسيطاً باللغة الطبيعية:

```
list my authenticated salesforce orgs
```

**ما يمكن توقعه:** جدول يُظهر جميع المؤسسات المتصلة مع الاسم المستعار
واسم المستخدم وعنوان URL للمثيل وحالة الاتصال.

### ابحث عن حساباتك

اكتشف الحسابات المعيّن إليها:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**ما يمكن توقعه:** قائمة بالحسابات مجمّعة حسب مالك الحساب، تُظهر
دورك كعضو في الفريق في كل منها. إذا ظهرت معظم الأدوار فارغة، فهذه
ثغرة شائعة في جودة البيانات — عضويات فريق الحساب موجودة لكن الأدوار
لم تُملأ.

### مقارنة التغطية مع زميل

قارن تغطية حسابك مع زميل للعثور على الثغرات:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**ما يمكن توقعه:** جدول ملخص يُظهر عدد التداخلات وعدد الثغرات،
متبوعاً بقائمتين. إذا غيّرت أدوارك مؤخراً، فقد تجد صفراً من التداخلات
— هذا يكشف الحسابات التي تحتاج إلى إضافتك إليها.

<Aside type="note">
  يمكنك أيضاً التحقق من سلف عن طريق إضافة: "Also check if
  Previous Person Name appears on any of those accounts." يساعد هذا
  في تحديد ثغرات التسليم عند استبدال شخص ما في منطقة.
</Aside>

### نظرة عامة على خط أنابيب المنطقة

اطلع على جميع الفرص المفتوحة عبر حساباتك:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**ما يمكن توقعه:** ملخص مع إجمالي عدد الفرص وخط الأنابيب الخام
وخط الأنابيب الموزون. ثم جدول توزيع المراحل وقائمة بأفضل
الفرص حسب المبلغ. تُعلّم المخرجات مشكلات جودة البيانات مثل
تواريخ الإغلاق المتأخرة أو المبالغ الفارغة.

### التعمق في الفرصة

احصل على الصورة الكاملة لصفقة محددة:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**ما يمكن توقعه:** نظرة عامة على الفرصة (المرحلة والمبلغ وتاريخ الإغلاق
والاحتمالية وفئة التوقع والمالك)، وأعضاء فريق الفرصة مع الأدوار،
وجهات الاتصال المعلّمة، والمهام والفعاليات الأخيرة، وسجل تغيير الحقل
الذي يُظهر كيف تحرّك مبلغ الصفقة وتاريخ الإغلاق مع مرور الوقت.

### حالات الدعم عبر المنطقة

تحقق من حالات الدعم المفتوحة على حساباتك:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**ما يمكن توقعه:** عدد الحالات المفتوحة مع تفصيل الأولوية والحالة،
ثم تفاصيل الحالات مجمّعة حسب الحساب. تُعلّم الحالات القديمة (أشهر
أو سنوات بدون نشاط) كمرشحات لجودة البيانات.

### توقعات خط أنابيب الربع

احصل على عرض جاهز للتوقع من خط أنابيبك:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**ما يمكن توقعه:** جدول بصف واحد لكل ربع يُظهر عدد الفرص
وإجمالي خط الأنابيب وخط الأنابيب الموزون. يتضمن تفصيل مزيج المراحل
لكل ربع ويسلّط الضوء على أفضل الصفقات موزوناً. تُجمّع الفرص
المتأخرة بشكل منفصل.

### نظرة عامة كاملة على الحساب

تعمق في عميل محدد:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**ما يمكن توقعه:** ملف الشركة (الصناعة والإيرادات والموظفون
والموقع)، وجهات الاتصال الرئيسية مع المسميات الوظيفية والبريد الإلكتروني،
والفرص المفتوحة مع المراحل والمبالغ، وحالات الدعم الأخيرة مع الحالة.

### البحث عن الحالة

ابحث عن حالة دعم محددة برقمها:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**ما يمكن توقعه:** موضوع الحالة والحالة والأولوية والوصف
وملف حساب العميل ومالك الحالة مع معلومات الاتصال.

## دليل الاستخدام: استعلامات البيانات

للمستخدمين المتقدمين، يمكنك طلب استعلامات بأسلوب SOQL محددة باللغة
الطبيعية. يترجم الملحق طلبك إلى SOQL المناسب وينفذه.

### حقول محددة

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### تصفية التاريخ

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### التجميع

```
count all open salesforce cases grouped by priority and status across my accounts
```

### الكائنات المخصصة

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  إذا فشل استعلام ما لأن حقلاً غير موجود، يعيد الملحق
  المحاولة تلقائياً بدون ذلك الحقل. قد تستخدم الحقول المخصصة في مؤسستك
  أسماء مختلفة — تحقق من إعداد Salesforce إذا لم يُعثر
  على حقل ما.
</Aside>
