يدمج مكون **salesforce** الإضافي Salesforce CLI مع Claude Code،
مما يوفر مصادقة المؤسسة ووكيل CLI للأغراض العامة القادر على
تشغيل أي أمر `sf` بما في ذلك استعلامات SOQL. ويدعم بيئات الحاويات
بدون واجهة رسومية مع تدفقات مصادقة JWT وتوكن الوصول وعنوان SFDX URL.
يوضح دليل الاستخدام أدناه كيفية استخدام اللغة الطبيعية للاستعلام عن الحسابات
والفرص والحالات وجهات الاتصال من خلال وكيل CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## التثبيت

```
/plugin install salesforce@f5-sales-demo-marketplace
```

## المتطلبات الأساسية

- **Salesforce CLI** (`@salesforce/cli`)
- **مؤسسة Salesforce** مع وصول API
- حساب مستخدم يتمتع بصلاحيات الاستعلام عن الكائنات (Cases، Opportunities، Accounts، Contacts)

قم بتثبيت Salesforce CLI على محطة العمل الخاصة بك:

```bash
brew install sf
```

تحقق من التثبيت:

```bash
sf --version
```

يجب أن تشاهد مخرجات مثل `@salesforce/cli/2.x.x`.

## إعداد المصادقة

### الخطوة 1: العثور على نطاق Salesforce الخاص بك

انظر إلى عنوان URL الخاص بـ Salesforce في المتصفح. إذا كنت تصل إلى Salesforce عبر
`https://acme.lightning.force.com`، فإن نطاق تسجيل الدخول هو
`https://acme.my.salesforce.com`.

<Aside type="tip">
  إذا كانت شركتك تستخدم SSO، فيجب عليك تحديد نطاقك المخصص باستخدام
  `--instance-url` حتى يتم توجيه تسجيل الدخول عبر موفر SSO الخاص بك
  بدلاً من صفحة تسجيل الدخول العامة لـ Salesforce.
</Aside>

### الخطوة 2: المصادقة عبر المتصفح

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

استبدل `YOUR-DOMAIN` بنطاق شركتك من الخطوة 1. سيفتح متصفحك —
أكمل تدفق SSO أو تسجيل الدخول وفوّض التطبيق. يجب أن تشاهد:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### الخطوة 3: التحقق من الاتصال

```bash
sf org display --target-org my-org
```

تأكد من أن المخرجات تُظهر `Connected Status: Connected` مع اسم المستخدم
وعنوان URL للنسخة.

### الخطوة 4: مصادقة قابلة للنقل للحاويات (اختياري)

إذا كنت بحاجة إلى المصادقة في بيئة بدون واجهة رسومية (CI/CD، حاوية،
جلسة عمل عن بُعد)، فقم بتصدير عنوان SFDX URL للمصادقة من محطة العمل الخاصة بك:

```bash
sf org display --verbose --target-org my-org
```

انسخ قيمة **Sfdx Auth Url** (تبدأ بـ `force://`). في
الحاوية، شغّل:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  عنوان SFDX URL للمصادقة هو بيانات اعتماد — عامله كلمة مرور. لا تقم
  بإيداعه في الملفات أو لصقه في الدردشة. استخدم متغيرات البيئة
  وأنابيب stdin فقط.
</Aside>

### مرجع طرق المصادقة

| الطريقة | الأنسب لـ | المتطلبات |
|--------|----------|----------|
| تسجيل الدخول عبر الويب | محطات العمل المزودة بمتصفح | المتصفح + SSO |
| SFDX URL | الحاويات، CI/CD، المصادقة المحمولة | عنوان URL للمصادقة من جلسة مصادقة |
| JWT Bearer | خطوط الأنابيب الآلية | تطبيق متصل + مفتاح خاص + مفتاح المستهلك |
| توكن الوصول | مصادقة متغير البيئة | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**ملاحظة:** تدفق الجهاز (`sf org login device`) محظور منذ أغسطس
2025 وغير مدعوم.

### متغيرات البيئة

| المتغير | الغرض |
|----------|---------|
| `SF_ACCESS_TOKEN` | توكن Bearer لمصادقة توكن الوصول |
| `SFDX_AUTH_URL` | عنوان URL للمصادقة القسرية لمصادقة SFDX URL |
| `SF_ORG_INSTANCE_URL` | عنوان URL لنسخة المؤسسة |
| `SF_JWT_KEY_FILE` | المسار إلى المفتاح الخاص لـ JWT |
| `SF_CLIENT_ID` | مفتاح مستهلك التطبيق المتصل |
| `SF_USERNAME` | اسم مستخدم Salesforce لـ JWT |

## الأوامر

### /salesforce:sf-status

تحقق من حالة اتصال مؤسسة Salesforce الخاصة بك، وإصدار CLI،
والمؤسسات المصادق عليها.

```
/salesforce:sf-status
```

**ما يمكن توقعه:** جدول يعرض إصدار sf CLI، واسم مستعار المؤسسة المصادق عليها،
واسم المستخدم، وعنوان URL للنسخة، وحالة الاتصال، وإصدار API.

### /salesforce:sf-login

المصادقة على مؤسسة Salesforce. يمكن اختياريًا توفير اسم مستعار.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**ما يمكن توقعه:** إذا كانت مؤسسة بهذا الاسم المستعار مصادقًا عليها بالفعل،
فإنها تُبلّغ بتفاصيل الاتصال. وإلا، فإنها تتحقق من توفر
بيانات الاعتماد (متغيرات البيئة) وتصادق باستخدام الطريقة المستوفاة أولاً بالكامل.

## المهارات

### salesforce-index

موجّه النوايا الرئيسي. يتفعّل تلقائيًا عند ذكر
Salesforce أو إدارة المؤسسة أو استعلامات البيانات باللغة الطبيعية.

| ما تقوله | يوجّه إلى |
|---------|-----------|
| "login to salesforce"، "authenticate org" | مهارة `salesforce-auth` |
| "check org status"، "list orgs" | وكيل `cli-operator` |
| "show me opportunities"، "look up case" | وكيل `cli-operator` (استعلام SOQL) |
| "generate Apex class"، "create Flow" | مهارات تطوير afv-library |

### salesforce-auth

مهارة المصادقة المتكيفة مع الحاويات. تدعم أربع طرق مصادقة و
تكتشف تلقائيًا بيانات الاعتماد المتاحة. الأولوية:
توكن الوصول، ثم JWT، ثم SFDX URL، ثم تسجيل الدخول عبر الويب.

## الوكلاء

### cli-operator

وكيل Salesforce CLI المستقل الذي ينفذ أوامر `sf` مع ضمانات السلامة.
يتم تفويض جميع عمليات Salesforce إلى هذا الوكيل للحفاظ على
سياق الجلسة الرئيسية مختصرًا.

**قواعد الأمان:**

- للقراءة فقط بشكل افتراضي — تتطلب عمليات الكتابة موافقة صريحة
- لا ينشر أبدًا دون خطوة معاينة وتأكيد
- لا يعكس أبدًا توكنات الوصول أو عناوين URL للمصادقة أو بيانات الاعتماد في المخرجات
- يُطهّر جميع القيم التي يوفرها المستخدم (يرفض الأحرف الخاصة بالقشرة)
- يستخدم مخرجات `--json` للحصول على نتائج منظمة وقابلة للتحليل

**تنسيق الاستجابة:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## دليل الاستخدام: إدارة الحسابات

<Aside type="note">
  الطلبات أدناه ليست ميزات مُدمجة في المكون الإضافي. تعمل لأن
  وكيل cli-operator يمكنه تشغيل أي أمر `sf` CLI، بما في ذلك
  استعلامات SOQL التعسفية. تعتمد النتائج على نموذج بيانات مؤسستك
  وصلاحيات المستخدم الخاص بك. الطلبات هنا تم اختبارها والتحقق
  من عملها مع كائنات Salesforce القياسية (Account، Contact، Case،
  Opportunity، AccountTeamMember، OpportunityTeamMember).
</Aside>

تعمل هذه الطلبات مع أي مؤسسة Salesforce. استبدل القيم النائبة
بمعلوماتك الخاصة. تُنتج كل طلب نتائج خاصة
بدورك وحساباتك وإقليمك.

<Aside type="tip">
  للحصول على أفضل النتائج، قم بتضمين عنوان بريدك الإلكتروني في الطلبات التي تحتاج
  إلى التعرف على سجل المستخدم الخاص بك. يمكن أن تحتوي مؤسسات Salesforce على مستخدمين متعددين
  بأسماء متشابهة.
</Aside>

### البدء

بعد المصادقة، تحقق من أن كل شيء يعمل:

```
/salesforce:sf-status
```

ثم جرب استعلامًا بسيطًا باللغة الطبيعية:

```
list my authenticated salesforce orgs
```

**ما يمكن توقعه:** جدول يعرض جميع المؤسسات المتصلة مع الاسم المستعار،
واسم المستخدم، وعنوان URL للنسخة، وحالة الاتصال.

### العثور على حساباتك

اكتشف الحسابات التي تم تعيينك إليها:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**ما يمكن توقعه:** قائمة بالحسابات مجمّعة حسب مالك الحساب، تُظهر
دورك كعضو في الفريق في كل حساب. إذا ظهرت معظم الأدوار فارغة، فهذه
فجوة شائعة في نظافة البيانات — عضويات فريق الحساب موجودة لكن الأدوار
لم تُملأ.

### المقارنة المتقاطعة للتغطية مع زميل

قارن تغطية حسابك مع أحد زملائك لإيجاد الفجوات:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**ما يمكن توقعه:** جدول ملخص يُظهر عدد التداخلات وعدد الفجوات،
متبوعًا بقائمتين. إذا غيّرت دورك مؤخرًا، فقد تجد صفر تداخل —
هذا يكشف الحسابات التي تحتاج إلى إضافتك إليها.

<Aside type="note">
  يمكنك أيضًا التحقق من شخص سبقك في الدور بإضافة: "Also check if
  Previous Person Name appears on any of those accounts." يساعد هذا
  في تحديد فجوات التسليم عند استبدال شخص ما في إقليم.
</Aside>

### نظرة عامة على خط أنابيب الإقليم

اطّلع على جميع الفرص المفتوحة عبر حساباتك:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**ما يمكن توقعه:** ملخص بإجمالي عدد الفرص وخط الأنابيب الخام،
وخط الأنابيب المرجّح. ثم جدول توزيع المراحل وقائمة بأبرز
الفرص حسب المبلغ. تُبرز المخرجات مشكلات نظافة البيانات مثل
تواريخ الإغلاق المتجاوزة أو المبالغ الفارغة.

### التعمق في فرصة محددة

احصل على الصورة الكاملة لصفقة بعينها:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**ما يمكن توقعه:** نظرة عامة على الفرصة (المرحلة، المبلغ، تاريخ الإغلاق،
الاحتمالية، فئة التوقع، المالك)، وأعضاء فريق الفرصة مع الأدوار،
وجهات الاتصال المرتبطة، والمهام والأحداث الأخيرة، وسجل تغيير الحقول
الذي يُظهر كيف تحرّك مبلغ الصفقة وتاريخ الإغلاق بمرور الوقت.

### حالات الدعم عبر الإقليم

تحقق من حالات الدعم المفتوحة على حساباتك:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**ما يمكن توقعه:** عدد الحالات المفتوحة مع تفاصيل الأولوية والحالة،
ثم تفاصيل الحالات مجمّعة حسب الحساب. يتم الإشارة إلى الحالات القديمة
(أشهر أو سنوات دون نشاط) كمرشحات لنظافة البيانات.

### توقعات خط الأنابيب الفصلي

احصل على عرض جاهز للتوقعات لخط أنابيبك:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**ما يمكن توقعه:** جدول بصف واحد لكل ربع سنوي يعرض عدد الفرص،
وإجمالي خط الأنابيب، وخط الأنابيب المرجّح. يتضمن تفاصيل توزيع
المراحل لكل ربع ويُبرز الصفقات الأعلى وزنًا. يتم تجميع الفرص
المتجاوزة مواعيدها بشكل منفصل.

### نظرة عامة شاملة على الحساب

التعمق في عميل محدد:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**ما يمكن توقعه:** ملف الشركة (الصناعة، الإيرادات، الموظفون،
الموقع)، وجهات الاتصال الرئيسية مع المسميات والبريد الإلكتروني، والفرص المفتوحة
مع المراحل والمبالغ، وحالات الدعم الأخيرة مع الحالة.

### البحث عن حالة

ابحث عن حالة دعم محددة برقمها:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**ما يمكن توقعه:** موضوع الحالة، والحالة، والأولوية، والوصف، وملف حساب
العميل، ومالك الحالة مع معلومات الاتصال.

## دليل الاستخدام: استعلامات البيانات

للمستخدمين المتقدمين، يمكنك طلب استعلامات بأسلوب SOQL محدد باللغة
الطبيعية. يترجم المكون الإضافي طلبك إلى SOQL المناسب وينفذه.

### حقول محددة

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### تصفية بالتاريخ

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### التجميع

```
count all open salesforce cases grouped by priority and status across my accounts
```

### الكائنات المخصصة

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  إذا فشل استعلام ما لأن حقلاً غير موجود، يُعيد المكون الإضافي
  المحاولة تلقائيًا دون ذلك الحقل. قد تستخدم الحقول المخصصة في مؤسستك
  أسماء مختلفة — تحقق من إعداد Salesforce إذا لم يُعثر
  على حقل ما.
</Aside>

والنشر. ثبّتها بشكل منفصل:

```bash
npx skills add forcedotcom/afv-library
```
