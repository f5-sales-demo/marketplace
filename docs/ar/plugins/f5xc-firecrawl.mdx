يوفر إضافة **f5xc-firecrawl** كشط الويب المحلي المستضاف ذاتياً
عبر محرك [firecrawl](https://github.com/mendableai/firecrawl)
مفتوح المصدر. لا مفاتيح API، ولا اشتراكات، ولا اعتماد على السحابة. تعمل جميع
العمليات على مثيل firecrawl المحلي على `localhost:3002`
داخل حاوية التطوير.

<Badge text="v1.1.0" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-firecrawl@f5xc-salesdemos-marketplace
```

## الأوامر

### /scrape

كشط عنوان URL واحد واستخراج المحتوى بصيغة markdown.

```
/scrape https://docs.example.com/getting-started
/scrape https://example.com --format markdown,links --wait 2000
```

### /batch-scrape

كشط عناوين URL متعددة دفعةً واحدة.

```
/batch-scrape https://example.com https://example.org https://example.net
```

### /crawl

استعراض صفحات متعددة انطلاقاً من عنوان URL ابتدائي.

```
/crawl https://docs.example.com --limit 20 --depth 2
/crawl https://docs.example.com --include /api/* --exclude /blog/*
```

### /map

اكتشاف جميع عناوين URL في موقع ويب.

```
/map https://docs.example.com
/map https://docs.example.com --search api --subdomains
```

### /search

البحث في الويب واستخراج النتائج بشكل اختياري.

```
/search "firecrawl web scraping" --limit 10
/search "AI tools 2026" --scrape --time month
```

### /extract

استخراج بيانات منظمة من صفحات الويب بتقنية LLM.

```
/extract https://example.com "Extract the main heading and any links"
/extract https://example.com/pricing --schema '{"plans": [{"name": "string", "price": "string"}]}'
```

<Aside type="note">
  يتطلب أمر extract وكيل LLM متوافق مع OpenAI
  مُهيَّأ عبر `OPENAI_BASE_URL` و`OPENAI_API_KEY`.
</Aside>

### /llmstxt

إنشاء ملف llms.txt لموقع ما.

```
/llmstxt https://docs.example.com
```

## المهارات

### web-scraper

يُفعَّل تلقائياً عند طلب كشط عنوان URL، أو استعراض موقع ويب، أو رسم خريطة
لعناوين URL في الموقع، أو البحث في الويب، أو استخراج بيانات منظمة، أو إنشاء llms.txt،
أو كشط عناوين URL متعددة دفعةً واحدة، أو تحويل صفحة ويب إلى markdown.
يُفوَّض الأمر فوراً إلى عميل firecrawl-operator.

## العملاء

### firecrawl-operator

عميل كشط ويب مستقل ينفّذ تسلسلات `curl` + `jq`
على واجهة firecrawl API المحلية. يدعم 11 بروتوكولاً تغطي جميع
نقاط نهاية v1. عميل للقراءة فقط (بلا أدوات Write أو Edit أو Agent).

| البروتوكول | نقطة النهاية | النوع |
|----------|----------|------|
| HEALTH | `GET /` | Sync |
| SCRAPE | `POST /v1/scrape` | Sync |
| BATCH_SCRAPE | `POST /v1/batch/scrape` | Async |
| CRAWL | `POST /v1/crawl` | Async |
| CRAWL_CANCEL | `DELETE /v1/crawl/:id` | Sync |
| CRAWL_ACTIVE | `GET /v1/crawl/active` | Sync |
| CRAWL_ERRORS | `GET /v1/crawl/:id/errors` | Sync |
| MAP | `POST /v1/map` | Sync |
| SEARCH | `POST /v1/search` | Sync |
| EXTRACT | `POST /v1/extract` | Async |
| LLMSTXT | `POST /v1/llmstxt` | Async |

## البنية التحتية

تتطلب الإضافة تشغيل مجموعة firecrawl داخل حاوية التطوير:

| المكوّن | المنفذ | الغرض |
|-----------|------|---------|
| Firecrawl API | 3002 | جميع نقاط نهاية scrape/crawl/map/search/extract |
| Playwright | 3000 | محرك عرض JavaScript |
| Redis | 6379 | الواجهة الخلفية لقائمة انتظار المهام |
| PostgreSQL | socket | استمرارية مهام Crawl/batch |
| LiteLLM proxy | OPENAI_BASE_URL | الواجهة الخلفية لـ LLM للاستخراج (اختياري) |

تبدأ المجموعة تلقائياً عند تعيين `ENABLE_FIRECRAWL=true` (الإعداد
الافتراضي). يتحقق خطاف SessionStart من إمكانية الوصول إلى API
ويُصدر تحذيراً إذا كانت الخدمة معطّلة.

## الفروقات عن Firecrawl السحابي

تستخدم هذه الإضافة الإصدار مفتوح المصدر المستضاف ذاتياً:

- لا يلزم توثيق أو مفاتيح API لعملية الكشط
- لا حدود للاعتمادات ولا تقييد لمعدل الطلبات
- يستخدم نقاط نهاية API بالإصدار v1 (وليس v2)
- جلسات المتصفح والبحث المعمّق غير متاحَين
- يستخدم Extract وكيل LLM الخاص بك بدلاً من النماذج المستضافة
- يعمل بالكامل داخل شبكة الحاوية المحلية
