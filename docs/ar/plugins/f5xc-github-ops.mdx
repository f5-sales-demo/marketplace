يوفر مكوّن **f5xc-github-ops** أتمتةً حصريةً لعمليات GitHub
لمستودعات f5-sales-demo. يتولى إدارة دورة حياة Git وGitHub بالكامل
بدءاً من إنشاء Issue وحتى مراقبة ما بعد الدمج، مع تطبيق التطوير
القائم على Issues وبوابات الفحص قبل الإيداع.

<Badge text="v2.2.2" variant="note" /> <Badge text="الإنتاجية" variant="tip" />

## التثبيت

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## المهارات

### workflow-lifecycle

سير عمل حوكمة المستودع يتولى إنشاء Issues واصطلاحات تسمية الفروع
وإنشاء PR واستطلاع CI مع إرسال تغذية راجعة للأخطاء إلى Issues
ومراقبة ما بعد الدمج والتحقق ومعايير اكتمال المهام. يُفعَّل عند
قولك "commit" أو "push" أو "create a PR" أو "merge"، أو عند تنفيذ
أي عملية Git.

### upstream-contribution

سير عمل المساهمة في المصادر الأصلية للتطوير القائم على Fork. يتولى
اكتشاف Fork وبحث المصادر الأصلية (Issues وPRs وCONTRIBUTING.md
وأنماط CI)، وإنشاء Issues وPR عبر المستودعات، وإدارة Issues التتبع
المحلية، ومراقبة CI للمصادر الأصلية، والتوجيه نحو الحل.

## الوكلاء

### github-ops

وكيل حصري لجميع عمليات Git وGitHub الميكانيكية. يتولى بوابات
الفحص قبل الإيداع، وإنشاء Issues، وإنشاء الفروع، والتهيئة،
والإيداع، والرفع، وإنشاء PR، واستطلاع CI مع تغذية راجعة للأخطاء،
وإعادة المحاولة عند فشل البنية التحتية، ومراقبة ما بعد الدمج،
وتنظيف الفروع، وإدارة إعدادات المستودع عبر GitHub API.

يُعيد تقارير الحالة المنظَّمة: `COMPLETE` أو
`PRE_COMMIT_FAILED` أو `CI_FAILED` أو `BLOCKED` أو `FAILED`.

<Aside type="caution">
  هذا الوكيل هو المعالج **الحصري** لعمليات Git.
  لا تُشغّل `git commit` أو `git push` أو `gh pr create` مباشرةً
  في المستودعات التي تستخدم هذا المكوّن.
</Aside>
