يوفر إضافة **f5xc-github-ops** أتمتة حصرية لعمليات GitHub
لمستودعات f5xc-salesdemos. تتعامل مع دورة حياة Git وGitHub
الكاملة بدءاً من إنشاء المشكلات وصولاً إلى المراقبة بعد الدمج،
مع فرض التطوير القائم على المشكلات وبوابات التحقق من الصياغة
قبل الإيداع.

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## المهارات

### workflow-lifecycle

سير عمل حوكمة المستودع الذي يتعامل مع إنشاء المشكلات، واصطلاحات
تسمية الفروع، وإنشاء طلبات السحب، واستطلاع التكامل المستمر مع
تغذية راجعة للأخطاء في المشكلات، والمراقبة بعد الدمج، والتحقق،
ومعايير إتمام المهام. يُفعَّل عندما تقول "commit" أو "push" أو
"create a PR" أو "merge" أو تنفذ أي عملية Git.

### upstream-contribution

سير عمل المساهمة في المستودع الأصلي للتطوير القائم على النسخ
المتفرعة. يتعامل مع اكتشاف النسخ المتفرعة، والبحث في المستودع
الأصلي (المشكلات، وطلبات السحب، وCONTRIBUTING.md، وأنماط التكامل
المستمر)، وإنشاء المشكلات وطلبات السحب عبر المستودعات، وإدارة
مشكلات التتبع المحلية، ومراقبة التكامل المستمر للمستودع الأصلي،
والتوجيه نحو الحل.

## الوكلاء

### github-ops

وكيل حصري لجميع عمليات Git وGitHub الميكانيكية.
يتعامل مع بوابات التحقق من الصياغة قبل الإيداع، وإنشاء المشكلات،
وإنشاء الفروع، والتجهيز، والإيداع، والدفع، وإنشاء طلبات السحب،
واستطلاع التكامل المستمر مع تغذية راجعة للأخطاء، وإعادة محاولة
فشل البنية التحتية، والمراقبة بعد الدمج، وتنظيف الفروع، وإدارة
إعدادات المستودع عبر واجهة GitHub API.

يُرجع تقارير حالة منظمة: `COMPLETE` أو
`PRE_COMMIT_FAILED` أو `CI_FAILED` أو `BLOCKED` أو `FAILED`.

<Aside type="caution">
  هذا الوكيل هو المعالج **الحصري** لعمليات Git.
  لا تقم أبداً بتشغيل `git commit` أو `git push` أو `gh pr create` مباشرة
  في المستودعات التي تستخدم هذه الإضافة.
</Aside>
