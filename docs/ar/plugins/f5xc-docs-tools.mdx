يتحقق مكوّن **f5xc-docs-tools** من صحة ملفات محتوى MDX
لخط أنابيب توثيق f5-sales-demo. يرصد هذا المكوّن المشكلات الشائعة
التي تُعطّل البناء قبل وصولها إلى CI، بما فيها أحرف JSX المكشوفة،
والاستيرادات غير الصالحة، ومراجع الصور المعطوبة،
والبيانات الأمامية غير المكتملة.

<Badge text="v1.1.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## التثبيت

```
/plugin install f5xc-docs-tools@f5-sales-demo-marketplace
```

## المهارات

### mdx-content-reviewer

تُفعَّل هذه المهارة تلقائيًا عندما يكتشف Claude أنك تعمل
مع ملفات MDX في مستودع محتوى f5-sales-demo. وتُجري
سبعة فحوصات للتحقق من الصحة:

<CardGrid>
  <Card title="التحقق من البيانات الأمامية">
    يفحص الحقول المطلوبة مثل `title` و`sidebar.order`.
    يجب أن تتضمن صفحات Splash حقول `hero`
    و`template: splash`.
  </Card>
  <Card title="مزالق صياغة MDX">
    يكتشف أحرف `&lt;` المكشوفة والأقواس `\{` `\}`
    غير المُهرَّبة التي تُعطّل تحليل MDX.
  </Card>
  <Card title="التحقق من الاستيرادات">
    يتحقق من صحة الاستيرادات مقابل قائمة مسموح بها من
    مكونات Starlight المدمجة ومكونات سمة f5-sales-demo.
  </Card>
  <Card title="سمات المكونات">
    يفحص الخصائص المطلوبة للمكونات مثل Screenshot,
    Aside, Code, LinkCard, Card, وBadge.
  </Card>
  <Card title="مراجع الصور">
    يتحقق من وجود الصور المُشار إليها في مجلد `docs/images/`.
  </Card>
  <Card title="فحوصات البنية">
    يضمن وجود `docs/index.mdx` وخلوّ مجلدات الصور
    من ملفات MDX الدخيلة.
  </Card>
  <Card title="التصديرات وكتل الكود">
    يتحقق من أن المتغيرات المستخدمة في خاصية `code`
    للمكوّن `Code` لها تصديرات مقابلة.
  </Card>
</CardGrid>

#### سلوك التحديد النطاق

تُحدّد المهارة نطاقها بذكاء أثناء المراجعة:

- إذا كانت هناك تغييرات غير مُودَعة أو مرحلية على ملفات
  `docs/**/*.mdx`، تُراجَع تلك الملفات فقط
- وإلا، تُراجَع جميع ملفات `docs/**/*.mdx`

#### تنسيق المخرجات

تُجمَّع النتائج حسب درجة الخطورة:

- **ERROR** — سيُعطّل البناء؛ يجب الإصلاح قبل الدمج
- **WARNING** — مشكلات محتملة ينبغي معالجتها
- **INFO** — اقتراحات وأفضل الممارسات

تتضمن كل نتيجة مسار الملف، ورقم السطر، ووصفًا للمشكلة.

## الأوامر

### /review-mdx

```
/f5xc-docs-tools:review-mdx [path-or-glob]
```

يُشغّل مهارة `mdx-content-reviewer` عند الطلب.

**الوسائط:**

| الوسيط | مطلوب | الوصف |
|--------|-------|-------|
| `path-or-glob` | لا | مسار ملف أو نمط glob لتحديد نطاق المراجعة. الافتراضي هو جميع ملفات `docs/**/*.mdx`. |

**أمثلة:**

```
# مراجعة جميع ملفات MDX في docs/
/f5xc-docs-tools:review-mdx

# مراجعة ملف محدد
/f5xc-docs-tools:review-mdx docs/getting-started.mdx

# مراجعة الملفات المطابقة لنمط glob
/f5xc-docs-tools:review-mdx docs/guides/*.mdx
```

**المخرجات:** يُبلَّغ في نهاية المخرجات بسطر ملخص يعرض إجمالي
الملفات التي جرت مراجعتها وعدد النتائج حسب درجة الخطورة.

</Aside>

## مزالق MDX

المشكلات الشائعة التي يرصدها المكوّن:

| المزلق | المشكلة | الحل |
|--------|---------|------|
| `&lt;` المكشوف | يُفسّره MDX بوصفه وسم JSX | استخدم `&amp;lt;`، أو الكود المضمّن، أو أعد الصياغة |
| `\{` `\}` غير المُهرَّبة | يعامله MDX بوصفه تعبير JSX | استخدم الكود المضمّن، أو الهروب بـ `\`، أو كتلة الكود |
| الأقواس في أسماء الملفات | لا يستطيع Astro معالجة الملف | لا تستخدم `\{` أو `\}` في أسماء ملفات `.mdx` |
