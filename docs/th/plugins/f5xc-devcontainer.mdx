ปลั๊กอิน **f5xc-devcontainer** ให้ความสามารถในการรับรู้คอนเทนเนอร์สำหรับ
สภาพแวดล้อมการพัฒนา f5xc-salesdemos โดยจะดูแลรักษา
แคตตาล็อกเครื่องมือที่ครอบคลุม เปิดใช้งานการตรวจสอบคอนเทนเนอร์แบบเรียลไทม์
และจัดการการติดตั้งเครื่องมือพร้อมการติดตามความคงอยู่ผ่าน
GitHub issues

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## ทักษะ

### tool-catalog

ฐานความรู้ของเครื่องมือ CLI ทุกตัวที่ติดตั้งใน devcontainer
จัดเรียงตามหมวดหมู่ (คลาวด์ เครือข่าย ความปลอดภัย การพัฒนา
เป็นต้น) จะทำงานเมื่อคุณถามว่าควรใช้เครื่องมือใด วิธีใช้
เครื่องมือ มีบางอย่างติดตั้งอยู่หรือไม่ หรือต้องการความช่วยเหลือเกี่ยวกับ
การดำเนินการเฉพาะ

### self-awareness

ทักษะการระบุตัวตนและการวินิจฉัยตนเองของคอนเทนเนอร์ ให้การ
ตรวจสอบแบบเรียลไทม์ของคอนเทนเนอร์ที่กำลังทำงานผ่าน GitHub API และ
เมตาดาต้าของบิลด์ในเครื่อง จะทำงานสำหรับคำถามเกี่ยวกับตัวตน เวอร์ชัน ข้อมูลบิลด์
การตรวจสอบสุขภาพ และประวัติผู้มีส่วนร่วม

<Aside type="tip">
  ถามว่า "คุณคือใคร?" หรือ "ฉันกำลังใช้เวอร์ชันอะไร?" เพื่อเรียกใช้
  self-awareness
</Aside>

## เอเจนต์

### tool-advisor

อ่านแคตตาล็อกเครื่องมือและส่งคืนคำแนะนำพร้อมวัตถุประสงค์
คำสั่งเริ่มต้นอย่างรวดเร็ว และข้อกำหนดการยืนยันตัวตนสำหรับคำถาม
เกี่ยวกับเครื่องมือ CLI ใดๆ

### tool-auditor

ตรวจสอบ Dockerfile ของ devcontainer เทียบกับไฟล์อ้างอิงของแคตตาล็อกเครื่องมือ
เพื่อตรวจจับความคลาดเคลื่อน &mdash; เครื่องมือที่ถูกเพิ่มเข้าไปในอิมเมจ
แต่ไม่ได้อยู่ในแคตตาล็อก หรือรายการแคตตาล็อกสำหรับเครื่องมือที่ถูกลบออกไปแล้ว

### container-introspector

ดำเนินการตรวจสอบคอนเทนเนอร์แบบเรียลไทม์ผ่าน GitHub API และ
เมตาดาต้าในเครื่องสำหรับตัวตน สายพันธุ์ การวินิจฉัยตนเอง ประวัติ
ผู้มีส่วนร่วม และสายบิลด์ ดำเนินการแบบอ่านอย่างเดียวเท่านั้น

### container-maintainer

ติดตั้ง ลบ ค้นหา และอัปเดตเครื่องมือ CLI ในคอนเทนเนอร์ที่กำลังทำงาน
โดยใช้รูปแบบตัวจัดการแพ็กเกจที่ถูกต้อง สร้าง GitHub
issues ไปยังที่เก็บ devcontainer เพื่อ
ติดตามความคงอยู่
