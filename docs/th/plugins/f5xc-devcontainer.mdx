ปลั๊กอิน **f5xc-devcontainer** ให้การรับรู้คอนเทนเนอร์สำหรับ
สภาพแวดล้อมการพัฒนา f5xc-salesdemos โดยรักษาแคตตาล็อก
เครื่องมือที่ครอบคลุม เปิดใช้งานการตรวจสอบคอนเทนเนอร์แบบสดๆ
และจัดการการติดตั้งเครื่องมือพร้อมการติดตามความคงอยู่ผ่าน
GitHub issues

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## ทักษะ

### tool-catalog

ฐานความรู้ของเครื่องมือ CLI ทุกรายการที่ติดตั้งใน คอนเทนเนอร์สำหรับพัฒนา
จัดระเบียบตามหมวดหมู่ (คลาวด์, เครือข่าย, ความปลอดภัย, การพัฒนา
ฯลฯ) จะเปิดใช้งานเมื่อคุณถามว่าควรใช้เครื่องมือใด วิธีใช้เครื่องมือ
มีบางอย่างติดตั้งอยู่หรือไม่ หรือต้องการความช่วยเหลือเกี่ยวกับ
การดำเนินการเฉพาะ

### self-awareness

ทักษะด้านอัตลักษณ์คอนเทนเนอร์และการวินิจฉัยตนเอง ให้การตรวจสอบ
แบบสดๆ ของคอนเทนเนอร์ที่กำลังรันผ่าน GitHub API และข้อมูลเมตา
การบิลด์ในเครื่อง จะเปิดใช้งานสำหรับคำถามเกี่ยวกับอัตลักษณ์ เวอร์ชัน
ข้อมูลการบิลด์ การตรวจสุขภาพ และประวัติผู้ร่วมพัฒนา

<Aside type="tip">
  ถาม "who are you?" หรือ "what version am I running?" เพื่อเรียกใช้งาน
  self-awareness
</Aside>

## ตัวแทน

### tool-advisor

อ่านแคตตาล็อกเครื่องมือและส่งคืนคำแนะนำพร้อมวัตถุประสงค์
คำสั่งเริ่มต้นใช้งานอย่างรวดเร็ว และข้อกำหนดการพิสูจน์ตัวตน
สำหรับคำถามเกี่ยวกับเครื่องมือ CLI ทุกรายการ

### tool-auditor

ตรวจสอบ Dockerfile ของคอนเทนเนอร์สำหรับพัฒนากับไฟล์อ้างอิง
แคตตาล็อกเครื่องมือเพื่อตรวจจับความเบี่ยงเบน &mdash; เครื่องมือที่ถูก
เพิ่มลงในอิมเมจแต่ยังไม่ได้บันทึกในแคตตาล็อก หรือรายการแคตตาล็อก
สำหรับเครื่องมือที่ถูกลบออกไปแล้ว

### container-introspector

ดำเนินการตรวจสอบคอนเทนเนอร์แบบสดๆ ผ่าน GitHub API และ
ข้อมูลเมตาในเครื่องสำหรับอัตลักษณ์ ลำดับวงศ์ตระกูล การวินิจฉัยตนเอง
ประวัติผู้ร่วมพัฒนา และสายการบิลด์ เฉพาะการดำเนินการแบบอ่านอย่างเดียวเท่านั้น

### container-maintainer

ติดตั้ง ลบ ค้นหา และอัปเดตเครื่องมือ CLI ในคอนเทนเนอร์ที่กำลังรัน
โดยใช้รูปแบบตัวจัดการแพ็กเกจที่ถูกต้อง ยื่น GitHub issues กับ
ที่เก็บข้อมูลคอนเทนเนอร์สำหรับพัฒนาเพื่อการติดตามความคงอยู่
