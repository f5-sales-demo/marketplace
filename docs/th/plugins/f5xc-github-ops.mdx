ปลั๊กอิน **f5xc-github-ops** ให้บริการการทำงานอัตโนมัติของ GitHub operations
เฉพาะสำหรับ repositories ของ f5xc-salesdemos โดยจัดการวงจรชีวิตทั้งหมดของ
Git และ GitHub ตั้งแต่การสร้าง issue ไปจนถึงการตรวจสอบหลัง merge
โดยบังคับใช้การพัฒนาแบบ issue-driven และ pre-commit
lint gates

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-github-ops@f5xc-salesdemos-marketplace
```

## ทักษะ

### workflow-lifecycle

เวิร์กโฟลว์การกำกับดูแล repository ที่จัดการการสร้าง issue, ข้อตกลง
การตั้งชื่อ branch, การสร้าง PR, การ polling CI พร้อมการตอบกลับข้อผิดพลาด
ไปยัง issues, การตรวจสอบหลัง merge, การยืนยัน และเกณฑ์
การเสร็จสิ้นงาน เปิดใช้งานเมื่อคุณพูดว่า "commit", "push",
"create a PR", "merge" หรือดำเนินการ Git ใดๆ

### upstream-contribution

เวิร์กโฟลว์การมีส่วนร่วมกับ upstream สำหรับการพัฒนาแบบ fork-based จัดการ
การตรวจจับ fork, การค้นคว้า upstream (issues, PRs, CONTRIBUTING.md,
รูปแบบ CI), การสร้าง issue และ PR ข้ามรีโป, การจัดการ
tracking issue ในเครื่อง, การตรวจสอบ CI ของ upstream และการแก้ไขแบบมีคำแนะนำ

## เอเจนต์

### github-ops

เอเจนต์เฉพาะสำหรับการดำเนินการทางกลไกทั้งหมดของ Git และ GitHub
จัดการ pre-commit lint gates, การสร้าง issue, การสร้าง branch,
การ staging, การ commit, การ push, การสร้าง PR, การ polling CI พร้อมการตอบกลับ
ข้อผิดพลาด, การลองใหม่เมื่อโครงสร้างพื้นฐานล้มเหลว, การตรวจสอบหลัง merge,
การล้าง branch และการจัดการการตั้งค่า repository ผ่าน
GitHub API

ส่งคืนรายงานสถานะแบบมีโครงสร้าง: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` หรือ `FAILED`

<Aside type="caution">
  เอเจนต์นี้เป็นตัวจัดการ**เฉพาะ**สำหรับการดำเนินการ Git
  อย่ารัน `git commit`, `git push` หรือ `gh pr create` โดยตรง
  ใน repositories ที่ใช้ปลั๊กอินนี้
</Aside>
