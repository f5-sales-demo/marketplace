ปลั๊กอิน **f5xc-github-ops** ให้การทำงานอัตโนมัติสำหรับ GitHub operations โดยเฉพาะสำหรับ repositories ของ f5-sales-demo รองรับวงจรชีวิตของ Git และ GitHub ทั้งหมดตั้งแต่การสร้าง issue จนถึงการตรวจสอบหลัง merge พร้อมบังคับใช้การพัฒนาแบบ issue-driven และ lint gates ก่อน commit

<Badge text="v2.2.2" variant="note" /> <Badge text="ประสิทธิภาพการทำงาน" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## ทักษะ

### workflow-lifecycle

การทำงานของ workflow สำหรับการกำกับดูแล repository ครอบคลุมการสร้าง issue, รูปแบบการตั้งชื่อ branch, การสร้าง PR, การ polling CI พร้อมส่ง feedback ข้อผิดพลาดกลับไปยัง issue, การตรวจสอบหลัง merge, การยืนยัน และเกณฑ์การเสร็จสิ้นงาน เปิดใช้งานเมื่อคุณพูดว่า "commit", "push", "create a PR", "merge", หรือดำเนินการ Git ใดๆ

### upstream-contribution

workflow สำหรับการมีส่วนร่วมกับ upstream สำหรับการพัฒนาแบบ fork-based รองรับการตรวจจับ fork, การค้นคว้า upstream (issues, PRs, CONTRIBUTING.md, รูปแบบ CI), การสร้าง issue และ PR ข้ามที่เก็บ, การจัดการ tracking issue ภายใน, การตรวจสอบ upstream CI, และการแก้ไขแบบมีคำแนะนำ

## Agent

### github-ops

Agent เฉพาะสำหรับการดำเนินการ Git และ GitHub ทั้งหมด รองรับ lint gates ก่อน commit, การสร้าง issue, การสร้าง branch, การ staging, การ commit, การ push, การสร้าง PR, การ polling CI พร้อม feedback ข้อผิดพลาด, การลองใหม่เมื่อโครงสร้างพื้นฐานล้มเหลว, การตรวจสอบหลัง merge, การลบ branch, และการจัดการการตั้งค่า repository ผ่าน GitHub API

ส่งคืนรายงานสถานะแบบมีโครงสร้าง: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED`, หรือ `FAILED`

<Aside type="caution">
  Agent นี้เป็น**ผู้จัดการเฉพาะ**สำหรับการดำเนินการ Git ห้ามรัน `git commit`, `git push`, หรือ `gh pr create` โดยตรงใน repositories ที่ใช้ปลั๊กอินนี้
</Aside>
