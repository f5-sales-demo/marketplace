ปลั๊กอิน **salesforce** ผสานรวม Salesforce CLI เข้ากับ Claude Code
โดยมอบการพิสูจน์ตัวตนองค์กรและตัวแทน CLI อเนกประสงค์ที่สามารถ
รันคำสั่ง `sf` ใดๆ รวมถึงการสืบค้น SOQL รองรับสภาพแวดล้อมคอนเทนเนอร์แบบ headless ด้วย JWT, access-token และ
ขั้นตอนการพิสูจน์ตัวตน SFDX URL คู่มือการใช้งานด้านล่างแสดงวิธีการใช้ภาษาธรรมชาติเพื่อสืบค้นบัญชี โอกาสทางการขาย เคส และผู้ติดต่อผ่านตัวแทน CLI

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## การติดตั้ง

```
/plugin install salesforce@f5-sales-demo-marketplace
```

## ข้อกำหนดเบื้องต้น

- **Salesforce CLI** (`@salesforce/cli`)
- **องค์กร Salesforce** ที่มีสิทธิ์เข้าถึง API
- บัญชีผู้ใช้ที่มีสิทธิ์สืบค้นออบเจ็กต์ (Cases, Opportunities, Accounts, Contacts)

ติดตั้ง Salesforce CLI บนเวิร์กสเตชันของคุณ:

```bash
brew install sf
```

ตรวจสอบการติดตั้ง:

```bash
sf --version
```

คุณควรเห็นผลลัพธ์เช่น `@salesforce/cli/2.x.x`

## การตั้งค่าการพิสูจน์ตัวตน

### ขั้นตอนที่ 1: ค้นหาโดเมน Salesforce ของคุณ

ดู URL ของ Salesforce ในเบราว์เซอร์ หากคุณเข้าถึง Salesforce ที่
`https://acme.lightning.force.com` โดเมนสำหรับเข้าสู่ระบบของคุณคือ
`https://acme.my.salesforce.com`

<Aside type="tip">
  หากบริษัทของคุณใช้ SSO คุณต้องระบุโดเมนที่กำหนดเองด้วย
  `--instance-url` เพื่อให้การเข้าสู่ระบบเปลี่ยนเส้นทางผ่านผู้ให้บริการ SSO ของคุณ
  แทนหน้าเข้าสู่ระบบ Salesforce ทั่วไป
</Aside>

### ขั้นตอนที่ 2: พิสูจน์ตัวตนผ่านเบราว์เซอร์

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

แทนที่ `YOUR-DOMAIN` ด้วยโดเมนของบริษัทจากขั้นตอนที่ 1 เบราว์เซอร์ของคุณจะเปิดขึ้น — ดำเนินการขั้นตอน SSO หรือการเข้าสู่ระบบและอนุญาตแอป คุณควรเห็น:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### ขั้นตอนที่ 3: ตรวจสอบการเชื่อมต่อ

```bash
sf org display --target-org my-org
```

ยืนยันว่าผลลัพธ์แสดง `Connected Status: Connected` พร้อมชื่อผู้ใช้
และ URL ของ instance

### ขั้นตอนที่ 4: การพิสูจน์ตัวตนแบบพกพาสำหรับคอนเทนเนอร์ (ไม่บังคับ)

หากคุณต้องการพิสูจน์ตัวตนในสภาพแวดล้อมแบบ headless (CI/CD, คอนเทนเนอร์,
เซสชันระยะไกล) ให้ส่งออก SFDX auth URL จากเวิร์กสเตชันของคุณ:

```bash
sf org display --verbose --target-org my-org
```

คัดลอกค่า **Sfdx Auth Url** (ขึ้นต้นด้วย `force://`) ในคอนเทนเนอร์ให้รัน:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX auth URL คือข้อมูลรับรองตัวตน — ให้จัดการเหมือนรหัสผ่าน อย่า
  บันทึกลงในไฟล์หรือวางในแชท ใช้ตัวแปรสภาพแวดล้อมและ stdin pipes เท่านั้น
</Aside>

### ข้อมูลอ้างอิงวิธีการพิสูจน์ตัวตน

| วิธีการ | เหมาะสำหรับ | ต้องการ |
|--------|----------|----------|
| Web Login | เวิร์กสเตชันที่มีเบราว์เซอร์ | เบราว์เซอร์ + SSO |
| SFDX URL | คอนเทนเนอร์, CI/CD, การพิสูจน์ตัวตนแบบพกพา | Auth URL จากเซสชันที่พิสูจน์ตัวตนแล้ว |
| JWT Bearer | ไปป์ไลน์อัตโนมัติ | Connected App + private key + consumer key |
| Access Token | การพิสูจน์ตัวตนด้วยตัวแปรสภาพแวดล้อม | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**หมายเหตุ:** Device flow (`sf org login device`) ถูกบล็อกตั้งแต่เดือนสิงหาคม
2025 และไม่รองรับ

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | วัตถุประสงค์ |
|----------|---------|
| `SF_ACCESS_TOKEN` | Bearer token สำหรับการพิสูจน์ตัวตนแบบ access-token |
| `SFDX_AUTH_URL` | Force auth URL สำหรับการพิสูจน์ตัวตนแบบ SFDX URL |
| `SF_ORG_INSTANCE_URL` | URL ของ instance องค์กร |
| `SF_JWT_KEY_FILE` | เส้นทางไปยัง JWT private key |
| `SF_CLIENT_ID` | Consumer key ของ Connected App |
| `SF_USERNAME` | ชื่อผู้ใช้ Salesforce สำหรับ JWT |

## คำสั่ง

### /salesforce:sf-status

ตรวจสอบสถานะการเชื่อมต่อองค์กร Salesforce เวอร์ชัน CLI และ
องค์กรที่พิสูจน์ตัวตนแล้ว

```
/salesforce:sf-status
```

**สิ่งที่คาดหวัง:** ตารางแสดงเวอร์ชัน sf CLI ของคุณ, alias องค์กรที่พิสูจน์ตัวตนแล้ว, ชื่อผู้ใช้, URL ของ instance, สถานะการเชื่อมต่อ และเวอร์ชัน API

### /salesforce:sf-login

พิสูจน์ตัวตนกับองค์กร Salesforce สามารถระบุ alias ได้ตามต้องการ

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**สิ่งที่คาดหวัง:** หากองค์กรที่มี alias นั้นได้รับการพิสูจน์ตัวตนแล้ว
จะรายงานรายละเอียดการเชื่อมต่อ หากไม่มี จะตรวจสอบข้อมูลรับรองตัวตนที่มีอยู่
(ตัวแปรสภาพแวดล้อม) และพิสูจน์ตัวตนโดยใช้วิธีแรกที่พร้อมใช้งานครบถ้วน

## ทักษะ

### salesforce-index

ตัวกำหนดเส้นทางความตั้งใจระดับบนสุด เปิดใช้งานอัตโนมัติเมื่อคุณกล่าวถึง
Salesforce, การจัดการองค์กร หรือการสืบค้นข้อมูลด้วยภาษาธรรมชาติ

| คุณพูด | กำหนดเส้นทางไปยัง |
|---------|-----------|
| "login to salesforce", "authenticate org" | ทักษะ `salesforce-auth` |
| "check org status", "list orgs" | ตัวแทน `cli-operator` |
| "show me opportunities", "look up case" | ตัวแทน `cli-operator` (SOQL query) |
| "generate Apex class", "create Flow" | ทักษะการพัฒนา afv-library |

### salesforce-auth

ทักษะการพิสูจน์ตัวตนที่ปรับให้เข้ากับคอนเทนเนอร์ รองรับสี่วิธีการพิสูจน์ตัวตนและ
ตรวจจับข้อมูลรับรองตัวตนที่มีอยู่โดยอัตโนมัติ ให้ความสำคัญตามลำดับ:
access-token, JWT, SFDX URL และ web login

## ตัวแทน

### cli-operator

ตัวแทน Salesforce CLI อัตโนมัติที่รันคำสั่ง `sf` พร้อมระบบป้องกัน
ความปลอดภัย การดำเนินการ Salesforce ทั้งหมดถูกมอบหมายให้ตัวแทนนี้เพื่อ
ให้บริบทของเซสชันหลักกระชับ

**กฎความปลอดภัย:**

- อ่านอย่างเดียวโดยค่าเริ่มต้น — การดำเนินการเขียนต้องได้รับการอนุมัติอย่างชัดเจน
- ไม่ deploy โดยไม่มีขั้นตอนการแสดงตัวอย่างและการยืนยัน
- ไม่แสดง access token, auth URL หรือข้อมูลรับรองตัวตนในผลลัพธ์
- ทำความสะอาดค่าทั้งหมดที่ผู้ใช้ระบุ (ปฏิเสธอักขระพิเศษ shell)
- ใช้ผลลัพธ์ `--json` สำหรับผลลัพธ์ที่มีโครงสร้างและแยกวิเคราะห์ได้

**รูปแบบการตอบสนอง:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## คู่มือการใช้งาน: การจัดการบัญชี

<Aside type="note">
  พรอมต์ด้านล่างไม่ใช่ฟีเจอร์ที่ฮาร์ดโค้ดไว้ในปลั๊กอิน พรอมต์เหล่านี้ทำงานได้
  เพราะตัวแทน cli-operator สามารถรันคำสั่ง `sf` CLI ใดๆ รวมถึง
  การสืบค้น SOQL ตามอำเภอใจ ผลลัพธ์ขึ้นอยู่กับโมเดลข้อมูลขององค์กรของคุณและ
  สิทธิ์ของผู้ใช้ พรอมต์เหล่านี้ได้รับการทดสอบและยืนยันแล้วว่าทำงานได้กับ
  ออบเจ็กต์ Salesforce มาตรฐาน (Account, Contact, Case, Opportunity,
  AccountTeamMember, OpportunityTeamMember)
</Aside>

พรอมต์เหล่านี้ทำงานกับองค์กร Salesforce ใดๆ แทนที่ค่าตัวแทน
ด้วยข้อมูลของคุณเอง พรอมต์แต่ละรายการให้ผลลัพธ์ที่เฉพาะเจาะจง
กับบทบาท บัญชี และพื้นที่ของคุณ

<Aside type="tip">
  เพื่อผลลัพธ์ที่ดีที่สุด ให้ระบุที่อยู่อีเมลของคุณในพรอมต์ที่ต้องการ
  ระบุบันทึกผู้ใช้ของคุณ องค์กร Salesforce อาจมีผู้ใช้หลายคน
  ที่มีชื่อคล้ายกัน
</Aside>

### เริ่มต้นใช้งาน

หลังจากพิสูจน์ตัวตนแล้ว ตรวจสอบให้แน่ใจว่าทุกอย่างทำงานได้:

```
/salesforce:sf-status
```

จากนั้นลองใช้การสืบค้นด้วยภาษาธรรมชาติอย่างง่าย:

```
list my authenticated salesforce orgs
```

**สิ่งที่คาดหวัง:** ตารางแสดงองค์กรที่เชื่อมต่อทั้งหมดพร้อม alias,
ชื่อผู้ใช้, URL ของ instance และสถานะการเชื่อมต่อ

### ค้นหาบัญชีของคุณ

ค้นหาบัญชีที่คุณได้รับมอบหมาย:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**สิ่งที่คาดหวัง:** รายการบัญชีจัดกลุ่มตามเจ้าของบัญชี แสดง
บทบาทสมาชิกในทีมของคุณในแต่ละรายการ หากบทบาทส่วนใหญ่แสดงว่าว่างเปล่า นั่นเป็น
ช่องว่างด้านความสะอาดของข้อมูลที่พบได้บ่อย — การเป็นสมาชิกในทีมบัญชีมีอยู่แต่บทบาท
ไม่ได้ถูกกรอก

### อ้างอิงข้ามการครอบคลุมกับเพื่อนร่วมงาน

เปรียบเทียบการครอบคลุมบัญชีของคุณกับสมาชิกในทีมเพื่อค้นหาช่องว่าง:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**สิ่งที่คาดหวัง:** ตารางสรุปแสดงจำนวนที่ทับซ้อนและจำนวนช่องว่าง
ตามด้วยสองรายการ หากคุณเพิ่งเปลี่ยนบทบาท คุณอาจพบว่าไม่มีการทับซ้อนเลย
ซึ่งเผยให้เห็นบัญชีที่คุณต้องได้รับการเพิ่มเข้าไป

<Aside type="note">
  คุณยังสามารถตรวจสอบผู้ดำรงตำแหน่งก่อนหน้าได้โดยเพิ่ม: "Also check if
  Previous Person Name appears on any of those accounts." ซึ่งช่วย
  ระบุช่องว่างการส่งมอบเมื่อแทนที่บุคคลในพื้นที่
</Aside>

### ภาพรวมไปป์ไลน์ตามพื้นที่

ดูโอกาสทางการขายที่เปิดอยู่ทั้งหมดในบัญชีของคุณ:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**สิ่งที่คาดหวัง:** สรุปพร้อมจำนวนโอกาสทางการขายทั้งหมด, ไปป์ไลน์ดิบ
และไปป์ไลน์ถ่วงน้ำหนัก จากนั้นตารางการกระจายตามขั้นตอนและรายการโอกาสทางการขาย
ยอดนิยมตามจำนวนเงิน ผลลัพธ์จะแจ้งเตือนปัญหาความสะอาดของข้อมูล เช่น
วันปิดที่เลยกำหนดหรือจำนวนเงินที่เป็น null

### การเจาะลึกโอกาสทางการขาย

รับภาพรวมครบถ้วนของดีลเฉพาะ:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**สิ่งที่คาดหวัง:** ภาพรวมโอกาสทางการขาย (ขั้นตอน, จำนวนเงิน, วันปิด,
ความน่าจะเป็น, หมวดหมู่การพยากรณ์, เจ้าของ), สมาชิกในทีมโอกาสทางการขายพร้อมบทบาท,
ผู้ติดต่อที่ถูกแท็ก, งานและกิจกรรมล่าสุด และประวัติการเปลี่ยนแปลงฟิลด์
แสดงการเปลี่ยนแปลงจำนวนเงินและวันปิดของดีลตามเวลา

### เคสสนับสนุนทั่วทั้งพื้นที่

ตรวจสอบเคสสนับสนุนที่เปิดอยู่ในบัญชีของคุณ:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**สิ่งที่คาดหวัง:** จำนวนเคสที่เปิดอยู่พร้อมการแบ่งตามลำดับความสำคัญและสถานะ
จากนั้นรายละเอียดเคสจัดกลุ่มตามบัญชี เคสที่ค้างนาน (เป็นเดือนหรือเป็นปีโดยไม่มีกิจกรรม) จะถูกแจ้งเตือนว่าเป็นตัวเลือกในการทำความสะอาดข้อมูล

### การพยากรณ์ไปป์ไลน์รายไตรมาส

รับมุมมองไปป์ไลน์ที่พร้อมสำหรับการพยากรณ์:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**สิ่งที่คาดหวัง:** ตารางที่มีหนึ่งแถวต่อไตรมาสแสดงจำนวนโอกาสทางการขาย,
ไปป์ไลน์รวมทั้งหมด และไปป์ไลน์ถ่วงน้ำหนัก รวมถึงการแบ่งส่วนผสมของขั้นตอน
ต่อไตรมาสและเน้นดีลที่มีน้ำหนักมากที่สุด โอกาสทางการขายที่เลยกำหนดจะถูกจัดกลุ่มแยกต่างหาก

### ภาพรวมบัญชีแบบครบถ้วน

เจาะลึกลูกค้าเฉพาะราย:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**สิ่งที่คาดหวัง:** โปรไฟล์บริษัท (อุตสาหกรรม, รายได้, จำนวนพนักงาน,
ที่ตั้ง), ผู้ติดต่อหลักพร้อมตำแหน่งและอีเมล, โอกาสทางการขายที่เปิดอยู่พร้อมขั้นตอน
และจำนวนเงิน และเคสสนับสนุนล่าสุดพร้อมสถานะ

### การค้นหาเคส

ค้นหาเคสสนับสนุนเฉพาะตามหมายเลข:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**สิ่งที่คาดหวัง:** หัวข้อเคส, สถานะ, ลำดับความสำคัญ, คำอธิบาย, โปรไฟล์บัญชีลูกค้า
และเจ้าของเคสพร้อมข้อมูลติดต่อ

## คู่มือการใช้งาน: การสืบค้นข้อมูล

สำหรับผู้ใช้ขั้นสูง คุณสามารถขอการสืบค้นแบบ SOQL ในภาษาธรรมชาติได้
ปลั๊กอินจะแปลคำขอของคุณเป็น SOQL ที่เหมาะสมและรันการสืบค้น

### ฟิลด์เฉพาะ

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### การกรองตามวันที่

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### การรวมข้อมูล

```
count all open salesforce cases grouped by priority and status across my accounts
```

### ออบเจ็กต์ที่กำหนดเอง

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  หากการสืบค้นล้มเหลวเพราะฟิลด์ไม่มีอยู่ ปลั๊กอินจะลองใหม่อีกครั้งโดยไม่มีฟิลด์นั้น
  ฟิลด์ที่กำหนดเองในองค์กรของคุณอาจใช้ชื่อแตกต่างกัน — ตรวจสอบการตั้งค่า
  Salesforce หากไม่พบฟิลด์
</Aside>

## ทักษะการพัฒนา

ทักษะการพัฒนา Salesforce 30 รายการจาก
[forcedotcom/afv-library](https://github.com/forcedotcom/afv-library)
เปิดใช้งานอัตโนมัติสำหรับ Apex, Flow, LWC, SOQL, metadata, Agentforce
และงาน deployment ติดตั้งแยกต่างหาก:

```bash
npx skills add forcedotcom/afv-library
```

| หัวข้อ | ทักษะ |
|-------|-------|
| Apex classes และ services | `generating-apex` |
| Apex tests | `generating-apex-test` |
| Flows | `generating-flow` |
| LWC และ UI bundles | `building-ui-bundle-app` |
| Custom objects | `generating-custom-object` |
| Custom fields | `generating-custom-field` |
| Validation rules | `generating-validation-rule` |
| Permission sets | `generating-permission-set` |
| FlexiPages | `generating-flexipage` |
| Agentforce agents | `developing-agentforce` |
| Agentforce testing | `testing-agentforce` |
| Deployment | `deploying-ui-bundle` |
| SLDS2 migration | `uplifting-components-to-slds2` |
| Trigger refactoring | `trigger-refactor-pipeline` |
