ปลั๊กอิน **salesforce** ผสานรวม Salesforce CLI เข้ากับ Claude Code
โดยให้ความสามารถในการยืนยันตัวตนกับ org และ CLI agent สำหรับใช้งานทั่วไปที่สามารถ
รันคำสั่ง `sf` ใดก็ได้ รวมถึง SOQL queries ปลั๊กอินนี้รองรับ
สภาพแวดล้อมแบบ headless container ด้วยการยืนยันตัวตนแบบ JWT, access-token และ SFDX URL
คู่มือการใช้งานด้านล่างสาธิตวิธีการใช้
ภาษาธรรมชาติเพื่อสอบถามข้อมูล accounts, opportunities, cases และ contacts
ผ่าน CLI agent

## การติดตั้ง

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## ข้อกำหนดเบื้องต้น

- **Salesforce CLI** (`@salesforce/cli`)
- **Salesforce org** ที่มีสิทธิ์เข้าถึง API
- บัญชีผู้ใช้ที่มีสิทธิ์ในการสอบถามอ็อบเจกต์ (Cases, Opportunities, Accounts, Contacts)

ติดตั้ง Salesforce CLI บนเครื่องของคุณ:

```bash
brew install sf
```

ตรวจสอบการติดตั้ง:

```bash
sf --version
```

คุณควรเห็นผลลัพธ์เช่น `@salesforce/cli/2.x.x`

## การตั้งค่าการยืนยันตัวตน

### ขั้นตอนที่ 1: ค้นหาโดเมน Salesforce ของคุณ

ดู URL ของ Salesforce ในเบราว์เซอร์ หากคุณเข้าถึง Salesforce ที่
`https://acme.lightning.force.com` โดเมนสำหรับล็อกอินของคุณคือ
`https://acme.my.salesforce.com`

<Aside type="tip">
  หากบริษัทของคุณใช้ SSO คุณต้องระบุโดเมนที่กำหนดเองด้วย
  `--instance-url` เพื่อให้การล็อกอินเปลี่ยนเส้นทางผ่าน SSO provider ของคุณ
  แทนที่จะเป็นหน้าล็อกอิน Salesforce ทั่วไป
</Aside>

### ขั้นตอนที่ 2: ยืนยันตัวตนผ่านเบราว์เซอร์

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

แทนที่ `YOUR-DOMAIN` ด้วยโดเมนของบริษัทคุณจากขั้นตอนที่ 1 เบราว์เซอร์ของคุณ
จะเปิดขึ้น — ดำเนินการขั้นตอน SSO หรือล็อกอินให้เสร็จสิ้นและอนุญาตแอป คุณควร
เห็น:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### ขั้นตอนที่ 3: ตรวจสอบการเชื่อมต่อ

```bash
sf org display --target-org my-org
```

ยืนยันว่าผลลัพธ์แสดง `Connected Status: Connected` พร้อมชื่อผู้ใช้
และ instance URL ของคุณ

### ขั้นตอนที่ 4: การยืนยันตัวตนแบบพกพาสำหรับ containers (ไม่บังคับ)

หากคุณต้องการยืนยันตัวตนในสภาพแวดล้อมแบบ headless (CI/CD, container,
remote session) ให้ส่งออก SFDX auth URL จากเครื่องของคุณ:

```bash
sf org display --verbose --target-org my-org
```

คัดลอกค่า **Sfdx Auth Url** (เริ่มต้นด้วย `force://`) ใน
container ให้รัน:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX auth URL เป็นข้อมูลประจำตัว — ปฏิบัติต่อมันเหมือนรหัสผ่าน อย่า
  บันทึกลงในไฟล์หรือวางในแชท ใช้ตัวแปรสภาพแวดล้อม
  และ stdin pipes เท่านั้น
</Aside>

### ตารางอ้างอิงวิธีการยืนยันตัวตน

| วิธีการ | เหมาะสำหรับ | ต้องการ |
|--------|----------|----------|
| Web Login | เครื่องที่มีเบราว์เซอร์ | เบราว์เซอร์ + SSO |
| SFDX URL | Containers, CI/CD, การยืนยันตัวตนแบบพกพา | Auth URL จากเซสชันที่ยืนยันตัวตนแล้ว |
| JWT Bearer | ไปป์ไลน์อัตโนมัติ | Connected App + private key + consumer key |
| Access Token | การยืนยันตัวตนผ่านตัวแปรสภาพแวดล้อม | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**หมายเหตุ:** Device flow (`sf org login device`) ถูกบล็อกตั้งแต่เดือนสิงหาคม
2025 และไม่รองรับ

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | วัตถุประสงค์ |
|----------|---------|
| `SF_ACCESS_TOKEN` | Bearer token สำหรับการยืนยันตัวตนแบบ access-token |
| `SFDX_AUTH_URL` | Force auth URL สำหรับการยืนยันตัวตนแบบ SFDX URL |
| `SF_ORG_INSTANCE_URL` | URL ของ org instance |
| `SF_JWT_KEY_FILE` | เส้นทางไปยัง JWT private key |
| `SF_CLIENT_ID` | Consumer key ของ Connected App |
| `SF_USERNAME` | ชื่อผู้ใช้ Salesforce สำหรับ JWT |

## คำสั่ง

### /salesforce:sf-status

ตรวจสอบสถานะการเชื่อมต่อ Salesforce org, เวอร์ชัน CLI และ
org ที่ยืนยันตัวตนแล้ว

```
/salesforce:sf-status
```

**สิ่งที่คาดหวัง:** ตารางแสดงเวอร์ชัน sf CLI, alias ของ org ที่ยืนยันตัวตนแล้ว,
ชื่อผู้ใช้, instance URL, สถานะการเชื่อมต่อ และเวอร์ชัน API

### /salesforce:sf-login

ยืนยันตัวตนกับ Salesforce org สามารถระบุ alias ได้ตามต้องการ

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**สิ่งที่คาดหวัง:** หาก org ที่มี alias นั้นยืนยันตัวตนแล้ว
จะรายงานรายละเอียดการเชื่อมต่อ หากไม่ได้ยืนยันตัวตน จะตรวจสอบข้อมูลประจำตัว
ที่มีอยู่ (ตัวแปรสภาพแวดล้อม) และยืนยันตัวตนโดยใช้วิธีที่ตรงตามเงื่อนไขเป็นอันดับแรก

## ทักษะ

### salesforce-index

ตัวกำหนดเส้นทาง intent ระดับบนสุด เปิดใช้งานอัตโนมัติเมื่อคุณกล่าวถึง
Salesforce, การจัดการ org หรือการสอบถามข้อมูลด้วยภาษาธรรมชาติ

| คุณพูดว่า | กำหนดเส้นทางไปยัง |
|---------|-----------|
| "login to salesforce", "authenticate org" | ทักษะ `salesforce-auth` |
| "check org status", "list orgs" | agent `cli-operator` |
| "show me opportunities", "look up case" | agent `cli-operator` (SOQL query) |
| "generate Apex class", "create Flow" | ทักษะการพัฒนา afv-library |

### salesforce-auth

ทักษะการยืนยันตัวตนที่ปรับให้เหมาะกับ container รองรับวิธีการยืนยันตัวตนสี่แบบและ
ตรวจจับอัตโนมัติว่ามีข้อมูลประจำตัวใดบ้าง จัดลำดับความสำคัญ:
access-token, จากนั้น JWT, จากนั้น SFDX URL, จากนั้น web login

## Agents

### cli-operator

Agent Salesforce CLI อัตโนมัติที่รันคำสั่ง `sf` ด้วยกลไกความปลอดภัย
การดำเนินการ Salesforce ทั้งหมดถูกมอบหมายให้ agent นี้เพื่อ
รักษาบริบทของเซสชันหลักให้กระชับ

**กฎความปลอดภัย:**

- อ่านอย่างเดียวเป็นค่าเริ่มต้น — การดำเนินการเขียนต้องได้รับการอนุมัติอย่างชัดเจน
- ไม่ deploy โดยไม่มีการแสดงตัวอย่างและขั้นตอนยืนยัน
- ไม่แสดง access tokens, auth URLs หรือข้อมูลประจำตัวในผลลัพธ์
- ทำความสะอาดค่าที่ผู้ใช้ป้อนทั้งหมด (ปฏิเสธ shell metacharacters)
- ใช้ผลลัพธ์ `--json` สำหรับผลลัพธ์ที่มีโครงสร้างและแยกวิเคราะห์ได้

**รูปแบบการตอบกลับ:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## คู่มือการใช้งาน: การจัดการบัญชี

<Aside type="note">
  คำสั่งด้านล่างไม่ใช่ฟีเจอร์ที่เขียนไว้ตายตัวของปลั๊กอิน มันทำงานได้
  เพราะ cli-operator agent สามารถรันคำสั่ง `sf` CLI ใดก็ได้ รวมถึง
  SOQL queries ตามต้องการ ผลลัพธ์ขึ้นอยู่กับ data model ของ org และ
  สิทธิ์ของผู้ใช้ของคุณ คำสั่งที่แสดงที่นี่ได้รับการทดสอบและยืนยันแล้วว่า
  ใช้งานได้กับอ็อบเจกต์ Salesforce มาตรฐาน (Account, Contact, Case,
  Opportunity, AccountTeamMember, OpportunityTeamMember)
</Aside>

คำสั่งเหล่านี้ใช้งานได้กับ Salesforce org ใดก็ได้ แทนที่ค่า placeholder
ด้วยข้อมูลของคุณเอง แต่ละคำสั่งจะให้ผลลัพธ์เฉพาะเจาะจง
ตามบทบาท, บัญชี และเขตพื้นที่ของคุณ

<Aside type="tip">
  เพื่อผลลัพธ์ที่ดีที่สุด ให้รวมที่อยู่อีเมลของคุณในคำสั่งที่ต้อง
  ระบุตัวตนของเรคคอร์ดผู้ใช้ของคุณ Salesforce org อาจมีผู้ใช้หลายคน
  ที่มีชื่อคล้ายกัน
</Aside>

### เริ่มต้นใช้งาน

หลังจากยืนยันตัวตนแล้ว ตรวจสอบว่าทุกอย่างทำงานได้:

```
/salesforce:sf-status
```

จากนั้นลองสอบถามด้วยภาษาธรรมชาติง่ายๆ:

```
list my authenticated salesforce orgs
```

**สิ่งที่คาดหวัง:** ตารางแสดง org ที่เชื่อมต่อทั้งหมดพร้อม alias,
ชื่อผู้ใช้, instance URL และสถานะการเชื่อมต่อ

### ค้นหาบัญชีของคุณ

ค้นหาว่าคุณถูกกำหนดให้อยู่ในบัญชีใดบ้าง:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**สิ่งที่คาดหวัง:** รายการบัญชีจัดกลุ่มตาม account owner แสดง
บทบาทสมาชิกทีมของคุณในแต่ละบัญชี หากบทบาทส่วนใหญ่แสดงเป็นว่าง นั่นเป็น
ช่องว่างด้านคุณภาพข้อมูลที่พบได้ทั่วไป — สมาชิกทีมบัญชีมีอยู่แต่ไม่ได้กำหนดบทบาท

### ตรวจสอบความครอบคลุมกับเพื่อนร่วมงาน

เปรียบเทียบความครอบคลุมบัญชีของคุณกับเพื่อนร่วมทีมเพื่อหาช่องว่าง:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**สิ่งที่คาดหวัง:** ตารางสรุปแสดงจำนวนที่ทับซ้อนและจำนวนช่องว่าง
ตามด้วยสองรายการ หากคุณเพิ่งเปลี่ยนบทบาท คุณอาจพบว่าไม่มี
ความทับซ้อนเลย — สิ่งนี้เผยให้เห็นว่าคุณต้องถูกเพิ่มเข้าไปในบัญชีใดบ้าง

<Aside type="note">
  คุณยังสามารถตรวจสอบผู้ดำรงตำแหน่งก่อนหน้าโดยเพิ่ม: "Also check if
  Previous Person Name appears on any of those accounts." ซึ่งจะช่วย
  ระบุช่องว่างในการส่งมอบงานเมื่อเข้ามาแทนที่ใครบางคนในเขตพื้นที่
</Aside>

### ภาพรวมไปป์ไลน์ของเขตพื้นที่

ดู opportunities ที่เปิดอยู่ทั้งหมดในบัญชีของคุณ:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**สิ่งที่คาดหวัง:** สรุปจำนวน opportunity ทั้งหมด, ไปป์ไลน์ดิบ
และไปป์ไลน์ถ่วงน้ำหนัก จากนั้นตารางการกระจายตาม stage และรายการ
opportunity อันดับต้นตามจำนวนเงิน ผลลัพธ์จะแจ้งปัญหาคุณภาพข้อมูลเช่น
วันปิดที่เลยกำหนดหรือจำนวนเงินเป็น null

### การเจาะลึก Opportunity

ดูภาพรวมทั้งหมดของดีลเฉพาะ:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**สิ่งที่คาดหวัง:** ภาพรวม opportunity (stage, จำนวนเงิน, วันปิด,
ความน่าจะเป็น, forecast category, เจ้าของ), สมาชิกทีม opportunity
พร้อมบทบาท, contacts ที่เชื่อมโยง, tasks และ events ล่าสุด และ
ประวัติการเปลี่ยนแปลงฟิลด์แสดงว่าจำนวนเงินและวันปิดของดีลเปลี่ยนแปลงอย่างไรเมื่อเวลาผ่านไป

### เคสสนับสนุนทั่วเขตพื้นที่

ตรวจสอบเคสสนับสนุนที่เปิดอยู่ในบัญชีของคุณ:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**สิ่งที่คาดหวัง:** จำนวนเคสที่เปิดอยู่พร้อมการแจกแจงตามลำดับความสำคัญและสถานะ
จากนั้นรายละเอียดเคสจัดกลุ่มตามบัญชี เคสที่ไม่มีความเคลื่อนไหว (เป็นเดือนหรือ
ปีที่ไม่มีกิจกรรม) จะถูกแจ้งว่าเป็นผู้สมัครสำหรับการทำความสะอาดข้อมูล

### การพยากรณ์ไปป์ไลน์รายไตรมาส

ดูมุมมองไปป์ไลน์ที่พร้อมสำหรับการพยากรณ์:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**สิ่งที่คาดหวัง:** ตารางที่มีหนึ่งแถวต่อไตรมาสแสดงจำนวน
opportunity, ไปป์ไลน์รวม และไปป์ไลน์ถ่วงน้ำหนัก รวมถึงการแจกแจง stage mix
ต่อไตรมาสและเน้นดีลถ่วงน้ำหนักสูงสุด Opportunity ที่เลยกำหนดจะถูก
จัดกลุ่มแยกต่างหาก

### ภาพรวมบัญชีแบบครบถ้วน

เจาะลึกลูกค้าเฉพาะราย:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**สิ่งที่คาดหวัง:** โปรไฟล์บริษัท (อุตสาหกรรม, รายได้, จำนวนพนักงาน,
ที่ตั้ง), ผู้ติดต่อหลักพร้อมตำแหน่งและอีเมล, opportunity ที่เปิดอยู่พร้อม
stages และจำนวนเงิน และเคสสนับสนุนล่าสุดพร้อมสถานะ

### ค้นหาเคส

ค้นหาเคสสนับสนุนเฉพาะตามหมายเลข:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**สิ่งที่คาดหวัง:** หัวข้อเคส, สถานะ, ลำดับความสำคัญ, คำอธิบาย,
โปรไฟล์บัญชีลูกค้า และเจ้าของเคสพร้อมข้อมูลติดต่อ

## คู่มือการใช้งาน: การสอบถามข้อมูล

สำหรับผู้ใช้ขั้นสูง คุณสามารถขอ query แบบ SOQL เฉพาะเจาะจงด้วย
ภาษาธรรมชาติ ปลั๊กอินจะแปลคำขอของคุณเป็น SOQL ที่เหมาะสม
และรันมัน

### ฟิลด์เฉพาะ

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### การกรองตามวันที่

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### การรวมข้อมูล

```
count all open salesforce cases grouped by priority and status across my accounts
```

### อ็อบเจกต์ที่กำหนดเอง

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  หาก query ล้มเหลวเนื่องจากฟิลด์ไม่มีอยู่ ปลั๊กอินจะ
  ลองใหม่โดยอัตโนมัติโดยไม่มีฟิลด์นั้น ฟิลด์ที่กำหนดเองใน org ของคุณ
  อาจใช้ชื่อที่แตกต่างกัน — ตรวจสอบ Salesforce Setup ของคุณหากไม่พบฟิลด์
</Aside>

## ทักษะการพัฒนา

ทักษะการพัฒนา Salesforce 30 รายการจาก
[forcedotcom/afv-library](https://github.com/forcedotcom/afv-library)
จะเปิดใช้งานอัตโนมัติสำหรับ Apex, Flow, LWC, SOQL, metadata, Agentforce
และงาน deployment ติดตั้งแยกต่างหาก:

```bash
npx skills add forcedotcom/afv-library
```

| หัวข้อ | ทักษะ |
|-------|-------|
| Apex classes และ services | `generating-apex` |
| Apex tests | `generating-apex-test` |
| Flows | `generating-flow` |
| LWC และ UI bundles | `building-ui-bundle-app` |
| Custom objects | `generating-custom-object` |
| Custom fields | `generating-custom-field` |
| Validation rules | `generating-validation-rule` |
| Permission sets | `generating-permission-set` |
| FlexiPages | `generating-flexipage` |
| Agentforce agents | `developing-agentforce` |
| Agentforce testing | `testing-agentforce` |
| Deployment | `deploying-ui-bundle` |
| SLDS2 migration | `uplifting-components-to-slds2` |
| Trigger refactoring | `trigger-refactor-pipeline` |
