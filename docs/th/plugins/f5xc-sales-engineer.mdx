ปลั๊กอิน **f5xc-sales-engineer** มอบกรอบงาน Sales Engineer persona ที่ใช้ซ้ำได้
สำหรับที่เก็บสาธิต F5 XC โดยจัดการวงจรชีวิตการประชุมสาธิตทั้งหมด —
ตั้งแต่การตรวจสอบก่อนการประชุมไปจนถึงการดำเนินการสด การตอบคำถาม และการรื้อถอน

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## ทักษะ

### demo-executor

การดำเนินการสาธิตที่ขับเคลื่อนด้วย API พร้อมวงจรชีวิตการประชุมสี่ขั้นตอน:

1. **เตรียมการ** — การตรวจสอบสภาพแวดล้อมก่อนการประชุมผ่านตัวแทน
   `demo-housekeeping`
2. **ดำเนินการ** — การดำเนินการตามขั้นตอนแบบกำหนดค่าตายตัวพร้อมการบรรยายที่มุ่งสู่ลูกค้า
3. **ตอบคำถาม** — โหมดอิสระพร้อมสภาพแวดล้อมสด
4. **รื้อถอน** — การทำความสะอาดหลังการประชุมผ่านตัวแทน `demo-housekeeping`

อ่านเนื้อหาเฉพาะผลิตภัณฑ์จาก `DEMO_PRODUCT_EXPERTISE.md` และคำสั่งสาธิต
จาก `docs/demo/`

### presenter

การนำเสนอแบบ as-built โดยใช้สภาพแวดล้อมสาธิตที่กำหนดค่าไว้ล่วงหน้า
ปฏิบัติตามรูปแบบ แสดง → บรรยาย → เชื่อมโยง → หยุดชั่วคราว ในแต่ละขั้นตอน
อ่านลำดับการนำเสนอจาก `DEMO_WALKTHROUGH_CONFIG.md`

### subject-matter-expert

การตอบคำถามที่มีการอ้างอิงจากแหล่งข้อมูลอย่างเข้มงวด ไม่มีการเดาเด็ดขาด —
ทุกคำตอบมีการอ้างอิงหรือหลักฐานประกอบ มอบหมายการค้นคว้าให้ตัวแทน
`demo-researcher` เมื่อจำเป็น

### sales-engineer

ดัชนีบทบาทที่นำทางเจตนาของผู้ใช้ไปยังทักษะ persona ที่ถูกต้อง
เปิดใช้งานเมื่อคำขอเกี่ยวข้องกับการสาธิตแต่ไม่ตรงกับวลีทริกเกอร์เฉพาะ

## ตัวแทน

### demo-housekeeping

ตัวแทนอัตโนมัติสำหรับการตรวจสอบก่อนการประชุมและการรื้อถอนหลังการประชุม
รันเมทริกซ์การตรวจสอบความพร้อม (T0–T5) ที่กำหนดไว้ใน `DEMO_READINESS_MATRIX.md`
และดำเนินการคำสั่งรื้อถอนจาก `docs/demo/phase-4-teardown.mdx`

### demo-researcher

บรรณารักษ์วิจัยแบบอ่านอย่างเดียวที่ค้นหาคำตอบที่ตรวจสอบแล้วพร้อมการอ้างอิง
ค้นหาใน `docs/` ในเครื่องก่อน จากนั้นค้นหาแหล่งข้อมูลภายนอกที่จัดทำดัชนีจาก
`DEMO_SOURCE_INDEX.md` โดยมี WebSearch เป็นตัวสำรอง

## ไฟล์แบบแผน

<Aside type="tip" title="เนื้อหาเฉพาะผลิตภัณฑ์">
  ปลั๊กอินนี้ไม่ขึ้นกับผลิตภัณฑ์ใดผลิตภัณฑ์หนึ่ง แต่ละที่เก็บมอบ
  เนื้อหาเฉพาะผลิตภัณฑ์ผ่านไฟล์แบบแผนมาตรฐาน
</Aside>

สร้างไฟล์เหล่านี้ในรากของที่เก็บเพื่อกำหนดค่าปลั๊กอิน:

| ไฟล์ | วัตถุประสงค์ | จำเป็นสำหรับ |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | ความสามารถของผลิตภัณฑ์ สัญญาณการตรวจจับ ความครอบคลุมของภัยคุกคาม การจัดแนวการปฏิบัติตาม การอ้างอิง API | ทุกทักษะ |
| `DEMO_WALKTHROUGH_CONFIG.md` | URL แอปสาธิต ลำดับการนำเสนอ เวลาการตรวจจับ | presenter |
| `DEMO_SOURCE_INDEX.md` | แคตตาล็อกแหล่งข้อมูลการวิจัยพร้อมเอกสารในเครื่องและ URL ภายนอก | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | ตัวแปรที่จำเป็น/ไม่บังคับ การตรวจสอบความพร้อม (T0–T5) จุดปลาย API | demo-housekeeping |
| `docs/demo/` | ไฟล์ขั้นตอนพร้อมคำสั่ง cURL และเกตหลักฐาน | demo-executor |

## สไตล์การบรรยาย

ทุก persona ใช้สไตล์การบรรยายที่สอดคล้องกัน:

- **กาลปัจจุบัน บุรุษที่หนึ่งพหูพจน์** — "สิ่งที่เรากำลังมองอยู่ที่นี่…"
- **หนึ่งประเด็นต่อย่อหน้า** — การบรรยายแต่ละส่วนตอบคำถาม: นี่คืออะไร? ทำไมมันสำคัญ? ฉันควรทำอะไรกับมัน?
- **ระบุสัญญาณ** — ชี้ให้เห็นว่าความสามารถใดกำลังทำงานอยู่
- **เชื่อมโยงการปฏิบัติตามเมื่อเกี่ยวข้อง** — กล่าวถึงการจัดแนวกฎระเบียบอย่างเป็นธรรมชาติ
