ปลั๊กอิน **f5xc-docs-tools** ตรวจสอบความถูกต้องของไฟล์เนื้อหา MDX สำหรับ
pipeline เอกสาร f5xc-salesdemos โดยจะตรวจจับปัญหาทั่วไปที่ทำให้
การ build ล้มเหลวก่อนที่จะถึง CI รวมถึงอักขระ JSX ที่ไม่ถูกห่อหุ้ม
การ import ที่ไม่ถูกต้อง การอ้างอิงรูปภาพที่เสียหาย และ
frontmatter ที่ไม่สมบูรณ์

<Badge text="v1.1.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## การติดตั้ง

```
/plugin install f5xc-docs-tools@f5xc-salesdemos-marketplace
```

## ทักษะ

### mdx-content-reviewer

ทักษะนี้จะเปิดใช้งานโดยอัตโนมัติเมื่อ Claude ตรวจพบว่าคุณกำลัง
ทำงานกับไฟล์ MDX ใน repository เนื้อหา f5xc-salesdemos
โดยจะดำเนินการตรวจสอบ 7 รายการ:

<CardGrid>

#### พฤติกรรมการกำหนดขอบเขต

ทักษะนี้จะกำหนดขอบเขตการตรวจสอบอย่างชาญฉลาด:

- หากมีการเปลี่ยนแปลงที่ยังไม่ได้ commit หรือ staged ในไฟล์ `docs/**/*.mdx`
  จะตรวจสอบเฉพาะไฟล์เหล่านั้นเท่านั้น
- มิฉะนั้น จะตรวจสอบไฟล์ `docs/**/*.mdx` ทั้งหมด

#### รูปแบบผลลัพธ์

ผลการตรวจพบจะถูกจัดกลุ่มตามความรุนแรง:

- **ERROR** — จะทำให้การ build ล้มเหลว; ต้องแก้ไขก่อนการ merge
- **WARNING** — ปัญหาที่น่าจะเกิดขึ้นซึ่งควรได้รับการแก้ไข
- **INFO** — คำแนะนำและแนวปฏิบัติที่ดี

แต่ละผลการตรวจพบจะระบุเส้นทางไฟล์ หมายเลขบรรทัด และ
คำอธิบายของปัญหา

## คำสั่ง

### /review-mdx

```
/f5xc-docs-tools:review-mdx [path-or-glob]
```

เรียกใช้ทักษะ `mdx-content-reviewer` ตามต้องการ

**อาร์กิวเมนต์:**

| อาร์กิวเมนต์ | จำเป็น | คำอธิบาย |
|----------|----------|-------------|
| `path-or-glob` | ไม่ | เส้นทางไฟล์หรือ glob pattern เพื่อกำหนดขอบเขตการตรวจสอบ ค่าเริ่มต้นคือไฟล์ `docs/**/*.mdx` ทั้งหมด |

**ตัวอย่าง:**

```
# ตรวจสอบไฟล์ MDX ทั้งหมดใน docs/
/f5xc-docs-tools:review-mdx

# ตรวจสอบไฟล์ที่ระบุ
/f5xc-docs-tools:review-mdx docs/getting-started.mdx

# ตรวจสอบไฟล์ที่ตรงกับ glob
/f5xc-docs-tools:review-mdx docs/guides/*.mdx
```

**ผลลัพธ์:** บรรทัดสรุปท้ายจะรายงานจำนวนไฟล์ทั้งหมด
ที่ตรวจสอบแล้วและจำนวนผลการตรวจพบตามความรุนแรง

## Import ที่อนุญาต

ปลั๊กอินจะตรวจสอบ import เทียบกับแหล่งเหล่านี้:

**คอมโพเนนต์ในตัวของ Starlight** (`@astrojs/starlight/components`):

`Aside` `Badge` `Card` `CardGrid` `Code` `FileTree` `Icon`
`LinkCard` `Steps` `TabItem` `Tabs`

**คอมโพเนนต์ธีม f5xc-salesdemos** (`@f5xc-salesdemos/docs-theme/components/`):

`Banner` `Icon` `LinkCard` `Screenshot`

<Aside type="caution">
  คอมโพเนนต์ธีมใช้ default import จากไฟล์ `.astro` แต่ละไฟล์
  การ import จาก `node_modules` โดยตรง, relative paths และ
  การเรียก `require()` จะถูกแจ้งเตือนเป็น warning
</Aside>

## ปัญหาที่พบบ่อยใน MDX

ปัญหาทั่วไปที่ปลั๊กอินตรวจจับได้:

| ปัญหา | สาเหตุ | วิธีแก้ไข |
|---------|---------|-----|
| `&lt;` ที่ไม่ถูกห่อหุ้ม | MDX ตีความว่าเป็น JSX tag | ใช้ `&amp;lt;`, inline code หรือเขียนใหม่ |
| `\{` `\}` ที่ไม่ได้ escape | MDX ถือว่าเป็น JSX expression | ใช้ inline code, escape ด้วย `\` หรือใช้ code block |
| วงเล็บปีกกาในชื่อไฟล์ | Astro ไม่สามารถประมวลผลไฟล์ได้ | อย่าใช้ `\{` หรือ `\}` ในชื่อไฟล์ `.mdx` |
