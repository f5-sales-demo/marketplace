## สคีมา marketplace.json

ไฟล์ manifest ของตลาดกลางอยู่ที่
`.claude-plugin/marketplace.json` ในไดเรกทอรีรากของ repository

### ฟิลด์ระดับบนสุด

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `$schema` | ไม่ | URL ของ JSON Schema สำหรับการตรวจสอบ |
| `name` | ใช่ | ตัวระบุตลาดกลางที่ไม่ซ้ำกัน |
| `version` | ไม่ | เวอร์ชันสคีมาตลาดกลาง (semver) |
| `metadata.description` | ใช่ | คำอธิบายตลาดกลางที่มนุษย์อ่านได้ |
| `owner.name` | ใช่ | องค์กรหรือผู้ใช้ที่เป็นเจ้าของตลาดกลาง |
| `owner.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของเจ้าของ |
| `owner.email` | ไม่ | อีเมลติดต่อ |
| `plugins` | ใช่ | อาร์เรย์ของรายการปลั๊กอิน |

### ฟิลด์รายการปลั๊กอิน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน (ต้องตรงกับ `plugin.json`) |
| `description` | ใช่ | คำอธิบายสั้นของปลั๊กอิน |
| `version` | ใช่ | สตริงเวอร์ชัน Semantic |
| `author.name` | ใช่ | ผู้เขียนปลั๊กอิน |
| `source` | ใช่ | พาธหรือ URL ไปยังปลั๊กอิน (ดูประเภทแหล่งที่มา) |
| `category` | ใช่ | หมวดหมู่ปลั๊กอิน (เช่น `productivity`) |
| `homepage` | ไม่ | URL ไปยังเอกสารหรือหน้าหลักของปลั๊กอิน |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `keywords` | ไม่ | อาร์เรย์ของคำค้นหา |
| `tags` | ไม่ | อาร์เรย์ของแท็กสำหรับการกรอง |
| `repository` | ไม่ | URL ของ repository |

## ประเภทแหล่งที่มาของปลั๊กอิน

ฟิลด์ `source` ใน marketplace.json รองรับหลายรูปแบบ:

| ประเภท | ตัวอย่าง | คำอธิบาย |
|------|---------|-------------|
| พาธสัมพัทธ์ | `./plugins/example-plugin` | ปลั๊กอินใน repository เดียวกัน |
| GitHub shorthand | `owner/repo` | ปลั๊กอินที่รากของ repository |
| GitHub พร้อมพาธ | `owner/repo/path/to/plugin` | ปลั๊กอินในไดเรกทอรีย่อย |
| Git URL | `https://github.com/owner/repo.git` | Git repository |
| npm package | `npm:@scope/package` | npm package ที่เผยแพร่แล้ว |

## สคีมา plugin.json

แต่ละปลั๊กอินมีไฟล์ manifest ที่
`.claude-plugin/plugin.json` ภายในไดเรกทอรีของมัน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน |
| `description` | ใช่ | สิ่งที่ปลั๊กอินทำ |
| `version` | ใช่ | Semantic version |
| `author.name` | ใช่ | ผู้เขียนปลั๊กอิน |
| `author.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของผู้เขียน |
| `homepage` | ไม่ | URL เอกสาร |
| `keywords` | ไม่ | คำค้นหา |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `repository` | ไม่ | URL ของ repository |

## โครงสร้างไดเรกทอรีปลั๊กอิน

```
plugins/example-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## ตัวแปรสภาพแวดล้อม

ปลั๊กอินสามารถใช้ตัวแปรเหล่านี้ในไฟล์ทักษะและคำสั่ง:

| ตัวแปร | คำอธิบาย |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | พาธสัมบูรณ์ไปยังไดเรกทอรีรากของปลั๊กอิน |

## การตรวจสอบความถูกต้อง

ตรวจสอบความถูกต้องของ manifest ตลาดกลางและปลั๊กอินในเครื่อง:

```bash
claude plugin validate .
```

การตรวจสอบนี้ครอบคลุม:

- การปฏิบัติตามสคีมา `marketplace.json`
- ไดเรกทอรีปลั๊กอินที่อ้างถึงทั้งหมดมีอยู่จริง
- ปลั๊กอินแต่ละตัวมี `plugin.json` ที่ถูกต้อง
- ทักษะมี frontmatter ของ `SKILL.md` ที่จัดรูปแบบอย่างถูกต้อง
- คำสั่งมีฟิลด์ frontmatter ที่จำเป็น

## การกำหนดค่าทีม

### การกำหนดค่าตลาดกลางล่วงหน้า

เพิ่มลงใน `.claude/settings.json` ใน repository ใดก็ได้:

```json
{
  "extraKnownMarketplaces": [
    "f5-sales-demo/marketplace"
  ]
}
```

### การเปิดใช้งานปลั๊กอินล่วงหน้า

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5-sales-demo-marketplace"
  ]
}
```

<Aside type="tip">
  การ commit การตั้งค่าเหล่านี้ลงใน repository ของคุณจะทำให้
  สมาชิกในทีมทุกคนได้รับการกำหนดค่าตลาดกลางและปลั๊กอิน
  เดียวกันโดยอัตโนมัติ
</Aside>

## การแก้ไขปัญหา

### ไม่พบตลาดกลาง

ตรวจสอบว่าได้เพิ่มตลาดกลางแล้ว:

```
/plugin marketplace list
```

หากไม่พบ ให้เพิ่มใหม่:

```
/plugin marketplace add f5-sales-demo/marketplace
```

### ปลั๊กอินไม่เปิดใช้งาน

- ยืนยันว่าติดตั้งปลั๊กอินแล้ว:
  `/plugin list`
- ตรวจสอบว่าเวอร์ชัน Claude Code ของคุณรองรับปลั๊กอิน
  (v1.0.33+)
- ตรวจสอบว่าเงื่อนไขทริกเกอร์ของทักษะตรงกับบริบทปัจจุบันของคุณ

### ทักษะไม่ทริกเกอร์โดยอัตโนมัติ

ทักษะจะเปิดใช้งานตามฟิลด์ `description` ใน frontmatter ของ SKILL.md หากทักษะไม่ทริกเกอร์ตามที่คาดไว้:

- ตรวจสอบว่าคำอธิบายอธิบายบริบททริกเกอร์ได้อย่างถูกต้อง
- ใช้คำสั่งโดยตรง (เช่น `/f5xc-docs-tools:review-mdx`)
  เป็นวิธีแก้ปัญหาชั่วคราว

### ข้อผิดพลาด Build หลังการตรวจสอบ

ปลั๊กอินรายงานปัญหาแต่ไม่แก้ไขอัตโนมัติ ให้ใช้การแก้ไขที่แนะนำด้วยตนเอง จากนั้นรันการตรวจสอบอีกครั้งเพื่อยืนยันการแก้ไข
