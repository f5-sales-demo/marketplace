## สคีมา marketplace.json

ไฟล์ marketplace manifest อยู่ที่
`.claude-plugin/marketplace.json` ในรูทของ repository

### ฟิลด์ระดับบนสุด

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `$schema` | ไม่ | URL ของ JSON Schema สำหรับการตรวจสอบความถูกต้อง |
| `name` | ใช่ | ตัวระบุเฉพาะของ marketplace |
| `version` | ไม่ | เวอร์ชันสคีมาของ marketplace (semver) |
| `metadata.description` | ใช่ | คำอธิบาย marketplace ที่มนุษย์อ่านได้ |
| `owner.name` | ใช่ | องค์กรหรือผู้ใช้ที่เป็นเจ้าของ marketplace |
| `owner.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของเจ้าของ |
| `owner.email` | ไม่ | อีเมลสำหรับติดต่อ |
| `plugins` | ใช่ | อาร์เรย์ของรายการปลั๊กอิน |

### ฟิลด์ของรายการปลั๊กอิน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน (ต้องตรงกับ `plugin.json`) |
| `description` | ใช่ | คำอธิบายสั้นๆ ของปลั๊กอิน |
| `version` | ใช่ | สตริงเวอร์ชันแบบ Semantic |
| `author.name` | ใช่ | ผู้เขียนปลั๊กอิน |
| `source` | ใช่ | เส้นทางหรือ URL ไปยังปลั๊กอิน (ดูประเภทของ Source) |
| `category` | ใช่ | หมวดหมู่ปลั๊กอิน (เช่น `productivity`) |
| `homepage` | ไม่ | URL ไปยังเอกสารหรือหน้าแรกของปลั๊กอิน |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `keywords` | ไม่ | อาร์เรย์ของคำค้นหา |
| `tags` | ไม่ | อาร์เรย์ของแท็กสำหรับการกรอง |
| `repository` | ไม่ | URL ของ repository |

## ประเภทของ Plugin Source

ฟิลด์ `source` ใน marketplace.json รองรับหลายรูปแบบ:

| ประเภท | ตัวอย่าง | คำอธิบาย |
|------|---------|-------------|
| เส้นทางแบบสัมพัทธ์ | `./plugins/my-plugin` | ปลั๊กอินใน repository เดียวกัน |
| GitHub แบบย่อ | `owner/repo` | ปลั๊กอินที่รูทของ repository |
| GitHub พร้อมเส้นทาง | `owner/repo/path/to/plugin` | ปลั๊กอินในไดเรกทอรีย่อย |
| Git URL | `https://github.com/owner/repo.git` | Git repository |
| แพ็กเกจ npm | `npm:@scope/package` | แพ็กเกจ npm ที่เผยแพร่แล้ว |

## สคีมา plugin.json

แต่ละปลั๊กอินมีไฟล์ manifest อยู่ที่
`.claude-plugin/plugin.json` ภายในไดเรกทอรีของมัน

| ฟิลด์ | จำเป็น | คำอธิบาย |
|-------|----------|-------------|
| `name` | ใช่ | ตัวระบุปลั๊กอิน |
| `description` | ใช่ | สิ่งที่ปลั๊กอินทำ |
| `version` | ใช่ | เวอร์ชันแบบ Semantic |
| `author.name` | ใช่ | ผู้เขียนปลั๊กอิน |
| `author.url` | ไม่ | URL ไปยังโปรไฟล์หรือเว็บไซต์ของผู้เขียน |
| `homepage` | ไม่ | URL เอกสาร |
| `keywords` | ไม่ | คำค้นหา |
| `license` | ไม่ | ตัวระบุสัญญาอนุญาต SPDX |
| `repository` | ไม่ | URL ของ repository |

## โครงสร้างไดเรกทอรีของปลั๊กอิน

```
plugins/my-plugin/
  .claude-plugin/
    plugin.json          # Plugin manifest (required)
  skills/
    skill-name/
      SKILL.md           # Skill definition (required per skill)
      references/        # Reference data files (optional)
  commands/
    command-name.md      # Command definition (one per command)
  agents/
    agent-name.md        # Agent definition (one per agent)
  README.md              # Plugin documentation (recommended)
```

## ตัวแปรสภาพแวดล้อม

ปลั๊กอินสามารถใช้ตัวแปรเหล่านี้ในไฟล์ skill และ command ของตน:

| ตัวแปร | คำอธิบาย |
|----------|-------------|
| `$\{CLAUDE_PLUGIN_ROOT\}` | เส้นทางแบบสัมบูรณ์ไปยังไดเรกทอรีรูทของปลั๊กอิน |

## การตรวจสอบความถูกต้อง

ตรวจสอบความถูกต้องของ marketplace และ plugin manifest ในเครื่องของคุณ:

```bash
claude plugin validate .
```

คำสั่งนี้ตรวจสอบ:

- ความสอดคล้องกับสคีมา `marketplace.json`
- ไดเรกทอรีปลั๊กอินทั้งหมดที่ถูกอ้างอิงมีอยู่จริง
- แต่ละปลั๊กอินมี `plugin.json` ที่ถูกต้อง
- Skill มี frontmatter ของ `SKILL.md` ที่มีรูปแบบถูกต้อง
- Command มีฟิลด์ frontmatter ที่จำเป็น

## การกำหนดค่าทีม

### การกำหนดค่า marketplace ล่วงหน้า

เพิ่มใน `.claude/settings.json` ใน repository ใดก็ได้:

```json
{
  "extraKnownMarketplaces": [
    "f5xc-salesdemos/marketplace"
  ]
}
```

### การเปิดใช้งานปลั๊กอินล่วงหน้า

```json
{
  "enabledPlugins": [
    "f5xc-docs-tools@f5xc-salesdemos-marketplace"
  ]
}
```

<Aside type="tip">
  การคอมมิตการตั้งค่าเหล่านี้ลงใน repository ของคุณจะช่วยให้สมาชิกทุกคนในทีมได้รับการกำหนดค่า marketplace และปลั๊กอินเดียวกันโดยอัตโนมัติ
</Aside>

## การแก้ไขปัญหา

### ไม่พบ marketplace

ตรวจสอบว่า marketplace ถูกเพิ่มแล้ว:

```
/plugin marketplace list
```

หากไม่พบ ให้เพิ่มอีกครั้ง:

```
/plugin marketplace add f5xc-salesdemos/marketplace
```

### ปลั๊กอินไม่ทำงาน

- ยืนยันว่าปลั๊กอินถูกติดตั้งแล้ว:
  `/plugin list`
- ตรวจสอบว่า Claude Code เวอร์ชันของคุณรองรับปลั๊กอิน
  (v1.0.33+)
- ตรวจสอบว่าเงื่อนไขการเรียกใช้ของ skill ตรงกับบริบทปัจจุบันของคุณ

### Skill ไม่ถูกเรียกใช้โดยอัตโนมัติ

Skill จะเปิดใช้งานตามฟิลด์ `description` ใน frontmatter ของ SKILL.md หาก skill ไม่ถูกเปิดใช้งานตามที่คาดหวัง:

- ตรวจสอบว่าคำอธิบายระบุบริบทการเรียกใช้ได้อย่างถูกต้อง
- ใช้คำสั่งโดยตรง (เช่น `/f5xc-docs-tools:review-mdx`)
  เป็นวิธีแก้ปัญหาเบื้องต้น

### ข้อผิดพลาดในการ build หลังจากการตรวจสอบ

ปลั๊กอินรายงานปัญหาแต่ไม่แก้ไขอัตโนมัติ ให้นำการแก้ไขที่แนะนำไปใช้ด้วยตนเอง จากนั้นรันการตรวจสอบอีกครั้งเพื่อยืนยันว่าปัญหาได้รับการแก้ไขแล้ว
