**salesforce** プラグインは Salesforce CLI を Claude Code と統合し、
組織認証および任意の `sf` コマンド（SOQL クエリを含む）を実行できる
汎用 CLI エージェントを提供します。JWT、アクセストークン、SFDX URL
認証フローによるヘッドレスコンテナ環境をサポートします。
以下の使用ガイドでは、CLI エージェントを通じて自然言語でアカウント、
商談、ケース、コンタクトを照会する方法を示します。

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## インストール

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## 前提条件

- **Salesforce CLI** (`@salesforce/cli`)
- **Salesforce 組織**（API アクセス権限付き）
- オブジェクト（ケース、商談、アカウント、コンタクト）を照会する権限を持つユーザーアカウント

ワークステーションに Salesforce CLI をインストールします：

```bash
brew install sf
```

インストールを確認します：

```bash
sf --version
```

`@salesforce/cli/2.x.x` のような出力が表示されます。

## 認証のセットアップ

### ステップ 1：Salesforce ドメインを確認する

ブラウザの Salesforce URL を確認します。Salesforce に
`https://acme.lightning.force.com` でアクセスしている場合、
ログインドメインは `https://acme.my.salesforce.com` です。

<Aside type="tip">
  会社が SSO を使用している場合、`--instance-url` でカスタムドメインを
  指定する必要があります。これにより、一般的な Salesforce ログインページ
  ではなく SSO プロバイダー経由でログインがリダイレクトされます。
</Aside>

### ステップ 2：ブラウザ経由で認証する

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

`YOUR-DOMAIN` をステップ 1 で確認した会社ドメインに置き換えます。
ブラウザが開きますので、SSO またはログインフローを完了してアプリを
承認してください。以下のように表示されます：

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### ステップ 3：接続を確認する

```bash
sf org display --target-org my-org
```

出力に `Connected Status: Connected` が表示され、ユーザー名と
インスタンス URL が表示されることを確認します。

### ステップ 4：コンテナ用のポータブル認証（オプション）

ヘッドレス環境（CI/CD、コンテナ、リモートセッション）で認証する
必要がある場合は、ワークステーションから SFDX 認証 URL をエクスポート
します：

```bash
sf org display --verbose --target-org my-org
```

**Sfdx Auth Url** の値（`force://` で始まる）をコピーします。
コンテナで以下を実行します：

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  SFDX 認証 URL は認証情報です — パスワードと同様に扱ってください。
  ファイルにコミットしたり、チャットに貼り付けたりしないでください。
  環境変数と stdin パイプのみを使用してください。
</Aside>

### 認証方法のリファレンス

| 方法 | 最適な用途 | 必要なもの |
|--------|----------|----------|
| Web ログイン | ブラウザが使用できるワークステーション | ブラウザ + SSO |
| SFDX URL | コンテナ、CI/CD、ポータブル認証 | 認証済みセッションからの認証 URL |
| JWT Bearer | 自動化パイプライン | 接続アプリ + 秘密鍵 + コンシューマーキー |
| アクセストークン | 環境変数による認証 | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**注意：** デバイスフロー（`sf org login device`）は 2025 年 8 月以降
ブロックされており、サポートされていません。

### 環境変数

| 変数 | 目的 |
|----------|---------|
| `SF_ACCESS_TOKEN` | アクセストークン認証用の Bearer トークン |
| `SFDX_AUTH_URL` | SFDX URL 認証用の Force 認証 URL |
| `SF_ORG_INSTANCE_URL` | 組織インスタンス URL |
| `SF_JWT_KEY_FILE` | JWT 秘密鍵へのパス |
| `SF_CLIENT_ID` | 接続アプリのコンシューマーキー |
| `SF_USERNAME` | JWT 用の Salesforce ユーザー名 |

## コマンド

### /salesforce:sf-status

Salesforce 組織の接続ステータス、CLI バージョン、認証済み組織を確認します。

```
/salesforce:sf-status
```

**期待される結果：** sf CLI バージョン、認証済み組織エイリアス、
ユーザー名、インスタンス URL、接続ステータス、API バージョンを表示する
テーブル。

### /salesforce:sf-login

Salesforce 組織に認証します。オプションでエイリアスを指定できます。

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**期待される結果：** そのエイリアスを持つ組織がすでに認証されている場合は
接続詳細を報告します。認証されていない場合は、利用可能な認証情報
（環境変数）を確認し、最初に条件が満たされた方法で認証します。

## スキル

### salesforce-index

トップレベルのインテントルーター。自然言語で Salesforce、組織管理、
またはデータクエリについて言及した際に自動的に有効化されます。

| 発言内容 | ルーティング先 |
|---------|-----------|
| "login to salesforce"、"authenticate org" | `salesforce-auth` スキル |
| "check org status"、"list orgs" | `cli-operator` エージェント |
| "show me opportunities"、"look up case" | `cli-operator` エージェント（SOQL クエリ） |
| "generate Apex class"、"create Flow" | afv-library 開発スキル |

### salesforce-auth

コンテナ対応の認証スキル。4 つの認証方法をサポートし、利用可能な
認証情報を自動検出します。優先順位：アクセストークン、JWT、
SFDX URL、Web ログインの順。

## エージェント

### cli-operator

安全ガードレール付きで `sf` コマンドを実行する自律型 Salesforce CLI
エージェント。メインセッションのコンテキストをスリムに保つため、
すべての Salesforce 操作はこのエージェントに委任されます。

**安全ルール：**

- デフォルトでは読み取り専用 — 書き込み操作には明示的な承認が必要
- プレビューと確認ステップなしにデプロイしない
- 出力にアクセストークン、認証 URL、または認証情報をエコーしない
- ユーザー指定の値をすべてサニタイズする（シェルのメタ文字を拒否）
- 構造化された解析可能な結果のために `--json` 出力を使用

**レスポンス形式：**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## 使用ガイド：アカウント管理

<Aside type="note">
  以下のプロンプトはプラグインにハードコードされた機能ではありません。
  cli-operator エージェントが任意の SOQL クエリを含む任意の `sf` CLI
  コマンドを実行できるため機能します。結果は組織のデータモデルと
  ユーザー権限によって異なります。ここのプロンプトは標準 Salesforce
  オブジェクト（Account、Contact、Case、Opportunity、
  AccountTeamMember、OpportunityTeamMember）で動作することが
  テストおよび検証済みです。
</Aside>

これらのプロンプトはあらゆる Salesforce 組織で機能します。
プレースホルダーの値は実際の情報に置き換えてください。各プロンプトは
あなたの役割、アカウント、テリトリーに固有の結果を生成します。

<Aside type="tip">
  最良の結果を得るには、ユーザーレコードを特定する必要があるプロンプトに
  メールアドレスを含めてください。Salesforce 組織には似た名前のユーザーが
  複数存在する場合があります。
</Aside>

### はじめに

認証後、すべてが正常に動作することを確認します：

```
/salesforce:sf-status
```

次に簡単な自然言語クエリを試します：

```
list my authenticated salesforce orgs
```

**期待される結果：** エイリアス、ユーザー名、インスタンス URL、
接続ステータスを含む、接続済みすべての組織を示すテーブル。

### アカウントを確認する

自分がアカウントチームに割り当てられているアカウントを確認します：

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**期待される結果：** アカウントオーナー別にグループ化されたアカウントの
リストと、各アカウントでのチームメンバーロール。ほとんどのロールが空の場合、
これはよくあるデータ品質の問題です — アカウントチームメンバーシップは
存在しますが、ロールが入力されていません。

### 同僚とのカバレッジを相互参照する

チームメートとのアカウントカバレッジを比較してギャップを見つけます：

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**期待される結果：** 重複数とギャップ数を示すサマリーテーブル、続いて
2 つのリスト。役割が最近変わった場合、重複がゼロになることがあります —
これにより追加が必要なアカウントが明らかになります。

<Aside type="note">
  「Also check if Previous Person Name appears on any of those accounts.」
  を追加することで、前任者の確認も行えます。これはテリトリーで
  誰かを引き継ぐ際の引き継ぎギャップを特定するのに役立ちます。
</Aside>

### テリトリーのパイプライン概要

アカウント全体のすべての進行中の商談を確認します：

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**期待される結果：** 商談の総数、生のパイプライン、重み付きパイプラインを
含むサマリー。次にステージ分布テーブルと金額上位の商談リスト。
期限切れのクローズ日や金額が null などのデータ品質の問題にフラグが立てられます。

### 商談の詳細分析

特定の案件の全体像を把握します：

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**期待される結果：** 商談の概要（ステージ、金額、クローズ日、確度、
予測カテゴリ、オーナー）、ロールを含む商談チームメンバー、タグ付きコンタクト、
最近のタスクとイベント、および案件の金額とクローズ日の推移を示すフィールド
変更履歴。

### テリトリー全体のサポートケース

アカウントのオープンサポートケースを確認します：

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**期待される結果：** 優先度とステータスの内訳を含むオープンケースの数、
次にアカウント別にグループ化されたケースの詳細。活動のない古いケース
（数か月または数年）はデータ品質の候補としてフラグが立てられます。

### 四半期パイプライン予測

パイプラインの予測に対応したビューを取得します：

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**期待される結果：** 四半期ごとに商談数、パイプライン合計、
重み付きパイプラインを示す 1 行のテーブル。四半期ごとのステージミックス
の内訳と上位の重み付き案件のハイライトを含みます。期限切れの商談は
別途グループ化されます。

### アカウントの全体概要

特定の顧客を詳しく分析します：

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**期待される結果：** 会社プロフィール（業界、売上、従業員数、所在地）、
役職とメールを含む主要コンタクト、ステージと金額を含むオープン商談、
ステータス付きの最近のサポートケース。

### ケース検索

ケース番号で特定のサポートケースを検索します：

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**期待される結果：** ケースの件名、ステータス、優先度、説明、顧客アカウントの
プロフィール、および連絡先情報を含むケースオーナー。

## 使用ガイド：データクエリ

パワーユーザー向けに、自然言語で特定の SOQL スタイルのクエリをリクエスト
できます。プラグインがリクエストを適切な SOQL に変換して実行します。

### 特定フィールド

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### 日付フィルタリング

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### 集計

```
count all open salesforce cases grouped by priority and status across my accounts
```

### カスタムオブジェクト

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  フィールドが存在しないためにクエリが失敗した場合、プラグインは
  そのフィールドなしで自動的に再試行します。組織のカスタムフィールドは
  異なる名前を使用している場合があります — フィールドが見つからない場合は
  Salesforce の設定を確認してください。
</Aside>

npx skills add forcedotcom/afv-library
