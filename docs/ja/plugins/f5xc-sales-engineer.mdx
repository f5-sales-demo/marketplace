**f5xc-sales-engineer** プラグインは、F5 XC デモリポジトリ向けの再利用可能なセールスエンジニアペルソナフレームワークを提供します。ミーティング前の確認からライブ実行、Q&A、ティアダウンまで、デモミーティングのライフサイクル全体を管理します。

<Badge text="v1.0.6" variant="note" /> <Badge text="生産性" variant="tip" />

## インストール

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## スキル

### demo-executor

四段階のミーティングライフサイクルによる API 駆動のデモ実行：

1. **準備 (Prepare)** — `demo-housekeeping` エージェントによるミーティング前の環境確認
2. **実行 (Execute)** — 顧客向けナレーションを伴う決定論的フェーズ実行
3. **Q&A** — ライブ環境でのアドリブモード
4. **ティアダウン (Teardown)** — `demo-housekeeping` エージェントによるミーティング後のクリーンアップ

`DEMO_PRODUCT_EXPERTISE.md` から製品固有のコンテンツを、`docs/demo/` からデモコマンドを読み込みます。

### presenter

事前設定されたデモ環境を使用したビルド済みウォークスループレゼンテーション。各ステップで「表示 → ナレーション → 接続 → 一時停止」のパターンに従います。ウォークスルーの順序は `DEMO_WALKTHROUGH_CONFIG.md` から読み込まれます。

### subject-matter-expert

厳格な引用要件を持つ参考資料に基づく Q&A。推測は一切行わず、すべての回答に参考資料または証拠が含まれます。必要に応じてリサーチを `demo-researcher` エージェントに委任します。

### sales-engineer

ユーザーの意図を適切なペルソナスキルにルーティングするロールインデックス。リクエストがデモ関連であるものの、特定のトリガーフレーズに一致しない場合に起動します。

## エージェント

### demo-housekeeping

ミーティング前の確認とミーティング後のティアダウンのための自律エージェント。`DEMO_READINESS_MATRIX.md` で定義されたレディネス確認マトリクス (T0〜T5) を実行し、`docs/demo/phase-4-teardown.mdx` からティアダウンコマンドを実行します。

### demo-researcher

検証済みの回答を引用付きで検索する読み取り専用のリサーチライブラリアン。最初にローカルの `docs/` を検索し、次に `DEMO_SOURCE_INDEX.md` のインデックス済み外部ソースを検索し、フォールバックとして WebSearch を使用します。

## 規約ファイル

<Aside type="tip" title="製品固有のコンテンツ">
  このプラグインは製品非依存です。各リポジトリは標準化された規約ファイルを通じて製品固有のコンテンツを提供します。
</Aside>

プラグインを設定するために、リポジトリのルートに以下のファイルを作成してください：

| ファイル | 目的 | 必須対象 |
| ---- | ------- | ------------ |
| `DEMO_PRODUCT_EXPERTISE.md` | 製品の機能、検出シグナル、脅威のカバレッジ、コンプライアンスの整合性、API リファレンス | すべてのスキル |
| `DEMO_WALKTHROUGH_CONFIG.md` | デモアプリの URL、ウォークスルーの順序、検出タイミング | presenter |
| `DEMO_SOURCE_INDEX.md` | ローカルドキュメントと外部 URL を含むリサーチソースカタログ | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | 必須/任意の変数、レディネスチェック (T0〜T5)、API エンドポイント | demo-housekeeping |
| `docs/demo/` | cURL コマンドとエビデンスゲートを含むフェーズファイル | demo-executor |

## トリガーフレーズ

`docs-control` によって管理される `.claude/CLAUDE.md` のトリガーマップは、ユーザーの意図に基づいてスキルを起動します：

| フレーズ | スキル | ステージ |
| ------ | ----- | ----- |
| "run the demo", "execute the demo" | demo-executor | 実行 |
| "prepare the demo", "pre-flight" | demo-executor | 準備 |
| "Q&A", "question and answer" | demo-executor | Q&A |
| "tear down", "clean up" | demo-executor | ティアダウン |
| "walk through the demo", "present" | presenter | — |
| "answer questions", "explain" | subject-matter-expert | — |

## ナレーションスタイル

すべてのペルソナは一貫したナレーションスタイルを共有しています：

- **現在形、一人称複数** — 「ここでご覧いただいているのは…」
- **段落ごとに一つの懸念事項** — 各ナレーションは「これは何か？なぜ重要か？どう対処すべきか？」に答えます
- **シグナルの明示** — どの機能が働いているかを明示します
- **関連する場合はコンプライアンスへの言及** — 規制との整合性を自然に言及します
- **ペーシングポーズマーカー** — `> ⏸ *[Pause for audience — ready to continue?]*`
