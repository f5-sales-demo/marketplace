**f5xc-devcontainer** プラグインは、f5-sales-demo 開発環境にコンテナ認識機能を提供します。包括的なツールカタログを維持し、ライブコンテナイントロスペクションを有効にし、GitHub Issues を通じた永続性追跡によるツールインストールを処理します。

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## インストール

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## スキル

### tool-catalog

クラウド、ネットワーク、セキュリティ、開発などのカテゴリ別に整理された、開発コンテナにインストールされているすべての CLI ツールのナレッジベース。使用するツールの選択方法、ツールの使用方法、インストール済みかどうかの確認、または特定の操作に関するサポートが必要な場合にアクティブになります。

### self-awareness

コンテナのアイデンティティと自己診断スキル。GitHub API およびローカルビルドメタデータを通じて、実行中のコンテナのライブイントロスペクションを提供します。アイデンティティ、バージョン、ビルド情報、ヘルスチェック、およびコントリビューター履歴に関する質問でアクティブになります。

<Aside type="tip">
  「who are you?」または「what version am I running?」と尋ねることで、self-awareness をトリガーできます。
</Aside>

## エージェント

### tool-advisor

ツールカタログを参照し、CLI ツールに関するあらゆる質問に対して、目的、クイックスタートコマンド、および認証要件とともに推奨事項を返します。

### tool-auditor

ツールカタログのリファレンスファイルと照合して開発コンテナの Dockerfile を監査し、ドリフトを検出します &mdash; イメージに追加されたがカタログに登録されていないツール、または削除されたツールのカタログエントリを検出します。

### container-introspector

GitHub API およびローカルメタデータを通じて、アイデンティティ、系譜、自己診断、コントリビューター履歴、およびビルド系統のためのライブコンテナイントロスペクションを実行します。読み取り専用操作のみ対応しています。

### container-maintainer

正しいパッケージマネージャーパターンを使用して、実行中のコンテナに CLI ツールをインストール、削除、検索、および更新します。永続性追跡のために、開発コンテナリポジトリに対して GitHub Issues を登録します。
