**f5xc-devcontainer** プラグインは、f5xc-salesdemos 開発環境にコンテナ認識機能を提供します。包括的なツールカタログを維持し、ライブコンテナイントロスペクションを有効にし、GitHub issues による永続性追跡を伴うツールインストールを処理します。

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## インストール

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## スキル

### tool-catalog

クラウド、ネットワーク、セキュリティ、開発などのカテゴリ別に整理された、開発コンテナにインストールされたすべての CLI ツールのナレッジベース。使用するツールの確認、ツールの使い方、インストール状況の確認、または特定の操作に関するヘルプが必要な場合に有効化されます。

### self-awareness

コンテナのアイデンティティと自己診断スキル。GitHub API およびローカルビルドメタデータを通じて、実行中のコンテナのライブイントロスペクションを提供します。アイデンティティ、バージョン、ビルド情報、ヘルスチェック、およびコントリビューター履歴に関する質問に対して有効化されます。

<Aside type="tip">
  「who are you?」または「what version am I running?」と質問することで、self-awareness をトリガーできます。
</Aside>

## エージェント

### tool-advisor

ツールカタログを参照し、CLI ツールに関するあらゆる質問に対して、目的・クイックスタートコマンド・認証要件を含む推奨情報を返します。

### tool-auditor

ツールカタログのリファレンスファイルに対して開発コンテナの Dockerfile を監査し、ドリフトを検出します。イメージには追加されているがカタログに登録されていないツール、または削除済みのツールのカタログエントリを検出します。

### container-introspector

GitHub API およびローカルメタデータを通じてライブコンテナイントロスペクションを実行し、アイデンティティ、系譜、自己診断、コントリビューター履歴、およびビルド系統を確認します。読み取り専用操作のみを実行します。

### container-maintainer

正しいパッケージマネージャーパターンを使用して、実行中のコンテナの CLI ツールのインストール、削除、検索、および更新を行います。永続性追跡のために、開発コンテナリポジトリに対して GitHub issues を作成します。
