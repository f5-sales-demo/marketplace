**f5xc-github-ops** プラグインは、f5-sales-demo リポジトリ向けの GitHub 操作自動化を専用に提供します。issue の作成からマージ後のモニタリングまで、Git および GitHub のライフサイクル全体を管理し、issue 駆動開発とコミット前のリントゲートを適用します。

<Badge text="v2.2.2" variant="note" /> <Badge text="生産性" variant="tip" />

## インストール

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## スキル

### workflow-lifecycle

issue の作成、ブランチ命名規則、PR の作成、エラーフィードバックを issue に返す CI ポーリング、マージ後のモニタリング、検証、およびタスク完了基準を処理するリポジトリガバナンスワークフロー。「commit」、「push」、「create a PR」、「merge」と言ったとき、または Git 操作を実行したときに起動します。

### upstream-contribution

フォークベースの開発向けアップストリーム貢献ワークフロー。フォーク検出、アップストリームのリサーチ（issue、PR、CONTRIBUTING.md、CI パターン）、クロスリポジトリの issue および PR の作成、ローカルトラッキング issue の管理、アップストリーム CI のモニタリング、およびガイド付き解決を処理します。

## エージェント

### github-ops

すべての機械的な Git および GitHub 操作専用のエージェント。コミット前のリントゲート、issue の作成、ブランチの作成、ステージング、コミット、プッシュ、PR の作成、エラーフィードバック付き CI ポーリング、インフラ障害の再試行、マージ後のモニタリング、ブランチのクリーンアップ、および GitHub API を介したリポジトリ設定の管理を処理します。

構造化されたステータスレポートを返します: `COMPLETE`、`PRE_COMMIT_FAILED`、`CI_FAILED`、`BLOCKED`、または `FAILED`。

<Aside type="caution">
  このエージェントは Git 操作の**専用**ハンドラーです。
  このプラグインを使用するリポジトリでは、`git commit`、`git push`、または `gh pr create` を直接実行しないでください。
</Aside>
