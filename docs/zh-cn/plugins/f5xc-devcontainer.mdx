**f5xc-devcontainer** 插件为 f5xc-salesdemos 开发环境提供容器感知能力。它维护一个全面的工具目录，支持实时容器内省，并通过 GitHub Issues 处理带持久化跟踪的工具安装。

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## 安装

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## 技能

### tool-catalog

devcontainer 中所有已安装 CLI 工具的知识库，按类别（云、网络、安全、开发等）组织。当您询问应使用哪个工具、如何使用某工具、某工具是否已安装，或需要特定操作的帮助时，该技能将被激活。

### self-awareness

容器身份识别与自我诊断技能。通过 GitHub API 和本地构建元数据提供运行中容器的实时内省。适用于身份识别、版本信息、构建详情、健康检查及贡献者历史等问题时激活。

<Aside type="tip">
  输入"who are you?"或"what version am I running?"可触发自我感知技能。
</Aside>

## 代理

### tool-advisor

读取工具目录，并针对任何 CLI 工具问题返回包含用途说明、快速启动命令及身份验证要求的推荐建议。

### tool-auditor

对照工具目录参考文件审计 devcontainer 的 Dockerfile，以检测漂移——即已添加到镜像但未入目录的工具，或已从目录中删除的工具条目。

### container-introspector

通过 GitHub API 和本地元数据执行实时容器内省，涵盖身份识别、血缘关系、自我诊断、贡献者历史及构建溯源。仅支持只读操作。

### container-maintainer

使用正确的包管理器模式在运行中的容器内安装、移除、搜索和更新 CLI 工具。针对 devcontainer 仓库提交 GitHub Issues 以进行持久化跟踪。
