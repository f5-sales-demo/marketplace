**f5xc-devcontainer** 插件为 f5-sales-demo 开发环境提供容器感知能力。它维护一个完整的工具目录，支持实时容器自省，并通过 GitHub Issues 实现带持久性追踪的工具安装管理。

<Badge text="v1.1.6" variant="note" /> <Badge text="生产力" variant="tip" />

## 安装

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## 技能

### tool-catalog

devcontainer 中已安装的每个 CLI 工具的知识库，按类别（云、网络、安全、开发等）进行组织。当您询问应使用哪个工具、如何使用某工具、某工具是否已安装，或需要特定操作帮助时，此技能将被激活。

### self-awareness

容器身份识别与自诊断技能。通过 GitHub API 和本地构建元数据提供运行中容器的实时自省能力。在询问身份信息、版本信息、构建信息、健康检查及贡献者历史时激活。

<Aside type="tip">
  输入"who are you?"或"what version am I running?"可触发自我感知功能。
</Aside>

## 代理

### tool-advisor

读取工具目录，并针对任意 CLI 工具问题返回包含用途说明、快速入门命令和身份验证要求的推荐内容。

### tool-auditor

根据工具目录参考文件对 devcontainer Dockerfile 进行审计，以检测偏差 &mdash; 包括已添加到镜像但未在目录中记录的工具，以及目录中存在但已从镜像中移除的工具条目。

### container-introspector

通过 GitHub API 和本地元数据执行实时容器自省，涵盖身份识别、谱系溯源、自诊断、贡献者历史和构建血缘分析。仅支持只读操作。

### container-maintainer

使用正确的包管理器模式在运行中的容器内安装、移除、搜索和更新 CLI 工具。向 devcontainer 仓库提交 GitHub Issues 以进行持久性追踪。
