Il plugin **f5xc-devcontainer** fornisce la consapevolezza dei container per
l'ambiente di sviluppo f5xc-salesdemos. Mantiene un
catalogo completo degli strumenti, abilita l'introspezione live del container
e gestisce l'installazione degli strumenti con tracciamento della persistenza tramite
issue GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installazione

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Competenze

### tool-catalog

Base di conoscenza di ogni strumento CLI installato nel contenitore di sviluppo,
organizzata per categoria (cloud, rete, sicurezza, sviluppo,
ecc.). Si attiva quando si chiede quale strumento utilizzare, come utilizzare
uno strumento, se qualcosa è installato, o si necessita di assistenza con
un'operazione specifica.

### self-awareness

Competenza di identità del container e autodiagnosi. Fornisce
l'introspezione live del container in esecuzione tramite API GitHub e metadati
di build locali. Si attiva per domande su identità, versione, informazioni di build,
controlli di integrità e cronologia dei contributori.

<Aside type="tip">
  Chiedi "chi sei?" o "quale versione sto eseguendo?" per attivare
  la self-awareness.
</Aside>

## Agenti

### tool-advisor

Legge il catalogo degli strumenti e restituisce raccomandazioni con scopo,
comandi di avvio rapido e requisiti di autenticazione per qualsiasi domanda
relativa agli strumenti CLI.

### tool-auditor

Verifica il Dockerfile del contenitore di sviluppo rispetto ai file di riferimento del catalogo
degli strumenti per rilevare derive &mdash; strumenti aggiunti all'immagine
ma non catalogati, o voci del catalogo per strumenti che sono stati
rimossi.

### container-introspector

Esegue l'introspezione live del container tramite API GitHub e metadati
locali per identità, genealogia, autodiagnosi, cronologia dei contributori
e lineage di build. Solo operazioni di sola lettura.

### container-maintainer

Installa, rimuove, cerca e aggiorna strumenti CLI nel container
in esecuzione utilizzando i pattern corretti del gestore di pacchetti. Apre issue GitHub
nel repository del contenitore di sviluppo per il
tracciamento della persistenza.
