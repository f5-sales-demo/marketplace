Il plugin **salesforce** integra la Salesforce CLI con Claude Code,
fornendo l'autenticazione org e un agente CLI generico in grado di
eseguire qualsiasi comando `sf`, incluse le query SOQL. Supporta ambienti
container headless con flussi di autenticazione JWT, access-token e SFDX URL.
La guida all'utilizzo di seguito illustra come usare il linguaggio naturale per
interrogare account, opportunità, casi e contatti tramite l'agente CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## Installazione

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## Prerequisiti

- **Salesforce CLI** (`@salesforce/cli`)
- **Org Salesforce** con accesso API
- Un account utente con autorizzazioni per interrogare gli oggetti (Cases, Opportunities, Accounts, Contacts)

Installa la Salesforce CLI sulla tua workstation:

```bash
brew install sf
```

Verifica l'installazione:

```bash
sf --version
```

Dovresti vedere un output simile a `@salesforce/cli/2.x.x`.

## Configurazione dell'autenticazione

### Passaggio 1: Trova il tuo dominio Salesforce

Osserva l'URL di Salesforce nel browser. Se accedi a Salesforce tramite
`https://acme.lightning.force.com`, il tuo dominio di accesso è
`https://acme.my.salesforce.com`.

<Aside type="tip">
  Se la tua azienda utilizza SSO, devi specificare il tuo dominio personalizzato con
  `--instance-url` affinché il login venga reindirizzato attraverso il tuo provider SSO
  invece della pagina di accesso generica di Salesforce.
</Aside>

### Passaggio 2: Autenticati tramite browser

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

Sostituisci `YOUR-DOMAIN` con il dominio aziendale del Passaggio 1. Si aprirà
il browser — completa il flusso SSO o di accesso e autorizza l'app. Dovresti
vedere:

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### Passaggio 3: Verifica la connessione

```bash
sf org display --target-org my-org
```

Conferma che l'output mostri `Connected Status: Connected` con il tuo nome
utente e l'URL dell'istanza.

### Passaggio 4: Auth portabile per container (opzionale)

Se devi autenticarti in un ambiente headless (CI/CD, container,
sessione remota), esporta l'SFDX auth URL dalla tua workstation:

```bash
sf org display --verbose --target-org my-org
```

Copia il valore **Sfdx Auth Url** (inizia con `force://`). Nel
container, esegui:

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  L'SFDX auth URL è una credenziale — trattala come una password. Non
  includerla in file né incollarla in chat. Utilizza esclusivamente
  variabili d'ambiente e pipe stdin.
</Aside>

### Riferimento ai metodi di autenticazione

| Metodo | Ideale per | Richiede |
|--------|------------|----------|
| Web Login | Workstation con browser | Browser + SSO |
| SFDX URL | Container, CI/CD, auth portabile | Auth URL da una sessione autenticata |
| JWT Bearer | Pipeline automatizzate | Connected App + chiave privata + consumer key |
| Access Token | Auth tramite variabile d'ambiente | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**Nota:** Il flusso device (`sf org login device`) è bloccato dall'agosto
2025 e non è supportato.

### Variabili d'ambiente

| Variabile | Scopo |
|-----------|-------|
| `SF_ACCESS_TOKEN` | Bearer token per l'auth con access-token |
| `SFDX_AUTH_URL` | Force auth URL per l'auth con SFDX URL |
| `SF_ORG_INSTANCE_URL` | URL dell'istanza org |
| `SF_JWT_KEY_FILE` | Percorso alla chiave privata JWT |
| `SF_CLIENT_ID` | Consumer key della Connected App |
| `SF_USERNAME` | Nome utente Salesforce per JWT |

## Comandi

### /salesforce:sf-status

Verifica lo stato della connessione alla tua org Salesforce, la versione
della CLI e le org autenticate.

```
/salesforce:sf-status
```

**Cosa aspettarsi:** Una tabella che mostra la versione della CLI sf, l'alias
dell'org autenticata, il nome utente, l'URL dell'istanza, lo stato della
connessione e la versione API.

### /salesforce:sf-login

Esegue l'autenticazione a una org Salesforce. È possibile fornire opzionalmente un alias.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**Cosa aspettarsi:** Se un'org con quell'alias è già autenticata,
vengono riportati i dettagli della connessione. In caso contrario, viene
verificata la disponibilità di credenziali (variabili d'ambiente) e
l'autenticazione viene eseguita con il primo metodo completamente soddisfatto.

## Competenze

### salesforce-index

Router di intenti di primo livello. Si attiva automaticamente quando
menzioni Salesforce, la gestione org o query dati in linguaggio naturale.

| Cosa dici | Instrada a |
|-----------|------------|
| "login to salesforce", "authenticate org" | competenza `salesforce-auth` |
| "check org status", "list orgs" | agente `cli-operator` |
| "show me opportunities", "look up case" | agente `cli-operator` (query SOQL) |
| "generate Apex class", "create Flow" | competenze di sviluppo afv-library |

### salesforce-auth

Competenza di autenticazione adattata ai container. Supporta quattro metodi
di autenticazione e rileva automaticamente quali credenziali sono disponibili.
Priorità: access-token, poi JWT, poi SFDX URL, poi web login.

## Agenti

### cli-operator

Agente autonomo della Salesforce CLI che esegue i comandi `sf` con
protezioni di sicurezza. Tutte le operazioni Salesforce vengono delegate a
questo agente per mantenere snello il contesto della sessione principale.

**Regole di sicurezza:**

- Solo lettura per impostazione predefinita — le operazioni di scrittura richiedono approvazione esplicita
- Non esegue mai deploy senza un passaggio di anteprima e conferma
- Non include mai access token, auth URL o credenziali nell'output
- Igienizza tutti i valori forniti dall'utente (rifiuta i metacaratteri shell)
- Utilizza l'output `--json` per risultati strutturati e analizzabili

**Formato di risposta:**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## Guida all'utilizzo: Gestione account

<Aside type="note">
  I prompt di seguito non sono funzionalità codificate nel plugin. Funzionano
  perché l'agente cli-operator può eseguire qualsiasi comando della CLI `sf`,
  incluse query SOQL arbitrarie. I risultati dipendono dal modello dati della
  tua org e dalle autorizzazioni del tuo utente. I prompt qui riportati sono
  stati testati e verificati con gli oggetti Salesforce standard (Account,
  Contact, Case, Opportunity, AccountTeamMember, OpportunityTeamMember).
</Aside>

Questi prompt funzionano con qualsiasi org Salesforce. Sostituisci i valori
segnaposto con le tue informazioni. Ogni prompt produce risultati specifici
per il tuo ruolo, i tuoi account e il tuo territorio.

<Aside type="tip">
  Per risultati ottimali, includi il tuo indirizzo email nei prompt che devono
  identificare il tuo record utente. Le org Salesforce possono avere più utenti
  con nomi simili.
</Aside>

### Per iniziare

Dopo l'autenticazione, verifica che tutto funzioni:

```
/salesforce:sf-status
```

Poi prova una semplice query in linguaggio naturale:

```
list my authenticated salesforce orgs
```

**Cosa aspettarsi:** Una tabella che mostra tutte le org connesse con alias,
nome utente, URL dell'istanza e stato della connessione.

### Trova i tuoi account

Scopri a quali account sei assegnato:

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**Cosa aspettarsi:** Un elenco di account raggruppati per proprietario, che
mostra il tuo ruolo nel team di account per ciascuno. Se la maggior parte
dei ruoli risulta vuota, si tratta di una lacuna comune nella qualità dei
dati — le appartenenze al team di account esistono, ma i ruoli non sono
stati popolati.

### Confronta la copertura con un collega

Confronta la copertura dei tuoi account con quella di un compagno di team
per individuare le lacune:

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**Cosa aspettarsi:** Una tabella riepilogativa con il conteggio delle
sovrapposizioni e delle lacune, seguita da due elenchi. Se hai cambiato
ruolo di recente, potresti trovare zero sovrapposizioni — questo rivela
gli account a cui devi essere aggiunto.

<Aside type="note">
  Puoi anche verificare la presenza di un predecessore aggiungendo: "Also check if
  Previous Person Name appears on any of those accounts." Questo aiuta
  a identificare le lacune di passaggio di consegne quando si sostituisce
  qualcuno su un territorio.
</Aside>

### Panoramica della pipeline del territorio

Visualizza tutte le opportunità aperte sui tuoi account:

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**Cosa aspettarsi:** Un riepilogo con il conteggio totale delle opportunità,
la pipeline grezza e la pipeline pesata. Poi una tabella di distribuzione
per fase e un elenco delle principali opportunità per importo. L'output
segnala problemi di qualità dei dati come date di chiusura scadute o
importi nulli.

### Analisi approfondita di un'opportunità

Ottieni il quadro completo di un'opportunità specifica:

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**Cosa aspettarsi:** La panoramica dell'opportunità (fase, importo, data di
chiusura, probabilità, categoria di previsione, proprietario), i membri del
team dell'opportunità con i ruoli, i contatti associati, le attività e gli
eventi recenti, e la cronologia delle modifiche ai campi che mostra come
l'importo e la data di chiusura della trattativa sono cambiati nel tempo.

### Casi di supporto nel territorio

Verifica i casi di supporto aperti sui tuoi account:

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**Cosa aspettarsi:** Un conteggio dei casi aperti con suddivisione per
priorità e stato, seguito dai dettagli dei casi raggruppati per account.
I casi obsoleti (mesi o anni senza attività) vengono segnalati come
candidati per la pulizia dei dati.

### Previsione pipeline trimestrale

Ottieni una vista della pipeline pronta per la previsione:

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**Cosa aspettarsi:** Una tabella con una riga per trimestre che mostra il
conteggio delle opportunità, la pipeline totale e la pipeline pesata. Include
una suddivisione del mix di fasi per trimestre ed evidenzia le trattative
più significative in termini di peso. Le opportunità scadute vengono
raggruppate separatamente.

### Panoramica completa dell'account

Analisi approfondita di un cliente specifico:

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**Cosa aspettarsi:** Profilo aziendale (settore, fatturato, dipendenti,
posizione), contatti chiave con titoli ed email, opportunità aperte con
fasi e importi, e casi di supporto recenti con stato.

### Ricerca caso

Cerca un caso di supporto specifico per numero:

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**Cosa aspettarsi:** Oggetto del caso, stato, priorità, descrizione, il
profilo dell'account cliente e il proprietario del caso con le informazioni
di contatto.

## Guida all'utilizzo: Query dati

Per gli utenti avanzati, è possibile richiedere query in stile SOQL in
linguaggio naturale. Il plugin traduce la richiesta nel SOQL appropriato
ed eseguirlo.

### Campi specifici

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### Filtraggio per data

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### Aggregazione

```
count all open salesforce cases grouped by priority and status across my accounts
```

### Oggetti personalizzati

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  Se una query fallisce perché un campo non esiste, il plugin
  riprova automaticamente senza quel campo. I campi personalizzati nella
  tua org potrebbero avere nomi diversi — controlla le impostazioni di
  Salesforce se un campo non viene trovato.
</Aside>
