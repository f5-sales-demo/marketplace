Il plugin **f5xc-docs-pipeline** fornisce conoscenza dell'architettura
della pipeline di documentazione per i repository f5-sales-demo. Guida
la creazione dei contenuti, chiarisce la proprietà delle configurazioni
nelle build multi-repository e fornisce un'anteprima locale basata su Docker.

<Badge text="v1.0.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installazione

```
/plugin install f5xc-docs-pipeline@f5-sales-demo-marketplace
```

## Competenze

### content-author

Guida alla creazione di contenuti che copre la struttura dei file, le regole di sintassi MDX,
i requisiti del frontmatter, i riferimenti alle immagini e l'anteprima locale con Docker.
Si attiva durante la creazione o la modifica di contenuti in `docs/`,
quando si lavora con file MDX o durante la configurazione dell'anteprima locale.

### pipeline-navigator

Architettura della pipeline di documentazione che spiega dove apportare
modifiche nei quattro repository upstream: docs-theme,
docs-builder, docs-control e docs-icons. Chiarisce la proprietà delle
configurazioni (`astro.config.mjs`, `content.config.ts`), la catena di
distribuzione delle release e il tracciamento dei file gestiti.

<Aside type="tip">
  Utilizzare pipeline-navigator quando è necessario comprendere quale
  repository è responsabile di uno specifico file di configurazione o componente.
</Aside>

## Comandi

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

Avvia un server di sviluppo Docker locale per l'anteprima della documentazione. Esegue
l'immagine Docker di build condivisa con la directory `docs/` locale
montata per l'anteprima in tempo reale.
