Il plugin **f5xc-sales-engineer** fornisce un framework di persona riutilizzabile per Sales Engineer per i repository demo di F5 XC. Gestisce l'intero ciclo di vita della riunione demo — dalla verifica pre-riunione fino all'esecuzione live, al Q&A e al teardown.

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installazione

```
/plugin install f5xc-sales-engineer@f5-sales-demo-marketplace
```

## Competenze

### demo-executor

Esecuzione della demo basata su API con un ciclo di vita della riunione in quattro fasi:

1. **Prepare** — verifica dell'ambiente pre-riunione tramite l'agente `demo-housekeeping`
2. **Execute** — esecuzione deterministica delle fasi con narrazione rivolta al cliente
3. **Q&A** — modalità improvvisazione con ambiente live
4. **Teardown** — pulizia post-riunione tramite l'agente `demo-housekeeping`

Legge i contenuti specifici del prodotto da `DEMO_PRODUCT_EXPERTISE.md` e i comandi demo da `docs/demo/`.

### presenter

Presentazione della procedura dettagliata as-built utilizzando un ambiente demo preconfigurato. Segue uno schema mostra → narra → connetti → pausa a ogni passaggio. Legge l'ordine della procedura dettagliata da `DEMO_WALKTHROUGH_CONFIG.md`.

### subject-matter-expert

Q&A con riferimenti, con requisiti rigorosi di citazione. Non fa mai ipotesi — ogni risposta include un riferimento o una prova. Delega la ricerca all'agente `demo-researcher` quando necessario.

### sales-engineer

Indice dei ruoli che instrada l'intento dell'utente alla competenza di persona corretta. Si attiva quando la richiesta è correlata alla demo ma non corrisponde a una frase di attivazione specifica.

## Agenti

### demo-housekeeping

Agente autonomo per la verifica pre-riunione e il teardown post-riunione. Esegue la Matrice di Verifica della Prontezza (T0–T5) definita in `DEMO_READINESS_MATRIX.md` e i comandi di teardown da `docs/demo/phase-4-teardown.mdx`.

### demo-researcher

Bibliotecario di ricerca in sola lettura che trova risposte verificate con citazioni. Cerca prima nella directory locale `docs/`, poi nelle fonti esterne indicizzate da `DEMO_SOURCE_INDEX.md`, con WebSearch come fallback.

## File di convenzione

<Aside type="tip" title="Contenuto specifico del prodotto">
  Questo plugin è agnostico rispetto al prodotto. Ogni repository fornisce
  contenuto specifico del prodotto tramite file di convenzione standardizzati.
</Aside>

Creare questi file nella root del repository per configurare il plugin:

| File | Scopo | Richiesto per |
| ---- | ----- | ------------- |
| `DEMO_PRODUCT_EXPERTISE.md` | Funzionalità del prodotto, segnali di rilevamento, copertura delle minacce, allineamento alla conformità, riferimento API | Tutte le competenze |
| `DEMO_WALKTHROUGH_CONFIG.md` | URL dell'app demo, ordine della procedura dettagliata, tempi di rilevamento | presenter |
| `DEMO_SOURCE_INDEX.md` | Catalogo delle fonti di ricerca con documentazione locale e URL esterni | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | Variabili obbligatorie/opzionali, verifiche di prontezza (T0–T5), endpoint API | demo-housekeeping |
| `docs/demo/` | File di fase con comandi cURL e gate di evidenza | demo-executor |

## Frasi di attivazione

La mappa di attivazione `.claude/CLAUDE.md` (gestita da `docs-control`) attiva le competenze in base all'intento dell'utente:

| Frase | Competenza | Fase |
| ----- | ---------- | ---- |
| "run the demo", "execute the demo" | demo-executor | Execute |
| "prepare the demo", "pre-flight" | demo-executor | Prepare |
| "Q&A", "question and answer" | demo-executor | Q&A |
| "tear down", "clean up" | demo-executor | Teardown |
| "walk through the demo", "present" | presenter | — |
| "answer questions", "explain" | subject-matter-expert | — |

## Stile di narrazione

Tutte le persone condividono uno stile di narrazione coerente:

- **Presente, prima persona plurale** — "Quello che stiamo guardando qui…"
- **Un argomento per paragrafo** — ogni narrazione risponde a: Cos'è questo? Perché è importante? Cosa dovrei fare al riguardo?
- **Nominare il segnale** — indicare quale funzionalità è in uso
- **Hook di conformità quando pertinente** — menzionare l'allineamento normativo in modo naturale
- **Indicatore di pausa nel ritmo** — `> ⏸ *[Pause for audience — ready to continue?]*`
