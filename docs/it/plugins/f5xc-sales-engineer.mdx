Il plugin **f5xc-sales-engineer** fornisce un framework persona Sales Engineer
riutilizzabile per i repository demo di F5 XC. Gestisce l'intero ciclo di vita
di una riunione demo — dalla verifica pre-meeting all'esecuzione live,
Q&A e teardown.

<Badge text="v1.0.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installazione

```
/plugin install f5xc-sales-engineer@f5xc-salesdemos-marketplace
```

## Skill

### demo-executor

Esecuzione demo basata su API con un ciclo di vita della riunione in quattro fasi:

1. **Preparazione** — verifica dell'ambiente pre-meeting tramite
   l'agente `demo-housekeeping`
2. **Esecuzione** — esecuzione deterministica delle fasi con
   narrazione rivolta al cliente
3. **Q&A** — modalità di improvvisazione con ambiente live
4. **Teardown** — pulizia post-meeting tramite l'agente `demo-housekeeping`

Legge i contenuti specifici del prodotto da `DEMO_PRODUCT_EXPERTISE.md` e i
comandi demo da `docs/demo/`.

### presenter

Presentazione walkthrough as-built utilizzando un ambiente demo
preconfigurato. Segue un pattern mostra → narra → collega → pausa ad
ogni passaggio. Legge l'ordine del walkthrough da `DEMO_WALKTHROUGH_CONFIG.md`.

### subject-matter-expert

Q&A con riferimenti e requisiti rigorosi di citazione. Non
improvvisa mai — ogni risposta include un riferimento o una prova. Delega
la ricerca all'agente `demo-researcher` quando necessario.

### sales-engineer

Indice dei ruoli che instrada l'intento dell'utente verso lo skill persona
corretto. Si attiva quando la richiesta è relativa alla demo ma non corrisponde
a una frase trigger specifica.

## Agenti

### demo-housekeeping

Agente autonomo per la verifica pre-meeting e il teardown
post-meeting. Esegue la Matrice di Verifica della Prontezza (T0–T5) definita
in `DEMO_READINESS_MATRIX.md` ed esegue i comandi di teardown da
`docs/demo/phase-4-teardown.mdx`.

### demo-researcher

Bibliotecario di ricerca in sola lettura che trova risposte verificate con
citazioni. Cerca prima nella documentazione locale `docs/`, poi nelle fonti esterne
indicizzate da `DEMO_SOURCE_INDEX.md`, con WebSearch come fallback.

## File di Convenzione

<Aside type="tip" title="Contenuti specifici del prodotto">
  Questo plugin è agnostico rispetto al prodotto. Ogni repository fornisce
  contenuti specifici del prodotto attraverso file di convenzione standardizzati.
</Aside>

Create questi file nella root del vostro repository per configurare il plugin:

| File | Scopo | Richiesto Per |
| ---- | ----- | ------------- |
| `DEMO_PRODUCT_EXPERTISE.md` | Capacità del prodotto, segnali di rilevamento, copertura delle minacce, allineamento alla conformità, riferimento API | Tutti gli skill |
| `DEMO_WALKTHROUGH_CONFIG.md` | URL dell'app demo, ordine del walkthrough, tempistiche di rilevamento | presenter |
| `DEMO_SOURCE_INDEX.md` | Catalogo delle fonti di ricerca con documenti locali e URL esterni | demo-researcher |
| `DEMO_READINESS_MATRIX.md` | Variabili obbligatorie/opzionali, controlli di prontezza (T0–T5), endpoint API | demo-housekeeping |
| `docs/demo/` | File di fase con comandi cURL e gate di evidenza | demo-executor |

## Frasi Trigger

La mappa trigger in `.claude/CLAUDE.md` (gestita da `docs-control`)
attiva gli skill in base all'intento dell'utente:

| Frase | Skill | Fase |
| ----- | ----- | ---- |
| "run the demo", "execute the demo" | demo-executor | Esecuzione |
| "prepare the demo", "pre-flight" | demo-executor | Preparazione |
| "Q&A", "question and answer" | demo-executor | Q&A |
| "tear down", "clean up" | demo-executor | Teardown |
| "walk through the demo", "present" | presenter | — |
| "answer questions", "explain" | subject-matter-expert | — |

## Stile di Narrazione

Tutte le persona condividono uno stile di narrazione coerente:

- **Tempo presente, prima persona plurale** — "Quello che stiamo
  osservando qui…"
- **Un concetto per paragrafo** — ogni narrazione risponde a: Cos'è
  questo? Perché è importante? Cosa dovrei fare al riguardo?
- **Nominare il segnale** — evidenziare quale capacità è in azione
- **Riferimento alla conformità quando pertinente** — menzionare l'allineamento normativo
  in modo naturale
- **Marcatore di pausa per il ritmo** — `> ⏸ *[Pausa per il pubblico — pronti a continuare?]*`
