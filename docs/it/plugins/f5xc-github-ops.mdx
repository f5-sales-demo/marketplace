Il plugin **f5xc-github-ops** fornisce un'Automazione esclusiva delle operazioni GitHub
per i repository f5-sales-demo. Gestisce l'intero ciclo di vita
Git e GitHub, dalla creazione delle issue fino al monitoraggio
post-merge, applicando lo sviluppo basato su issue e i controlli
di lint pre-commit.

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installazione

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## Competenze

### workflow-lifecycle

Workflow di governance del repository che gestisce la creazione delle issue, le convenzioni
di denominazione dei branch, la creazione delle PR, il polling CI con feedback degli errori
alle issue, il monitoraggio post-merge, la verifica e i criteri di completamento delle attività.
Si attiva quando si dice "commit", "push", "create a PR", "merge", o si esegue qualsiasi operazione Git.

### upstream-contribution

Workflow di contribuzione upstream per lo sviluppo basato su fork. Gestisce
il rilevamento del fork, la ricerca upstream (issue, PR, CONTRIBUTING.md,
pattern CI), la creazione di issue e PR cross-repo, la gestione delle issue
di tracciamento locale, il monitoraggio CI upstream e la risoluzione guidata.

## Agenti

### github-ops

Agente esclusivo per tutte le operazioni meccaniche Git e GitHub.
Gestisce i controlli di lint pre-commit, la creazione di issue, la creazione di branch,
la staging, il commit, il push, la creazione di PR, il polling CI con feedback
degli errori, il retry in caso di guasti infrastrutturali, il monitoraggio post-merge,
la pulizia dei branch e la gestione delle impostazioni del repository tramite le
API GitHub.

Restituisce report di stato strutturati: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` o `FAILED`.

<Aside type="caution">
  Questo agente è il gestore **esclusivo** per le operazioni Git.
  Non eseguire mai direttamente `git commit`, `git push` o `gh pr create`
  nei repository che utilizzano questo plugin.
</Aside>
