Das **osint-framework**-Plugin bietet einen OSINT-Werkzeugkatalog und
Ermittlungsfähigkeiten — 1.064 kostenlose Tools zur Informationsgewinnung
in 34 Kategorien, abgebildet von [osintframework.com](https://osintframework.com).
Es umfasst kategoriebasierte Fähigkeiten, ausführbare Ermittlungs-Pipelines,
CLI-Tool-Ausführung, OPSEC-bewusste Workflows und ein persistentes
Entitätsgraph für ermittlungsübergreifende Korrelation.

<Badge text="v1.0.1" variant="note" /> <Badge text="Security" variant="tip" />

## Installation

```
/plugin install osint-framework@f5xc-salesdemos-marketplace
```

## Befehle

### /osint-investigate

Führt eine vollständige OSINT-Untersuchung gegen ein beliebiges Ziel durch.
Erkennt automatisch den Zieltyp (E-Mail, Domain, IP, Benutzername, Unternehmen,
Personenname) und leitet an die entsprechenden Kategoriefähigkeiten weiter.

```
/osint-investigate j.smith@example.com
/osint-investigate example.com
/osint-investigate 198.51.100.1
/osint-investigate example-user
```

### /osint-search

Durchsucht den Werkzeugkatalog nach Tools, die einer Fähigkeitsabfrage entsprechen.

```
/osint-search subdomain enumeration
/osint-search email breach check
/osint-search satellite imagery
```

### /osint-catalog

Durchsucht den vollständigen Werkzeugkatalog nach Kategorien. Ohne Argumente werden
alle 34 Kategorien mit Tool-Anzahlen angezeigt.

```
/osint-catalog
/osint-catalog domain
/osint-catalog threat-intel
```

## Fähigkeiten

### osint-index

Übergeordneter Absichts-Router. Wird automatisch aktiviert, wenn Sie „Informationen
suchen über", „untersuchen", „nachschlagen", „Hintergrundüberprüfung" sagen oder
eine beliebige Open-Source-Intelligence-Aufgabe beschreiben. Leitet basierend auf
dem Zieltyp zur richtigen Kategoriefähigkeit weiter.

| Ziel | Weiterleitung zu |
|------|-----------------|
| Benutzername oder Handle | `username-recon` |
| E-Mail-Adresse | `email-recon` |
| Domainname | `domain-recon` |
| IP-Adresse | `ip-address-recon` |
| Personenname + Unternehmen | `people-search` + `business-records` |
| Unternehmen / Organisation | `business-records` + `domain-recon` |
| Malware-Hash | `malicious-file-analysis` + `threat-intelligence` |

### osint-catalog

Werkzeugkatalog-Browser. Bietet Zugriff auf die vollständige Werkzeugregistrierung,
den Kategorieindex, Ermittlungs-Pipelines, Ratenbegrenzungsreferenzen und die
Dokumentation der Korrelations-Engine.

## Agenten

### osint-investigator

Autonomer Multi-Tool-Ermittlungsorchestrierungs-Agent. Führt vollständige
Ermittlungs-Workflows durch — Werkzeugverfügbarkeitsprüfung, Zieltyperkennung,
sequenzielle Tool-Ausführung (zuerst passiv), Ergebnisaggregation und strukturierte
Berichtserstellung mit Konfidenzwerten und OPSEC-Klassifikationen.

Integriert mit dem Entitätsgraph: Jedes Tool-Ergebnis erstellt Entitäten und
Beziehungen in `/tmp/osint-graph/` für ermittlungsübergreifende Korrelation.

<Aside type="caution">
  Aktive Reconnaissance-Werkzeuge (nmap, nuclei, sherlock) nehmen direkten Kontakt
  mit der Zielinfrastruktur auf. Der OPSEC-Warnungs-Hook benachrichtigt vor der
  Ausführung aktiver Werkzeuge. Stellen Sie stets sicher, dass Sie eine Genehmigung haben.
</Aside>

### osint-researcher

Nur-Lese-Katalogsuche-Agent. Durchsucht Referenzdateien, um die besten Werkzeuge
für eine bestimmte Aufgabe zu empfehlen. Führt niemals Werkzeuge aus — empfiehlt
und erklärt nur. Wird intern von Kategoriefähigkeiten zur Werkzeugentdeckung verwendet.

### osint-executor

Einzelnes CLI-Ausführungs-Tool-Agent. Führt jeweils ein Werkzeug mit spezifischen
Parametern aus, analysiert die Ausgabe und gibt strukturierte Ergebnisse zurück.
Enthält werkzeugspezifische Ausführungsrezepte für 20+ Werkzeuge mit genauen Flags,
Ausgabeformaten und jq-Parsing-Mustern.

## Ratenbegrenzung

Alle API-Aufrufe verwenden den `osint_curl`-Helfer mit eingebautem exponentiellem
Backoff bei HTTP 429/503. Wichtige Grenzwerte:

| API | Kostenloses Limit | Umgebungsvariable |
|-----|------------------|------------------|
| ipinfo.io | 1.000/Tag | — |
| crt.sh | 60/Min | — |
| NVD | 5/30s (ohne Schlüssel) | `NVD_API_KEY` |
| OpenCorporates | Erfordert Schlüssel | `OPENCORPORATES_API_KEY` |
| Shodan | 1/Sek | `SHODAN_API_KEY` |
| GitHub API | 60/Std (ohne Authentifizierung) | `GH_TOKEN` |

APIs, die Schlüssel benötigen, werden automatisch übersprungen und eine
klare Meldung ausgegeben, wenn die Umgebungsvariable nicht gesetzt ist.

## Korrelations-Engine

Das Plugin enthält einen persistenten JSON-Entitätsgraph unter
`/tmp/osint-graph/`, der Erkenntnisse aus mehreren Untersuchungen ansammelt.

```bash
source plugins/osint-framework/scripts/osint-graph.sh
osint_graph_init

# Add entities
P=$(osint_entity_add "person" "J. Smith" --tool github-api)
C=$(osint_entity_add "company" "F5" --tool github-api)

# Link them
osint_rel_add "$P" "$C" "works_at" --tool github-api

# Query: all entities within 2 hops of the person
osint_graph_query "$P" 2

# Generate report
osint_graph_report
```

**Konfidenzwertung** verwendet Multi-Quellen-Fusion:
`confidence = 1 - product(1 - ci)` für jede bestätigende Quelle.
Zwei Quellen bei 0,80 → kombinierte Konfidenz von 0,96.

## Rechtlicher Hinweis

Alle Werkzeuge verwenden ausschließlich öffentlich zugängliche Informationen. Die
Nutzer sind dafür verantwortlich, die Einhaltung geltender Gesetze, Vorschriften
und Nutzungsbedingungen der Plattformen sicherzustellen. Verwenden Sie OSINT-Werkzeuge
niemals für unbefugten Zugriff, Belästigung oder illegale Aktivitäten.
