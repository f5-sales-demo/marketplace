Das **f5xc-docs-tools**-Plugin validiert MDX-Inhaltsdateien für
die f5xc-salesdemos-Dokumentationspipeline. Es erkennt häufige
Build-brechende Probleme, bevor sie die CI erreichen, darunter
ungeschützte JSX-Zeichen, ungültige Imports, fehlerhafte
Bildreferenzen und unvollständige Frontmatter.

<Badge text="v1.1.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-docs-tools@f5xc-salesdemos-marketplace
```

## Skills

### mdx-content-reviewer

Dieser Skill wird automatisch aktiviert, wenn Claude erkennt, dass Sie
mit MDX-Dateien in einem f5xc-salesdemos-Inhalts-Repository
arbeiten. Er führt sieben Validierungsprüfungen durch:

<CardGrid>
  <Card title="Frontmatter-Validierung">
    Prüft erforderliche Felder wie `title` und `sidebar.order`.
    Splash-Seiten müssen `hero`-Felder und
    `template: splash` enthalten.
  </Card>
  <Card title="MDX-Syntaxfallen">
    Erkennt ungeschützte `&lt;`-Zeichen und nicht-escapte `\{` `\}`-
    Klammern, die das MDX-Parsing unterbrechen.
  </Card>
  <Card title="Import-Validierung">
    Validiert Imports gegen eine Allowlist von Starlight-
    integrierten und f5xc-salesdemos-Theme-Komponenten.
  </Card>
  <Card title="Komponentenattribute">
    Prüft erforderliche Props für Komponenten wie Screenshot,
    Aside, Code, LinkCard, Card und Badge.
  </Card>
  <Card title="Bildreferenzen">
    Überprüft, ob referenzierte Bilder im Verzeichnis `docs/images/`
    vorhanden sind.
  </Card>
  <Card title="Strukturprüfungen">
    Stellt sicher, dass `docs/index.mdx` existiert und Bildverzeichnisse
    keine verstreuten MDX-Dateien enthalten.
  </Card>
  <Card title="Exports und Codeblöcke">
    Überprüft, ob Variablen, die in `Code`-Komponenten-`code`-
    Props verwendet werden, passende Exports haben.
  </Card>
</CardGrid>

#### Scoping-Verhalten

Der Skill begrenzt seine Überprüfung intelligent:

- Wenn es nicht commitete oder gestagete Änderungen an `docs/**/*.mdx`-
  Dateien gibt, werden nur diese Dateien überprüft
- Andernfalls werden alle `docs/**/*.mdx`-Dateien überprüft

#### Ausgabeformat

Ergebnisse werden nach Schweregrad gruppiert:

- **ERROR** — wird den Build unterbrechen; muss vor dem Mergen behoben werden
- **WARNING** — wahrscheinliche Probleme, die behoben werden sollten
- **INFO** — Vorschläge und Best Practices

Jedes Ergebnis enthält den Dateipfad, die Zeilennummer und eine
Beschreibung des Problems.

## Befehle

### /review-mdx

```
/f5xc-docs-tools:review-mdx [path-or-glob]
```

Führt den `mdx-content-reviewer`-Skill bei Bedarf aus.

**Argumente:**

| Argument | Erforderlich | Beschreibung |
|----------|----------|-------------|
| `path-or-glob` | Nein | Dateipfad oder Glob-Muster zur Eingrenzung der Überprüfung. Standardmäßig werden alle `docs/**/*.mdx`-Dateien verwendet. |

**Beispiele:**

```
# Alle MDX-Dateien in docs/ überprüfen
/f5xc-docs-tools:review-mdx

# Eine bestimmte Datei überprüfen
/f5xc-docs-tools:review-mdx docs/getting-started.mdx

# Dateien überprüfen, die einem Glob entsprechen
/f5xc-docs-tools:review-mdx docs/guides/*.mdx
```

**Ausgabe:** Eine Zusammenfassungszeile am Ende gibt die Gesamtzahl
der überprüften Dateien und die Anzahl der Ergebnisse nach Schweregrad aus.

## Erlaubte Imports

Das Plugin validiert Imports gegen diese Quellen:

**Starlight-integrierte Komponenten** (`@astrojs/starlight/components`):

`Aside` `Badge` `Card` `CardGrid` `Code` `FileTree` `Icon`
`LinkCard` `Steps` `TabItem` `Tabs`

**f5xc-salesdemos-Theme-Komponenten** (`@f5xc-salesdemos/docs-theme/components/`):

`Banner` `Icon` `LinkCard` `Screenshot`

<Aside type="caution">
  Theme-Komponenten verwenden Default-Imports aus einzelnen `.astro`-
  Dateien. Direkte `node_modules`-Imports, relative Pfade und
  `require()`-Aufrufe werden als Warnungen markiert.
</Aside>

## Komponenten-Kurzreferenz

| Komponente | Erforderliche Props | Optionale Props |
|-----------|---------------|----------------|
| Screenshot | `alt` + mindestens eines von `light` oder `dark` | — |
| Aside | `type` | `title` |
| Code | `code`, `lang` | `title`, `frame`, `mark`, `ins`, `del` |
| LinkCard (Theme) | `title`, `href` | `description`, `icon` |
| Card | `title` | `icon` |
| Badge | `text` | `variant` |
| Steps | — | — |
| Tabs / TabItem | TabItem: `label` | — |
| CardGrid | — | — |
| FileTree | — | — |

## MDX-Stolperfallen

Häufige Probleme, die das Plugin erkennt:

| Stolperfalle | Problem | Lösung |
|---------|---------|-----|
| Ungeschütztes `&lt;` | MDX interpretiert es als JSX-Tag | Verwenden Sie `&amp;lt;`, Inline-Code oder formulieren Sie um |
| Nicht-escaptes `\{` `\}` | MDX behandelt es als JSX-Ausdruck | Verwenden Sie Inline-Code, escapen Sie mit `\` oder verwenden Sie einen Codeblock |
| Klammern in Dateinamen | Astro kann die Datei nicht verarbeiten | Verwenden Sie niemals `\{` oder `\}` in `.mdx`-Dateinamen |
