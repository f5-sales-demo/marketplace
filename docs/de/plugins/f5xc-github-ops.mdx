Das Plugin **f5xc-github-ops** bietet exklusive GitHub-Operationen-Automatisierung
für f5-sales-demo-Repositorys. Es verwaltet den vollständigen
Git- und GitHub-Lebenszyklus von der Issue-Erstellung bis zur
Post-Merge-Überwachung, setzt issue-getriebene Entwicklung durch
und erzwingt Pre-Commit-Lint-Gates.

<Badge text="v2.2.2" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## Skills

### workflow-lifecycle

Repository-Governance-Workflow zur Verwaltung von Issue-Erstellung, Branch-
Namenskonventionen, PR-Erstellung, CI-Polling mit Fehler-Feedback
an Issues, Post-Merge-Überwachung, Verifizierung und Aufgaben-
Abschlusskriterien. Wird aktiviert, wenn Sie „commit", „push",
„create a PR", „merge" sagen oder eine Git-Operation durchführen.

### upstream-contribution

Upstream-Beitrags-Workflow für fork-basierte Entwicklung. Verwaltet
Fork-Erkennung, Upstream-Recherche (Issues, PRs, CONTRIBUTING.md,
CI-Muster), repositoryübergreifende Issue- und PR-Erstellung,
lokales Tracking-Issue-Management, Upstream-CI-Überwachung
und geführte Problemlösung.

## Agenten

### github-ops

Exklusiver Agent für alle mechanischen Git- und GitHub-Operationen.
Verwaltet Pre-Commit-Lint-Gates, Issue-Erstellung, Branch-Erstellung,
Staging, Commit, Push, PR-Erstellung, CI-Polling mit Fehler-
Feedback, Wiederholung bei Infrastrukturfehlern, Post-Merge-Überwachung,
Branch-Bereinigung und Repository-Einstellungsverwaltung über die
GitHub-API.

Gibt strukturierte Statusberichte zurück: `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` oder `FAILED`.

<Aside type="caution">
  Dieser Agent ist der **exklusive** Handler für Git-Operationen.
  Führen Sie niemals `git commit`, `git push` oder `gh pr create` direkt
  in Repositorys aus, die dieses Plugin verwenden.
</Aside>
