Das **f5xc-devcontainer**-Plugin bietet Container-Bewusstsein für
die f5xc-salesdemos-Entwicklungsumgebung. Es pflegt einen
umfassenden Werkzeugkatalog, ermöglicht Live-Container-Introspektion
und verwaltet die Werkzeuginstallation mit Persistenzverfolgung über
GitHub Issues.

<Badge text="v1.1.6" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Fähigkeiten

### tool-catalog

Wissensbasis aller im Entwicklungscontainer installierten CLI-Werkzeuge,
organisiert nach Kategorien (Cloud, Netzwerk, Sicherheit, Entwicklung
usw.). Wird aktiviert, wenn Sie fragen, welches Werkzeug zu verwenden ist,
wie ein Werkzeug zu verwenden ist, ob etwas installiert ist, oder wenn Sie
Hilfe bei einem bestimmten Vorgang benötigen.

### self-awareness

Fähigkeit zur Container-Identität und Selbstdiagnose. Bietet Live-
Introspektion des laufenden Containers über die GitHub-API und lokale
Build-Metadaten. Wird aktiviert für Fragen zu Identität, Version,
Build-Informationen, Gesundheitsprüfungen und Beitragsverlauf.

<Aside type="tip">
  Fragen Sie „Wer bist du?" oder „Welche Version verwende ich?", um
  die Selbstwahrnehmung auszulösen.
</Aside>

## Agenten

### tool-advisor

Liest den Werkzeugkatalog und gibt Empfehlungen mit Zweck,
Schnellstart-Befehlen und Authentifizierungsanforderungen für jede
CLI-Werkzeug-Anfrage zurück.

### tool-auditor

Prüft das Devcontainer-Dockerfile anhand der Werkzeugkatalog-
Referenzdateien, um Abweichungen zu erkennen &mdash; Werkzeuge, die dem
Image hinzugefügt, aber nicht katalogisiert wurden, oder Katalogeinträge
für Werkzeuge, die entfernt wurden.

### container-introspector

Führt Live-Container-Introspektion über die GitHub-API und lokale
Metadaten durch für Identität, Genealogie, Selbstdiagnose, Beitragsverlauf
und Build-Herkunft. Nur Lesevorgänge.

### container-maintainer

Installiert, entfernt, sucht und aktualisiert CLI-Werkzeuge im laufenden
Container unter Verwendung korrekter Paketverwaltermuster. Erstellt GitHub
Issues im Devcontainer-Repository zur Persistenzverfolgung.
