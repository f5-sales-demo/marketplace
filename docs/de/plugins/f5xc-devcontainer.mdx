Das **f5xc-devcontainer**-Plugin bietet Container-Awareness für
die f5-sales-demo-Entwicklungsumgebung. Es pflegt einen
umfassenden Werkzeugkatalog, ermöglicht Live-Container-Introspektion
und verwaltet die Werkzeuginstallation mit Persistenzverfolgung über
GitHub-Issues.

<Badge text="v1.1.6" variant="note" /> <Badge text="Produktivität" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Skills

### tool-catalog

Wissensdatenbank aller im Entwicklungscontainer installierten CLI-Werkzeuge,
organisiert nach Kategorie (Cloud, Netzwerk, Sicherheit, Entwicklung
usw.). Wird aktiviert, wenn Sie fragen, welches Werkzeug zu verwenden ist,
wie ein Werkzeug genutzt wird, ob etwas installiert ist, oder wenn Sie
Hilfe bei einem bestimmten Vorgang benötigen.

### self-awareness

Skill für Container-Identität und Selbstdiagnose. Ermöglicht Live-
Introspektion des laufenden Containers über die GitHub API und lokale
Build-Metadaten. Wird für Fragen zu Identität, Version, Build-Informationen,
Systemzustandsprüfungen und Mitwirkungshistorie aktiviert.

<Aside type="tip">
  Fragen Sie „Wer bist du?" oder „Welche Version verwende ich?" um
  Self-Awareness auszulösen.
</Aside>

## Agenten

### tool-advisor

Liest den Werkzeugkatalog und gibt Empfehlungen mit Zweck,
Schnellstart-Befehlen und Authentifizierungsanforderungen für jede
CLI-Werkzeugfrage zurück.

### tool-auditor

Prüft das Devcontainer-Dockerfile gegen die Referenzdateien des Werkzeugkatalogs,
um Abweichungen zu erkennen &mdash; Werkzeuge, die dem Image hinzugefügt, aber
nicht katalogisiert wurden, oder Katalogeinträge für Werkzeuge, die entfernt
wurden.

### container-introspector

Führt Live-Container-Introspektion über die GitHub API und lokale
Metadaten durch – für Identität, Genealogie, Selbstdiagnose, Mitwirkungshistorie
und Build-Herkunft. Ausschließlich lesende Operationen.

### container-maintainer

Installiert, entfernt, sucht und aktualisiert CLI-Werkzeuge im laufenden
Container mithilfe korrekter Paketverwaltungsmuster. Erstellt GitHub-Issues
gegen das Devcontainer-Repository zur Persistenzverfolgung.
