Le plugin **f5xc-docs-pipeline** fournit des connaissances sur l'architecture
du pipeline de documentation pour les dépôts f5-sales-demo. Il guide
la rédaction de contenu, clarifie la propriété des configurations dans les
builds multi-dépôts, et propose une prévisualisation locale basée sur Docker.

<Badge text="v1.0.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-docs-pipeline@f5-sales-demo-marketplace
```

## Compétences

### content-author

Guide de rédaction de contenu couvrant la structure des fichiers, les règles
de syntaxe MDX, les exigences de frontmatter, les références d'images et la
prévisualisation locale via Docker. S'active lors de la création ou de la
modification de contenu dans `docs/`, du travail avec des fichiers MDX, ou
de la configuration d'une prévisualisation locale.

### pipeline-navigator

Architecture du pipeline de documentation expliquant où effectuer des
modifications dans les quatre dépôts en amont : docs-theme, docs-builder,
docs-control et docs-icons. Clarifie la propriété des configurations
(`astro.config.mjs`, `content.config.ts`), la chaîne de déclenchement
des releases et le suivi des fichiers gérés.

<Aside type="tip">
  Utilisez pipeline-navigator lorsque vous devez déterminer quel
  dépôt est propriétaire d'un fichier de configuration ou d'un composant
  spécifique.
</Aside>

## Commandes

### /preview-docs

```
/f5xc-docs-pipeline:preview-docs
```

Démarre un serveur de développement Docker local pour prévisualiser la
documentation. Exécute l'image Docker de build partagée avec votre répertoire
`docs/` local monté pour une prévisualisation en direct.
