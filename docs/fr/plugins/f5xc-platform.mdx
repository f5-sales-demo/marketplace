Le plugin **f5xc-platform** fournit une Automatisation complète pour la
Plateforme F5 Distributed Cloud. Il couvre les opérations de la console web via
Chrome DevTools MCP, les opérations d'API REST avec prise en charge des spécifications via cURL,
l'authentification multi-fournisseur et l'analyse de configuration sur
98 types de ressources dans 38 domaines.

<Badge text="v3.1.1" variant="note" /> <Badge text="Productivité" variant="tip" />

## Installation

```
/plugin install f5xc-platform@f5-sales-demo-marketplace
```

## Compétences

### api-operations

Opérations CRUD d'API avec prise en charge des spécifications couvrant 98 types de ressources sur
38 domaines. Fournit des points de terminaison précis, des modèles de charge utile,
un ordonnancement des dépendances et des compositions de workflows multi-étapes pour
les équilibreurs de charge HTTP, les pools d'origine, les politiques WAF, les zones DNS,
les certificats, les sites cloud et bien plus encore.

### config-analysis

Capacité d'analyse de configuration et de conseil. Analyse les
configurations JSON des clients pour répondre aux questions sur la posture de Sécurité,
l'activation des fonctionnalités, les paramètres de mode, les exclusions et les bonnes pratiques.
Prend en charge les questions-réponses multi-tours avec un contexte compressé entre les dispatches.

### api-auth

Authentification API prenant en charge l'authentification par en-tête de jeton API et
l'authentification par certificat P12 avec des procédures de validation de jeton.

### console-auth

Authentification multi-fournisseur pour la console web F5 XC.
Détecte automatiquement le type de connexion : connexion native F5 XC, SSO Azure avec
session en cache, ou SSO Azure avec MFA push vérifié DUO.

### console-navigator

Navigation déterministe vers les sections de la console F5 XC. Traduit
les noms de sections en chemins d'URL, vérifie l'état de la session et confirme
le chargement de la page.

### platform-index / api-index / console-index

Routeurs d'intention qui classifient les demandes des utilisateurs et les délèguent à
la compétence et à l'agent appropriés. Ces routeurs s'activent automatiquement et ne
sont pas invocables par l'utilisateur.

## Commandes

### /check-api-token

```
/f5xc-platform:check-api-token
```

Valide votre jeton API F5 XC en déléguant à l'agent
api-operator.

### /login-console

```
/f5xc-platform:login-console
```

Authentifie à la console F5 Distributed Cloud via
l'agent console-operator.

## Agents

### api-operator

Agent REST API autonome exécutant des séquences cURL et jq pour
les opérations CRUD sur les ressources, la validation des jetons et les opérations de configuration.
S'exécute dans un sous-agent pour maintenir les charges utiles JSON d'API volumineuses en dehors du
contexte de session principal.

### console-operator

Agent d'Automatisation de navigateur autonome exécutant des séquences d'outils Chrome DevTools MCP
pour l'authentification, la navigation et les interactions avec les formulaires.
S'exécute dans un sous-agent pour maintenir les instantanés de navigateur riches en jetons en dehors de la session principale.

### config-analyzer

Agent d'analyse de configuration en lecture seule. Analyse les configurations
JSON des clients par rapport aux fichiers de référence de profil de ressource pour répondre aux
questions sur la posture de Sécurité, l'activation des fonctionnalités, les transitions de mode
et les bonnes pratiques.

<Aside type="caution">
  L'agent config-analyzer est **éphémère** &mdash; instancié à nouveau
  pour chaque question sans mémoire entre les dispatches. Les questions de suivi
  doivent inclure les résultats précédents compressés dans l'invite de dispatch.
</Aside>
