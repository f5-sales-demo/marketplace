Le plugin **f5xc-devcontainer** fournit une connaissance du conteneur pour
l'environnement de développement f5-sales-demo. Il maintient un
catalogue d'outils complet, permet l'introspection en direct du conteneur
et gère l'installation des outils avec un suivi de persistance via
des tickets GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5-sales-demo-marketplace
```

## Compétences

### tool-catalog

Base de connaissances de tous les outils CLI installés dans le conteneur de développement,
organisée par catégorie (cloud, réseau, sécurité, développement,
etc.). S'active lorsque vous demandez quel outil utiliser, comment utiliser
un outil, si quelque chose est installé, ou si vous avez besoin d'aide pour une
opération spécifique.

### self-awareness

Compétence d'identité du conteneur et d'autodiagnostic. Fournit une
introspection en direct du conteneur en cours d'exécution via l'API GitHub et les
métadonnées de build locales. S'active pour les questions relatives à l'identité, à la version,
aux informations de build, aux vérifications de l'état de santé et à l'historique des contributeurs.

<Aside type="tip">
  Demandez « qui êtes-vous ? » ou « quelle version est-ce que j'utilise ? » pour déclencher
  l'autoconnaissance.
</Aside>

## Agents

### tool-advisor

Lit le catalogue d'outils et retourne des recommandations avec la finalité,
les commandes de démarrage rapide et les exigences d'authentification pour toute question
relative aux outils CLI.

### tool-auditor

Effectue un audit du Dockerfile du conteneur de développement par rapport aux fichiers de référence
du catalogue d'outils pour détecter les écarts &mdash; outils ajoutés à l'image
mais non répertoriés, ou entrées du catalogue pour des outils qui ont été
supprimés.

### container-introspector

Effectue une introspection en direct du conteneur via l'API GitHub et les
métadonnées locales pour l'identité, la généalogie, l'autodiagnostic, l'historique
des contributeurs et la traçabilité du build. Opérations en lecture seule uniquement.

### container-maintainer

Installe, supprime, recherche et met à jour des outils CLI dans le conteneur
en cours d'exécution en utilisant les modèles appropriés du gestionnaire de paquets. Crée des tickets GitHub
dans le dépôt du conteneur de développement pour le
suivi de la persistance.
