Le plugin **f5xc-devcontainer** fournit une conscience de conteneur pour
l'environnement de développement f5xc-salesdemos. Il maintient un
catalogue d'outils complet, permet l'introspection de conteneur en direct,
et gère l'installation d'outils avec suivi de persistance via
les issues GitHub.

<Badge text="v1.1.6" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-devcontainer@f5xc-salesdemos-marketplace
```

## Compétences

### tool-catalog

Base de connaissances de chaque outil CLI installé dans le conteneur de développement,
organisée par catégorie (cloud, réseau, sécurité, développement,
etc.). S'active lorsque vous demandez quel outil utiliser, comment utiliser
un outil, si quelque chose est installé, ou si vous avez besoin d'aide pour
une opération spécifique.

### self-awareness

Compétence d'identité et d'autodiagnostic du conteneur. Fournit une
introspection en direct du conteneur en cours d'exécution via l'API GitHub et les
métadonnées de build locales. S'active pour les questions d'identité, de version, d'informations
de build, de vérifications de santé et d'historique des contributeurs.

<Aside type="tip">
  Demandez « qui êtes-vous ? » ou « quelle version est en cours d'exécution ? » pour déclencher
  la conscience de soi.
</Aside>

## Agents

### tool-advisor

Lit le catalogue d'outils et retourne des recommandations avec la finalité,
les commandes de démarrage rapide et les exigences d'authentification pour toute question
relative aux outils CLI.

### tool-auditor

Audite le Dockerfile du conteneur de développement par rapport aux fichiers de référence du catalogue d'outils
pour détecter les dérives &mdash; outils ajoutés à l'image
mais non répertoriés, ou entrées de catalogue pour des outils qui ont été
supprimés.

### container-introspector

Effectue une introspection en direct du conteneur via l'API GitHub et les
métadonnées locales pour l'identité, la généalogie, l'autodiagnostic, l'historique
des contributeurs et la traçabilité des builds. Opérations en lecture seule uniquement.

### container-maintainer

Installe, supprime, recherche et met à jour les outils CLI dans le conteneur
en cours d'exécution en utilisant les schémas de gestionnaire de paquets appropriés. Crée des issues GitHub
dans le dépôt du conteneur de développement pour le suivi de persistance.
