Le plugin **f5xc-github-ops** fournit une Automatisation exclusive des opérations GitHub
pour les dépôts f5-sales-demo. Il gère l'intégralité du
cycle de vie Git et GitHub, de la création des issues jusqu'à la
surveillance post-fusion, en appliquant le développement piloté par les issues et les
contrôles de lint avant la validation.

<Badge text="v2.2.2" variant="note" /> <Badge text="Productivity" variant="tip" />

## Installation

```
/plugin install f5xc-github-ops@f5-sales-demo-marketplace
```

## Compétences

### workflow-lifecycle

Flux de travail de gouvernance de dépôt gérant la création des issues, les
conventions de nommage des branches, la création des PR, l'interrogation CI avec retour
d'erreurs vers les issues, la surveillance post-fusion, la vérification et les
critères de complétion des tâches. S'active lorsque vous dites « commit », « push »,
« create a PR », « merge », ou lors de toute opération Git.

### upstream-contribution

Flux de travail de contribution en amont pour le développement basé sur les forks. Gère
la détection des forks, la recherche en amont (issues, PR, CONTRIBUTING.md,
patterns CI), la création d'issues et de PR inter-dépôts, la gestion locale des
issues de suivi, la surveillance du CI en amont et la résolution guidée.

## Agents

### github-ops

Agent exclusif pour toutes les opérations Git et GitHub mécaniques.
Gère les contrôles de lint avant validation, la création des issues, la création des branches,
la mise en scène, la validation, le push, la création des PR, l'interrogation CI avec retour
d'erreurs, la relance en cas d'échec d'infrastructure, la surveillance post-fusion,
le nettoyage des branches et la gestion des paramètres du dépôt via l'
API GitHub.

Retourne des rapports de statut structurés : `COMPLETE`,
`PRE_COMMIT_FAILED`, `CI_FAILED`, `BLOCKED` ou `FAILED`.

<Aside type="caution">
  Cet agent est le gestionnaire **exclusif** des opérations Git.
  N'exécutez jamais `git commit`, `git push` ou `gh pr create` directement
  dans les dépôts qui utilisent ce plugin.
</Aside>
