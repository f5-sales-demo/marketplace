Le plugin **salesforce** intègre Salesforce CLI avec Claude Code,
en fournissant une authentification d'organisation et un agent CLI polyvalent capable
d'exécuter n'importe quelle commande `sf`, y compris les requêtes SOQL. Il prend en charge
les environnements de conteneurs sans interface graphique avec les flux d'authentification JWT,
jeton d'accès et URL SFDX. Le guide d'utilisation ci-dessous explique comment utiliser
le langage naturel pour interroger les comptes, opportunités, requêtes et contacts
via l'agent CLI.

<Badge text="v1.0.0" variant="note" /> <Badge text="Development" variant="tip" />

## Installation

```
/plugin install salesforce@f5xc-salesdemos-marketplace
```

## Prérequis

- **Salesforce CLI** (`@salesforce/cli`)
- **Organisation Salesforce** avec accès à l'API
- Un compte utilisateur avec les autorisations pour interroger les objets (Cases, Opportunities, Accounts, Contacts)

Installez Salesforce CLI sur votre poste de travail :

```bash
brew install sf
```

Vérifiez l'installation :

```bash
sf --version
```

Vous devriez voir une sortie similaire à `@salesforce/cli/2.x.x`.

## Configuration de l'authentification

### Étape 1 : Trouver votre domaine Salesforce

Regardez votre URL Salesforce dans le navigateur. Si vous accédez à Salesforce via
`https://acme.lightning.force.com`, votre domaine de connexion est
`https://acme.my.salesforce.com`.

<Aside type="tip">
  Si votre entreprise utilise SSO, vous devez spécifier votre domaine personnalisé avec
  `--instance-url` afin que la connexion soit redirigée via votre fournisseur SSO
  plutôt que vers la page de connexion Salesforce générique.
</Aside>

### Étape 2 : S'authentifier via le navigateur

```bash
sf org login web --alias my-org --set-default --instance-url https://YOUR-DOMAIN.my.salesforce.com
```

Remplacez `YOUR-DOMAIN` par le domaine de votre entreprise indiqué à l'étape 1. Votre navigateur
s'ouvre — suivez le flux SSO ou de connexion et autorisez l'application. Vous devriez
voir :

```
Successfully authorized your-email@company.com with org ID 00DXXXXXXXXXXXXXXX
```

### Étape 3 : Vérifier la connexion

```bash
sf org display --target-org my-org
```

Confirmez que la sortie affiche `Connected Status: Connected` avec votre nom d'utilisateur
et l'URL de l'instance.

### Étape 4 : Authentification portable pour les conteneurs (facultatif)

Si vous devez vous authentifier dans un environnement sans interface graphique (CI/CD, conteneur,
session distante), exportez l'URL d'authentification SFDX depuis votre poste de travail :

```bash
sf org display --verbose --target-org my-org
```

Copiez la valeur **Sfdx Auth Url** (commence par `force://`). Dans le
conteneur, exécutez :

```bash
export SFDX_AUTH_URL="force://PlatformCLI::YOUR_AUTH_TOKEN@your-domain.my.salesforce.com"
echo "$SFDX_AUTH_URL" | sf org login sfdx-url --sfdx-url-stdin=- --alias=my-org --set-default
```

<Aside type="caution">
  L'URL d'authentification SFDX est un identifiant — traitez-la comme un mot de passe. Ne la
  committez pas dans des fichiers et ne la collez pas dans une conversation. Utilisez uniquement
  des variables d'environnement et des pipes stdin.
</Aside>

### Référence des méthodes d'authentification

| Méthode | Idéal pour | Nécessite |
|--------|----------|----------|
| Connexion web | Postes de travail avec un navigateur | Navigateur + SSO |
| URL SFDX | Conteneurs, CI/CD, authentification portable | URL d'authentification d'une session authentifiée |
| JWT Bearer | Pipelines automatisés | Application connectée + clé privée + clé consommateur |
| Jeton d'accès | Authentification par variable d'environnement | `SF_ACCESS_TOKEN` + `SF_ORG_INSTANCE_URL` |

**Remarque :** Le flux appareil (`sf org login device`) est bloqué depuis août
2025 et n'est pas pris en charge.

### Variables d'environnement

| Variable | Objectif |
|----------|---------|
| `SF_ACCESS_TOKEN` | Jeton Bearer pour l'authentification par jeton d'accès |
| `SFDX_AUTH_URL` | URL d'authentification Force pour l'authentification par URL SFDX |
| `SF_ORG_INSTANCE_URL` | URL de l'instance de l'organisation |
| `SF_JWT_KEY_FILE` | Chemin vers la clé privée JWT |
| `SF_CLIENT_ID` | Clé consommateur de l'application connectée |
| `SF_USERNAME` | Nom d'utilisateur Salesforce pour JWT |

## Commandes

### /salesforce:sf-status

Vérifiez l'état de la connexion à votre organisation Salesforce, la version CLI et
les organisations authentifiées.

```
/salesforce:sf-status
```

**Ce à quoi s'attendre :** Un tableau affichant la version de votre CLI sf, l'alias de l'organisation
authentifiée, le nom d'utilisateur, l'URL de l'instance, l'état de connexion et la version de l'API.

### /salesforce:sf-login

Authentifiez-vous auprès d'une organisation Salesforce. Vous pouvez éventuellement fournir un alias.

```
/salesforce:sf-login
/salesforce:sf-login my-org
```

**Ce à quoi s'attendre :** Si une organisation avec cet alias est déjà authentifiée,
les détails de connexion sont affichés. Sinon, les identifiants disponibles
(variables d'environnement) sont vérifiés et l'authentification s'effectue via la première
méthode entièrement satisfaite.

## Compétences

### salesforce-index

Routeur d'intention de premier niveau. S'active automatiquement lorsque vous mentionnez
Salesforce, la gestion d'organisation ou des requêtes de données en langage naturel.

| Vous dites | Redirige vers |
|---------|-----------|
| « login to salesforce », « authenticate org » | Compétence `salesforce-auth` |
| « check org status », « list orgs » | Agent `cli-operator` |
| « show me opportunities », « look up case » | Agent `cli-operator` (requête SOQL) |
| « generate Apex class », « create Flow » | Compétences de développement afv-library |

### salesforce-auth

Compétence d'authentification adaptée aux conteneurs. Prend en charge quatre méthodes d'authentification et
détecte automatiquement quels identifiants sont disponibles. Priorité :
jeton d'accès, puis JWT, puis URL SFDX, puis connexion web.

## Agents

### cli-operator

Agent Salesforce CLI autonome qui exécute des commandes `sf` avec des
garde-fous de sécurité. Toutes les opérations Salesforce sont déléguées à cet agent pour
maintenir le contexte de session principal allégé.

**Règles de sécurité :**

- Lecture seule par défaut — les opérations d'écriture nécessitent une approbation explicite
- Ne déploie jamais sans étape de prévisualisation et de confirmation
- N'affiche jamais les jetons d'accès, les URL d'authentification ou les identifiants dans la sortie
- Assainit toutes les valeurs fournies par l'utilisateur (rejette les métacaractères shell)
- Utilise la sortie `--json` pour des résultats structurés et analysables

**Format de réponse :**

```
## Result: [SUCCESS | FAILURE | PARTIAL]
### Command Executed
### Output Summary
### Issues
```

## Guide d'utilisation : Gestion des comptes

<Aside type="note">
  Les invites ci-dessous ne sont pas des fonctionnalités codées en dur du plugin. Elles fonctionnent
  parce que l'agent cli-operator peut exécuter n'importe quelle commande `sf` CLI, y compris
  des requêtes SOQL arbitraires. Les résultats dépendent du modèle de données de votre organisation et
  de vos autorisations utilisateur. Les invites ici ont été testées et vérifiées
  pour fonctionner avec les objets Salesforce standard (Account, Contact, Case,
  Opportunity, AccountTeamMember, OpportunityTeamMember).
</Aside>

Ces invites fonctionnent avec n'importe quelle organisation Salesforce. Remplacez les valeurs
fictives par vos propres informations. Chaque invite produit des résultats spécifiques
à votre rôle, vos comptes et votre territoire.

<Aside type="tip">
  Pour de meilleurs résultats, incluez votre adresse e-mail dans les invites qui nécessitent
  d'identifier votre enregistrement utilisateur. Les organisations Salesforce peuvent avoir plusieurs utilisateurs
  avec des noms similaires.
</Aside>

### Premiers pas

Après l'authentification, vérifiez que tout fonctionne :

```
/salesforce:sf-status
```

Puis essayez une requête simple en langage naturel :

```
list my authenticated salesforce orgs
```

**Ce à quoi s'attendre :** Un tableau affichant toutes les organisations connectées avec l'alias,
le nom d'utilisateur, l'URL de l'instance et l'état de connexion.

### Trouver vos comptes

Découvrez les comptes auxquels vous êtes affecté :

```
what salesforce accounts am I on the account team for? My email is your-email@company.com
```

**Ce à quoi s'attendre :** Une liste de comptes regroupés par propriétaire de compte, affichant
votre rôle de membre d'équipe sur chacun. Si la plupart des rôles apparaissent vides, il s'agit
d'un problème courant de qualité des données — les appartenances à l'équipe de compte existent mais les rôles
n'ont pas été renseignés.

### Recoupement de la couverture avec un collègue

Comparez votre couverture de compte avec celle d'un collègue pour identifier les lacunes :

```
In salesforce, find all accounts where Colleague Name is on the account team. Then for each of those accounts, check if your-email@company.com is also on the account team. Show me two lists: accounts where we are BOTH tagged, and accounts where only my colleague is tagged but I am missing.
```

**Ce à quoi s'attendre :** Un tableau récapitulatif affichant le nombre de chevauchements et de lacunes,
suivi de deux listes. Si vous avez récemment changé de rôle, vous pourriez trouver zéro
chevauchement — cela révèle les comptes auxquels vous devez être ajouté.

<Aside type="note">
  Vous pouvez également vérifier la présence d'un prédécesseur en ajoutant : « Also check if
  Previous Person Name appears on any of those accounts. » Cela aide
  à identifier les lacunes de passation lors du remplacement de quelqu'un sur un territoire.
</Aside>

### Aperçu du pipeline territorial

Consultez toutes les opportunités ouvertes sur vos comptes :

```
show me all open salesforce opportunities on Colleague Name's account team accounts, sorted by close date, include the stage, amount, and probability
```

**Ce à quoi s'attendre :** Un résumé avec le nombre total d'opportunités, le pipeline brut
et le pipeline pondéré. Puis un tableau de distribution par étape et une liste des meilleures
opportunités par montant. La sortie signale les problèmes de qualité des données tels que
les dates de clôture dépassées ou les montants nuls.

### Analyse approfondie d'une opportunité

Obtenez une vue complète d'une transaction spécifique :

```
show me a detailed view of the OPPORTUNITY NAME opportunity in salesforce - include the opportunity team members, any activities or tasks, and the account contacts
```

**Ce à quoi s'attendre :** L'aperçu de l'opportunité (étape, montant, date de clôture,
probabilité, catégorie de prévision, propriétaire), les membres de l'équipe opportunité avec
leurs rôles, les contacts associés, les tâches et événements récents, et l'historique des modifications
de champ montrant comment le montant et la date de clôture de la transaction ont évolué au fil du temps.

### Requêtes de support sur le territoire

Vérifiez les requêtes de support ouvertes sur vos comptes :

```
show me all open salesforce cases across Colleague Name's account team accounts, grouped by account, sorted by most recent first
```

**Ce à quoi s'attendre :** Un nombre de requêtes ouvertes avec la répartition par priorité et statut,
puis les détails des requêtes regroupés par compte. Les requêtes anciennes (âgées de plusieurs mois ou
années sans activité) sont signalées comme candidates à un nettoyage des données.

### Prévision de pipeline trimestrielle

Obtenez une vue de votre pipeline prête pour les prévisions :

```
for Colleague Name's accounts, show me a quarterly pipeline summary - group the open opportunities by close date quarter with count, total amount, and weighted amount for each quarter
```

**Ce à quoi s'attendre :** Un tableau avec une ligne par trimestre affichant le nombre d'opportunités,
le pipeline total et le pipeline pondéré. Inclut une répartition du mix d'étapes par trimestre
et met en évidence les meilleures transactions pondérées. Les opportunités en retard sont regroupées séparément.

### Aperçu complet d'un compte

Analyse approfondie d'un client spécifique :

```
give me a full account overview for ACCOUNT NAME in salesforce including contacts, open opportunities, and recent cases
```

**Ce à quoi s'attendre :** Profil de l'entreprise (secteur, revenus, employés,
localisation), contacts clés avec titres et e-mail, opportunités ouvertes avec
étapes et montants, et requêtes de support récentes avec statut.

### Recherche de requête

Recherchez une requête de support spécifique par numéro :

```
look up salesforce case CASE-NUMBER and show me the case details, the customer account, and who owns it
```

**Ce à quoi s'attendre :** Objet de la requête, statut, priorité, description, le
profil du compte client et le propriétaire de la requête avec ses coordonnées.

## Guide d'utilisation : Requêtes de données

Pour les utilisateurs avancés, vous pouvez demander des requêtes de style SOQL spécifiques en
langage naturel. Le plugin traduit votre demande en SOQL approprié et l'exécute.

### Champs spécifiques

```
query salesforce for all Contacts at ACCOUNT NAME - show Name, Title, Email, Phone, and Department
```

### Filtrage par date

```
show me all salesforce opportunities that closed won in the last 90 days on ACCOUNT NAME
```

### Agrégation

```
count all open salesforce cases grouped by priority and status across my accounts
```

### Objets personnalisés

```
query the CUSTOM_OBJECT__c object in salesforce for records where Status__c = 'Active'
```

<Aside type="tip">
  Si une requête échoue parce qu'un champ n'existe pas, le plugin
  effectue automatiquement une nouvelle tentative sans ce champ. Les champs personnalisés dans votre organisation
  peuvent utiliser des noms différents — vérifiez votre configuration Salesforce si un champ
  est introuvable.
</Aside>
