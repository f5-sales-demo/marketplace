Une **place de marché de plugins Claude Code** est une collection organisée de
plugins qui étendent Claude Code avec de nouvelles compétences, commandes et
agents. Les places de marché permettent aux équipes de partager des outils afin
que chaque développeur dispose des mêmes capacités sans configuration manuelle.

La **Place de marché f5-sales-demo** fournit des plugins conçus pour
la [plateforme de documentation f5-sales-demo](https://github.com/f5-sales-demo)
— un écosystème de plus de 20 dépôts qui
publient des sites Astro/Starlight sur GitHub Pages à l'aide d'un pipeline de
build partagé.

## Fonctionnement

1. **Ajoutez la place de marché** à Claude Code — cela l'enregistre
   en tant que source de plugins
2. **Installez des plugins** depuis la place de marché — chaque plugin ajoute
   des compétences et des commandes à votre session Claude Code
3. **Utilisez les outils** — les plugins s'activent automatiquement en fonction du
   contexte, ou vous invoquez leurs commandes directement

## Démarrage rapide

Ajoutez la place de marché et installez le plugin d'outils de documentation :

```
/plugin marketplace add f5-sales-demo/marketplace
/plugin install f5xc-docs-tools@f5-sales-demo-marketplace
```

Consultez [Démarrage](/marketplace/getting-started/) pour un
guide complet.

## Architecture

Une place de marché est un dépôt Git qui contient un ou plusieurs
plugins. Chaque plugin regroupe des compétences, des commandes et des agents
dans une unité installable unique.

```
Marketplace (marketplace.json)
└── Plugin (plugin.json)
    ├── Skills (SKILL.md)
    │   └── References (data files)
    ├── Commands (.md)
    └── Agents (.md)
```

- **Place de marché** — le registre. Répertorie les plugins disponibles
  avec les métadonnées nécessaires à leur découverte et leur installation.
- **Plugin** — l'unité installable. Contient une ou plusieurs
  compétences, commandes ou agents.
- **Compétence** — connaissance contextuelle que Claude active
  automatiquement en fonction de la tâche en cours.
- **Commande** — une commande slash que l'utilisateur invoque directement
  (par ex., `/review-mdx`).
- **Agent** — un sous-agent spécialisé qui gère des tâches complexes et
  multi-étapes de manière autonome.
